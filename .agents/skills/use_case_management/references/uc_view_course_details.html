UC ID and Name UC-ViewCourseDetails - View Course Details
Created By: System Date Created: 09/06/2026
Primary Actor: User, Guest, Instructor, Staff, Admin Secondary Actors: N/A
Trigger: Actor clicks on a course from the public course list, category page, or search results.
Description: Allows an actor to view the detailed information of a course, including its description, curriculum, instructor profile, and student reviews. The system determines the visibility of course content and available actions (e.g., Add to cart, Enroll Now, Go to learning) based on the actor's authentication and enrollment status.
Preconditions:

None.
Post-conditions:

POST-1: Detailed course description, pricing card, interactive syllabus, instructor cards, and review modules render successfully on the frontend.

Normal Flow:

A. View Course Details Successfully

  1. Actor accesses the course details page.
  2. The system retrieves the course details.
  3. The system validates the actor's permission to view the course.
  4. The system checks the actor's wishlist status for the course.
  5. The system retrieves the course review statistics and feedback.
  6. The system displays the course details page with appropriate action buttons.
Alternative Flows:

A.3 View Unpublished Course (Instructor/Admin/Enrolled User)

  1. The system detects the course is not published.
  2. The system detects the actor is the course owner, an administrator, or an already enrolled student.
  3. The system displays a warning banner indicating the course's unpublished state.
  4. Back to step 4.

A.6.1 View Free Course Actions

  1. The system detects the course price is zero and the actor is not enrolled.
  2. The system displays the Enroll Now button.
  3. Back to step 6.

A.6.2 View Enrolled Course Actions

  1. The system detects the actor is already enrolled in the course.
  2. The system displays the Go to learning button.
  3. Back to step 6.
Exceptions:

EX-2 Course Not Found

  1. The system detects the course does not exist.
  2. The system redirects the actor to the Not Found page.

EX-3 Unauthorized Access (Unpublished Course)

  1. The system detects the course is not published.
  2. The system detects the actor is not the instructor, an administrator, or an enrolled student.
  3. The system redirects the actor to the Error page.
Priority: Medium
Frequency of Use: High
Business Rules:
  1. Curriculum Locking: Only the first material of the first lesson is available for preview to non-enrolled actors. All subsequent materials are visually locked and cannot be accessed from the details page.
  2. Unpublished Course Access Restrictions: Courses with statuses other than "published" (e.g., draft, pending, archived) are strictly hidden from the public and can only be viewed by the original instructor, system administrators, or students who were previously enrolled.
  3. Review Visibility: Course reviews and rating statistics are publicly visible to all actors.
  4. Wishlist State: The wishlist toggle option is only displayed to users. Instructors viewing their own course and system administrators cannot add courses to their wishlist.
  5. Course Reporting: The option to report a course is only visible to enrolled students. It is hidden from the course owner, administrators, and for courses that are pending or already permanently archived.
Other Information: N/A
Assumptions:

Actor has stable internet connection.
