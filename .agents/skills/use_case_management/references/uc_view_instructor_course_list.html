UC-ViewInstructorCourseList - View Instructor's Course List

Primary Actor:
Instructor

Secondary Actors:
None

Trigger:
Instructor navigates to the Instructor Dashboard.

Description:
Allows an approved instructor to view their course portfolio, overall performance statistics, and manage individual courses from a centralized dashboard. The system ensures only fully approved instructors can access this view, redirecting others to appropriate application workflows.

Preconditions:

PRE-1. The actor must be authenticated.

PRE-2. The actor must have an approved instructor profile.

Post-conditions:

POST-1. The system returns a paginated list of the instructor's courses along with their performance statistics.

Normal Flow

A. View Dashboard Successfully

  1. Instructor accesses the Instructor Dashboard page.
  2. The system validates the instructor's authentication and approval status.
  3. The system retrieves the instructor's overall performance statistics (e.g., total enrollment, average rating, active courses, total revenue).
  4. The system retrieves the instructor's paginated course list.
  5. The system displays the Instructor Dashboard, rendering the performance statistics and the course list.

Alternative Flows

A.4 Empty Course List

  1. The system detects the retrieved course list is empty.
  2. The system displays an empty state message indicating that no courses have been created.
  3. Use case ends.

Exceptions

EX-2.1 Unauthenticated Access

  1. The system detects the actor is not authenticated.
  2. The system redirects the actor to the Login page.

EX-2.2 Instructor Application Pending or Rejected

  1. The system detects the actor's instructor profile is not approved (e.g., pending or rejected).
  2. The system redirects the actor to the Application Status page.

EX-2.3 Instructor Profile Not Found

  1. The system detects the actor does not have an instructor profile.
  2. The system redirects the actor to the Instructor Application page.

Business Rules

  1. Approval Enforcement: The system strictly enforces that only fully approved instructors can access the dashboard. All other states (no profile, pending review, rejected) trigger immediate redirects.
  2. Status Badges: The system visually distinguishes courses using specific badges based on their current status (e.g., Published, Pending Review, Draft, Rejected, Archived, Removed, Permanently Locked).
  3. Contextual Actions: Available course actions (Edit, Hide, Unhide, Delete) are determined dynamically based on the course status. Permanently locked or removed courses have severely restricted action sets.
  4. Data Isolation: Instructors can only view statistics, revenue, and courses associated with their own account.
  5. Empty State Display: If the course list is empty, the system must show a clear empty state message instead of an empty table or blank space.

Assumptions

Instructor has a stable internet connection.
