UC ID and Name UC-02-View Course List
Created By: Truong Phuoc Tai Date Created: 10/06/2026
Primary Actor: User, Guest, Instructor, Staff, Admin Secondary Actors: N/A
Trigger: Actor accesses the explore courses page or searches for courses.
Description: Allows Guests and Users to browse, search, filter, and sort the public list of courses on the marketplace. Authenticated Users can toggle course wishlist status, add courses to their shopping cart, or view their enrollment status directly from the course cards.
Preconditions:

PRE-1. None.
Post-conditions:

POST-1. Detailed course card catalog, including thumbnail, title, category, instructor name, rating average, price, wishlist toggle, and cart action controls, is rendered successfully matching the active filter criteria.
Normal Flow:

A. Explore Courses Successfully

  1. Actor accesses the explore courses page.
  2. The system retrieves the active filter categories.
  3. The system retrieves the default paginated course list.
  4. The system checks the actor's authentication status.
  5. The system retrieves the actor's wishlist (if authenticated).
  6. The system maps wishlist status to each course in the list.
  7. The system displays the explore courses page showing the categories, filter options, and the course list.
Alternative Flows:

A.2.1 Select Category Pill

  1. Actor clicks on a category pill (e.g., a specific category name or the "All Categories" pill).
  2. The system updates the category filter parameter.
  3. Back to step 3.

A.3 Empty Catalog (No Courses Found)

  1. The system detects the retrieved course list is empty.
  2. The system displays a "No courses found" state message.
  3. The system renders a "Clear All Filters" button.
  4. Back to step 7.

A.3.1 Clear All Filters from Empty State

  1. Actor clicks the "Clear All Filters" button in the empty state block.
  2. The system resets all search keywords, categories, and advanced filters.
  3. Back to step 3.

A.7.1 Search by Keyword

  1. Actor enters a search term in the search input box.
  2. The system triggers the search after a brief debounce delay.
  3. Back to step 3.

A.7.2 Clear Search Keyword

  1. Actor clicks the close icon ("close" symbol button) next to the search input.
  2. The system clears the search input field.
  3. Back to step 3.

A.7.3 Select Sorting Option

  1. Actor selects a sorting option (e.g., "Newest", "Highest Rated", "Price: Low to High", or "Price: High to Low").
  2. The system updates the sort order parameter.
  3. Back to step 3.

A.7.4 Filter by Rating

  1. Actor selects a rating radio option under the "Rating" accordion (e.g., "4.5" and up, "4.0" and up, "3.5" and up, or "All Ratings").
  2. The system updates the rating filter parameter.
  3. Back to step 3.

A.7.5 Filter by Price

  1. Actor selects a price radio option under the "Price" accordion (e.g., "Free", "Paid", or "All Prices").
  2. The system updates the price filter parameter.
  3. Back to step 3.

A.7.6 Toggle Filter Accordion

  1. Actor clicks the toggle icon next to the accordion header (e.g., "Rating" or "Price" accordion).
  2. The system toggles the visibility of the accordion filter options.
  3. Back to step 7.

A.7.7 Apply Filters Manually

  1. Actor clicks the "Apply Filters" button.
  2. The system submits the filter form.
  3. Back to step 3.

A.7.8 Clear Filters

  1. Actor clicks the "Clear Filters" button.
  2. The system resets all search keyword, category, price, and rating inputs to their default states.
  3. Back to step 3.

A.7.9 Remove Active Filter Badge

  1. Actor clicks the close icon ("close" symbol button) on an active filter badge.
  2. The system resets the corresponding filter.
  3. Back to step 3.

A.7.10 Switch View Mode

  1. Actor clicks the view switcher icon (e.g., Grid view "grid_view" or List view "view_list" icon).
  2. The system updates the layout format of the course catalog and saves the preference locally.
  3. Back to step 7.

A.7.11 Add Course to Cart

  1. Actor clicks the "Add to cart" button ("add_shopping_cart" icon) on a course card.
  2. The system validates that the actor is an authenticated User.
  3. The system adds the course to the user's shopping cart.
  4. The system updates the shopping cart item count in the header.
  5. Back to step 7.

A.7.12 Toggle Wishlist Status

  1. Actor clicks the Wishlist Heart button ("favorite" icon) on a course card.
  2. The system validates that the actor is an authenticated User.
  3. The system toggles the course's wishlist state.
  4. The system displays a confirmation notification regarding the wishlist update.
  5. The system updates the styling of the Wishlist Heart button.
  6. Back to step 7.

A.7.13 Navigate Pagination

  1. Actor clicks a page link or page navigation control (e.g., page number, left chevron "chevron_left", or right chevron "chevron_right").
  2. The system updates the current page index.
  3. The system scrolls the viewport smoothly back to the top of the course catalog.
  4. Back to step 3.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.
Priority: High
Frequency of Use: High
Business Rules:
  1. Public Visibility: The course list displays only courses that are currently published. Draft, pending, rejected, archived, or locked courses are excluded from the public list.
  2. Wishlist Sync: Wishlist heart icons reflect the current wishlist status of the logged-in User in real-time, displaying a filled red heart for wishlisted courses and an unfilled icon otherwise.
  3. Cart Status Integration: If a logged-in User has already purchased/enrolled in a course, the "Add to cart" button is replaced by a checkmark icon to prevent double purchases.
  4. Debounced Live Search: Real-time keyword filtering is debounced by 350ms to minimize API calls during continuous typing.
  5. View Mode Retention: User layout preference (Grid vs. List view) is stored in the browser's local storage and persists across sessions.
  6. State Persistence: Active filter parameters are stored in the browser's local storage and are automatically applied when the user returns to the page without URL parameters.
Other Information: N/A
Assumptions:

User, Guest, Instructor, Staff, Admin has stable internet connection.
