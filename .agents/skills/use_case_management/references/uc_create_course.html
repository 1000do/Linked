UC ID and Name UC-52 - Create Courses
Created By: System Date Created: 09/06/2026
Primary Actor: Instructor Secondary Actors: Cloud Storage Service
Trigger: Instructor clicks the "Create New Course" button in the Instructor Management dashboard.
Description: Allows an instructor to initialize a new course by providing basic metadata, pricing, and instructional goals. The system creates the course with a draft status and computes a unique content fingerprint for duplication tracking.
Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.
Post-conditions:

POST-1. A new course is created in the database with draft status.

POST-2. A content fingerprint is computed and stored alongside the course record.

POST-3. The instructor is notified of the successful course creation.
Normal Flow:

A. Create Course Successfully

  1. Instructor accesses the course creation form.
  2. Instructor enters the required course details.
  3. Instructor uploads a thumbnail image to the cloud storage service.
  4. Instructor sets the course price.
  5. Instructor clicks the "Create Course" button.
  6. The system validates the form inputs.
  7. The system validates the instructor's rights.
  8. The system creates the course record with draft status in the database.
  9. The system retrieves the thumbnail from the cloud storage service and computes and stores the content fingerprint for the course.
  10. The system displays a success confirmation message to the instructor.
Alternative Flows:

A.6 Invalid Form Input

  1. The system detects one or more form inputs are invalid.
  2. The system displays a validation error message.
  3. Back to step 2.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-7 Invalid Instructor Rights

  1. The system detects the instructor's rights are invalid.
  2. The system displays an error message.
  3. Back to step 1.

EX-8.1 Duplicate Course Content Detected

  1. The system detects a duplication in course content using the computed fingerprint.
  2. The system displays an error message regarding content duplication.
  3. Back to step 1.

EX-8.2 Database Insert Failed

  1. The system fails to save the course record or its fingerprint to the database.
  2. The system displays an error message regarding database insertion failure.
  3. Back to step 1.
Priority: Medium
Frequency of Use: High
Business Rules:
  1. Free Course Limit for Unlinked Stripe: Instructors who have not linked an active Stripe account are limited to creating a maximum of 2 courses.
  2. Price Enforcement for Unlinked Stripe: If the instructor has not linked an active Stripe account, the system automatically sets the course price to 0 regardless of the submitted value.
  3. Content Integrity Hashing: Textual details and the thumbnail file are hashed using MD5 and stored as a unique fingerprint to prevent duplication and support anti-plagiarism tracking.
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
Cloud Storage Service is working properly.
