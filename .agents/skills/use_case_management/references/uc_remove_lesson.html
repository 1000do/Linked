UC ID and Name UC-RemoveLesson - Remove Lesson
Created By: System Date Created: 09/06/2026
Primary Actor: Instructor Secondary Actors: Cloud Storage Service
Trigger: Instructor clicks the "Remove Lesson" icon for a lesson in the course syllabus.
Description: Allows an instructor to remove an existing lesson from a course. The system requires confirmation, validates business rules (such as pending or archived statuses), soft deletes the lesson and its associated materials, and moves any cloud materials to the trash folder.
Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The course must exist and belong to the active instructor.
Post-conditions:

POST-1. The lesson and its associated materials are marked as removed.

POST-2. Associated cloud materials are moved to the trash folder.

POST-3. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

POST-4. Stale cache data is cleared.
Normal Flow:

A. Remove Lesson Successfully

  1. Instructor accesses the course editor.
  2. Instructor clicks the "Remove Lesson" icon for a lesson.
  3. The system displays a confirmation modal.
  4. Instructor clicks the "Delete" button.
  5. The system validates the instructor's rights.
  6. The system validates the course status.
  7. The system moves associated cloud materials to the trash folder.
  8. The system updates the materials' learning status to removed.
  9. The system updates the lesson status to removed, updates the course status to draft, and clears previous moderation feedback in the database.
  10. The system invalidates the cached course data.
  11. The system displays a success message.
  12. The system removes the lesson from the UI curriculum list.
Alternative Flows:

A.4 Cancel deletion

  1. Instructor clicks outside the modal or clicks the cancel button.
  2. The system closes the confirmation modal.
  3. Back to step 2.

A.9 Course is not published

  1. The system skips updating the course status and moderation feedback.
  2. Back to step 10.
Exceptions:

EX-1.1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-1.2 Lesson Not Found

  1. The system halts execution and displays a lesson not found error message.

EX-5 Invalid Instructor Rights

  1. The system detects the instructor is unauthorized to modify this lesson or has an active lockout status.
  2. The system displays an error message regarding invalid instructor rights.
  3. Back to step 2.

EX-6 Invalid Course Status

  1. The system detects the course is pending review or is permanently archived due to policy violations.
  2. The system displays an error message regarding invalid course status.
  3. Back to step 2.

EX-9 Database Save Failed

  1. The system fails to save the lesson changes to the database.
  2. The system displays a database transaction failure message.
  3. Back to step 2.
Priority: Medium
Frequency of Use: High
Business Rules:
  1. Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from deleting lessons.
  2. Pending Restriction: Lessons cannot be deleted from a course while it is currently in 'Pending' review status.
  3. Archived Course Restriction: Courses that are permanently archived due to policy violations (flag count >= 3) cannot be modified.
  4. Soft Deletion: The lesson and its associated materials are marked as removed rather than permanently deleted. Cloud materials are moved to the trash folder.
  5. Status Reversion: Deleting a lesson from a 'Published' course automatically changes its status back to 'Draft' and clears any existing moderation feedback.
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
Cloud Storage Service is working properly.
