UC-RemoveCourse - Remove Course

Primary Actor:
Instructor

Secondary Actors:
None

Trigger:
Instructor clicks the "Delete course" button for a course in the dashboard list view.

Description:
Allows an instructor to remove an existing course. The system requires confirmation, validates instructor rights and status, verifies that the course is not pending review and has no active student enrollments, soft deletes the course and all associated lessons, and invalidates the cached course data.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The course must exist and belong to the active instructor.

Post-conditions:

POST-1. The course is marked as removed in the database.

POST-2. All lessons associated with the course are marked as removed in the database.

POST-3. Cached course detail data is cleared.

Normal Flow

A. Remove Course Successfully

  1. Instructor accesses the instructor dashboard.
  2. Instructor clicks the "Delete course" button for a course.
  3. The system displays a deletion confirmation modal.
  4. Instructor clicks the "Delete" button.
  5. The system validates the instructor's rights.
  6. The system validates the course status.
  7. The system checks if the course has active student enrollments.
  8. The system marks the course as removed in the database.
  9. The system marks all associated lessons as removed in the database.
  10. The system invalidates the cached course data.
  11. The system reloads the page to update the course list.

Alternative Flows

A.2 Course has active students (Frontend check)

  1. The system detects that the course has active student enrollments.
  2. The system displays a warning message stating the course cannot be deleted because of enrolled learners.
  3. Instructor clicks the OK button.
  4. The system closes the warning dialog.
  5. Back to step 2.

A.4 Cancel deletion

  1. Instructor clicks the Cancel button or clicks outside the confirmation modal.
  2. The system closes the confirmation modal.
  3. Back to step 2.

Exceptions

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-5 Invalid Instructor Rights

  1. The system detects the instructor is unauthorized to delete this course or has an active lockout status.
  2. The system displays an error message regarding invalid instructor rights.
  3. Back to step 2.

EX-6 Invalid Course Status

  1. The system detects the course does not exist, is pending review, or is permanently discontinued due to policy violations.
  2. The system displays an error message regarding invalid course status.
  3. Back to step 2.

EX-7 Course Has Enrolled Students (Backend check)

  1. The system detects that the course has active student enrollments.
  2. The system displays a failure message stating the course cannot be deleted with active students.
  3. Back to step 2.

EX-8 Database Save Failed

  1. The system fails to save the course changes to the database.
  2. The system displays a database transaction failure message.
  3. Back to step 2.

Business Rules

  1. Locked Out Instructor Rights: Instructors with active lockout status are prohibited from deleting courses.
  2. Discontinued Course Restriction: Courses that are permanently discontinued due to policy violations (flag count >= 3 and status is archived) cannot be deleted by the instructor.
  3. Pending Restriction: Courses cannot be deleted while they are pending review.
  4. Active Student Restriction: Courses with active student enrollments cannot be deleted and must be archived instead.
  5. Soft Deletion: Deleting a course marks both the course and all its associated lessons as removed (IsRemoved = true) in the database, rather than permanently destroying the records.
  6. Cache Invalidation: Deleting a course invalidates its cached course detail data.

Assumptions

Instructor has stable internet connection.
