UC ID and Name UC-62-Hide courses
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Instructor Secondary Actors: N/A
Trigger: Instructor clicks the "archive" icon button on a published course.
Description: Temporarily hides a course from public display on the marketplace.
Preconditions:

PRE-1. Instructor must be authenticated.

PRE-2. Instructor must be the owner of the course.

PRE-3. Course status must be "Published".
Post-conditions:

POST-1. The course status is updated to "Archived" in the database.

POST-2. The course is no longer visible to guests/learners to buy on the public marketplace.

POST-3. Active student enrollments are preserved, allowing enrolled students to continue accessing the course.
Normal Flow:

A. Hide Course Successfully

  1. Instructor accesses the Instructor Studio dashboard page.
  2. The system checks the instructor's session authentication.
  3. The system checks the instructor's approval status.
  4. The system retrieves the instructor's courses list.
  5. The system displays the courses list on the dashboard.
  6. Instructor locates the published course they wish to hide.
  7. Instructor clicks the "archive" icon button on the corresponding course card.
  8. The system displays a confirmation popup dialog prompting the instructor to confirm hiding the course.
  9. Instructor clicks the "Yes" button on the popup dialog.
  10. The system checks the instructor's session authentication.
  11. The system checks if the instructor's rights are locked.
  12. The system checks if the course is permanently discontinued.
  13. The system checks if the instructor is the owner of the course.
  14. The system updates the course status to "archived" in the database.
  15. The system clears the cache for the course details.
  16. The system displays a success notification indicating the course has been hidden.
  17. The system reloads the page dynamically, showing the updated course status badge as "Archived".
Alternative Flows:

A.9.1 Cancel Hiding

  1. Instructor clicks the "Cancel" button on the confirmation popup dialog.
  2. The system closes the confirmation popup dialog.
  3. Back to step 6.

A.7.1 Unhide Course

  1. Instructor locates the archived course they wish to unhide.
  2. Instructor clicks the "unarchive" icon button on the corresponding course card.
  3. The system displays a confirmation popup dialog asking if they want to make the course visible again.
  4. Instructor clicks the "Yes" button on the popup dialog.
  5. The system checks the instructor's session authentication.
  6. The system checks if the instructor's rights are locked.
  7. The system checks if the course is permanently discontinued.
  8. The system checks if the instructor is the owner of the course.
  9. The system checks if the course was in "Archived" status before unarchiving.
  10. The system updates the course status to "published" in the database.
  11. The system clears the cache for the course details.
  12. The system displays a success notification indicating the course has been made visible.
  13. The system reloads the page dynamically, showing the updated course status badge as "Published".
  14. The flow terminates.
Exceptions:

EX-2 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-3 Account Not Approved

  1. The system detects the actor's instructor approval status is not "Approved".
  2. The system redirects the actor to the application status or apply page.
  3. The flow terminates.

EX-11 Instructor Rights Locked

  1. The system detects that the instructor's account rights are locked due to policy violations.
  2. The system displays an error message indicating their rights are locked and status cannot be changed.
  3. The flow terminates.

EX-12 Course Permanently Discontinued

  1. The system detects the course status is "archived" and it has 3 or more flags.
  2. The system displays an error message indicating the course has been permanently discontinued due to policy violations and status cannot be changed.
  3. The flow terminates.

EX-13 Unauthorized Access

  1. The system detects that the instructor is not the owner of the course.
  2. The system displays an error message indicating they do not have permission to modify this course.
  3. The flow terminates.

EX-14 Database Update Failure

  1. The database fails to save the status change (0 rows affected).
  2. The system displays a system error message detailing the status update failure.
  3. The flow terminates.
Priority: High
Frequency of Use: Medium
Business Rules:
  1. Status Transition Restrictions: An instructor can only set a course to status 'pending' (moderation), 'archived' (hidden), or 'published' (only when unarchiving from status 'archived').
  2. Permanently Discontinued Block: A course that is 'archived' and has 3 or more flags is considered permanently discontinued due to policy violations and cannot have its status changed or be edited.
  3. Active Enrollment Preservation: Archiving a course does not delete it or impact active student enrollments; active students enrolled in the course continue to have full access to its contents on their dashboards.
  4. Public Visibility: A course with 'archived' status is immediately hidden from the public marketplace search, grid views, and category lists.
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
