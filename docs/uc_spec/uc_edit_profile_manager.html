UC-76 - Edit Profile (For Manager)

Primary Actor:
Manager

Secondary Actors:
None

Trigger:
Manager clicks the "Edit Profile" link on the profile page.

Description:
Allows managers (Super Admin or Staff Support) to edit their profile information, upload a profile picture, and save the updated details to the system.

Preconditions:

PRE-1. Manager must be successfully authenticated as an approved administrator or staff support account.

PRE-2. Manager is currently viewing their profile page.

Post-conditions:

POST-1. The manager's profile information is successfully updated in the database.

POST-2. The manager's display cookies are updated to synchronize with the new data.

POST-3. The manager is shown a success confirmation.

Normal Flow

A. Edit Profile Successfully

  1. Manager clicks the "Edit Profile" link.
  2. The system retrieves and displays the manager's current profile details on the edit profile page.
  3. Manager updates the profile details in the form (optionally selecting a profile picture, and entering display name, full name, phone number, and biography).
  4. Manager clicks the "Save Profile" button.
  5. The system validates the input fields and displays a confirmation prompt.
  6. Manager clicks the confirmation button on the prompt.
  7. The system uploads the profile picture (if a new file was selected) and updates the profile details in the database.
  8. The system updates the cookies and layout dynamically, and displays a success confirmation message.

Alternative Flows

A.1 Cancel editing

  1. Manager clicks the "Cancel" link or the "Back to Profile" link.
  2. The system redirects the manager back to the profile page.
  3. Use case ends.

A.5 Invalid input data

  1. The system detects one or more invalid input fields (e.g., missing display name, field length exceeding limits, or incorrect contact number format).
  2. The system displays a validation error message.
  3. Back to step 3.

A.6 Cancel saving changes

  1. Manager clicks the cancellation button on the prompt.
  2. The system closes the confirmation prompt.
  3. Back to step 3.

Exceptions

EX-2 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-3 Manager Profile Not Found

  1. The system fails to retrieve the manager profile.
  2. The system displays an error message regarding profile retrieval failure.
  3. The system redirects the manager to the login page.

EX-7 File Upload Service offline

  1. The system fails to upload the profile picture.
  2. The system displays an error message regarding file upload failure.
  3. Back to step 3.

EX-7.1 Database update failed

  1. The system fails to save the updated profile changes.
  2. The system displays a database transaction failure message.
  3. Back to step 3.

Business Rules

  1. Required Display Name: Display name is mandatory and cannot be left blank. It has a maximum limit of 255 characters.
  2. Full Name Limit: Full name is optional and has a maximum limit of 255 characters.
  3. Contact Number constraints: Contact number must match the format ^\+?[0-9\s-]{9,15}$ and cannot exceed 50 characters.
  4. Biography constraints: Biography is optional and cannot exceed 1000 characters.
  5. Cookie Synchronization: Upon a successful profile update, the system automatically synchronizes user display cookies (UserName and AvatarUrl) to maintain visual consistency across frontend layout headers without requiring a page refresh.

Assumptions

Manager has stable internet connection.
