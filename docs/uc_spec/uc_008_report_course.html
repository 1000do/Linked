UC ID and Name UC-ReportCourse - Report a course
Created By: Huynh Kien An Date Created: 09/06/2026
Primary Actor: User Secondary Actors: N/A
Trigger: User clicks the "Report this course" button on the course details page.
Description: Allows a user to report a course for violations (e.g., misleading content, copyright violation, spam, adult content). The system allows adding multiple reasons, validates the input, and submits the report to the server for moderation.
Preconditions:

PRE-1. User must be successfully authenticated.

PRE-2. User must be enrolled in the course.

PRE-3. User must not be the course owner or a manager.

PRE-4. The course must be in a reportable state (not pending, draft, or permanently locked).
Post-conditions:

POST-1. The report is recorded in the database for moderation.
Normal Flow:

A. Report Course Successfully

  1. User accesses the course details page.
  2. User clicks the "Report this course" button.
  3. The system checks authentication.
  4. The system displays the report course form.
  5. User inputs the report details (reason and description).
  6. User clicks the Submit Report button.
  7. The system validates the report details.
  8. The system submits the report data to the server.
  9. The system saves the report in the database.
  10. The system displays a success message regarding report submission.
Alternative Flows:

A.5.1 Add another reason

  1. User clicks the Add another reason button.
  2. The system adds another reason and description field to the form.
  3. Back to step 5.

A.5.2 Remove a reason

  1. User clicks the remove icon for a specific reason field.
  2. The system removes the reason and description field from the form.
  3. Back to step 5.

A.6 Cancel reporting

  1. User clicks the Cancel button.
  2. The system closes the report course form.
  3. Back to step 2.

A.7 Invalid form input

  1. The system detects invalid input in the report form.
  2. The system displays a validation error message.
  3. Back to step 5.
Exceptions:

EX-3 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-9 Database Save Failed

  1. The system fails to save the report to the database.
  2. The system displays an error message regarding report submission failure.
  3. Back to step 2.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-1, BR-2
Other Information: N/A
Assumptions:

1. User has stable internet connection.
