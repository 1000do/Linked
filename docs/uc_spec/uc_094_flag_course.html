UC ID and Name UC-94-Flag Course for Content Violation
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Actor clicks the "Flag" button on a course row in the moderation list.
Description: Flags a course for content or policy violations, hiding it from public display.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must have staff or admin role.

PRE-3. The course status must be pending, approved/published, rejected, or archived.

PRE-4. The course's current flag count must be less than 3.
Post-conditions:

POST-1. The course's flag count is incremented by 1 in the database.

POST-2. The instructor receives a warning or discontinuation notification.

POST-3. If flag count reaches 3, the course status is updated to "archived" (Permanently Locked).
Normal Flow:

A. Flag Course for Content Violation Successfully

  1. Actor accesses the Course Moderation Page.
  2. Actor locates the course in the moderation list.
  3. Actor clicks the "Flag" button on the corresponding course row.
  4. The system displays a flag violation warning dialog containing a text field for the violation description.
  5. Actor enters the violation description.
  6. Actor clicks the "Confirm Flag" button.
  7. The system checks the course's current flag count.
  8. The system increments the flag count by 1 in the database.
  9. The system sets the course's "ModerationFeedback" containing the flag count and the violation description.
  10. The system updates the course's modified date.
  11. The system checks if the new flag count has reached 3.
  12. The system sends a policy warning notification (Strike 1 or 2) to the instructor.
  13. The system saves the changes in the database.
  14. The system clears the course details cache.
  15. The system updates the moderation page stats and dims the flagged course row.
Alternative Flows:

A.4.1 Cancel Flagging

  1. Actor clicks the "Cancel" button on the dialog.
  2. The system closes the dialog.
  3. Back to step 2.

A.11.1 Flag Count Reaches Three (Strike 3)

  1. The system detects that the new flag count is 3.
  2. The system updates the course status to "archived" (mapping to "permanently_locked").
  3. The system sends a permanent discontinuation notification (Strike 3) to the instructor.
  4. Back to step 13.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-2 Unauthorized Access

  1. The system detects that the actor does not have staff or admin roles.
  2. The system displays an access denied page or error message.
  3. The flow terminates.

EX-6.1 Empty Input Validation

  1. The system detects that the violation description text field is empty.
  2. The system displays an error message indicating a violation reason is required.
  3. The system halts submission.
  4. Back to step 5.

EX-7 Flag Lockout

  1. The system detects that the course's flag count is already 3 or more.
  2. The system displays an error message indicating the course is already permanently locked.
  3. The flow terminates.

EX-13 Database Save Failure

  1. The database fails to commit changes.
  2. The system displays a system error message.
  3. The flow terminates.
Priority: High
Frequency of Use: Low
