UC ID and Name UC-14-Add Personal Reviews
Created By: Truong Phuoc Tai Date Created: 12/06/2026
Primary Actor: User, Instructor Secondary Actors: N/A
Trigger: Actor submits a rating and comment for a course (or lesson) they are enrolled in or own.
Description: Allows learners to submit reviews and star ratings for courses (or lessons) they are enrolled in, enforcing learning progress checks.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must be enrolled in the course, or be the owner (Instructor) of the course.
Post-conditions:

POST-1. The review record (either Course Review or Lesson Review) is successfully created and saved in the database.

POST-2. If the reviewer is not the course owner, a notification is sent to the course instructor.
Normal Flow:

A. Submit Course Review from Course Details Page Successfully

  1. Actor accesses the Course Details page of a course.
  2. The system checks the actor's session authentication.
  3. The system checks if the actor is the owner of the course.
  4. The system retrieves the course structure statistics.
  5. The system retrieves the actor's completed materials count.
  6. The system validates that the actor's learning progress is 100%.
  7. The system displays the write review form.
  8. Actor clicks the desired "star" icon to select a rating and enters comments in the review comment input.
  9. Actor clicks the "Submit Review" button.
  10. The system validates that a star rating is selected.
  11. The system validates that the comment content is not empty.
  12. The system checks if the actor's account has an active review posting restriction.
  13. The system saves the course review in the database.
  14. The system displays a success message.
  15. The system dynamically updates and reloads the reviews list and rating statistics.
  16. The system sends a new review notification to the course instructor.

B. Submit Lesson Review from Course Player Page Successfully

  1. Actor accesses the Course Player page (Go to learning).
  2. The system checks the actor's session authentication.
  3. The system checks if the actor is the owner of the course.
  4. The system retrieves the course structure statistics.
  5. The system retrieves the actor's completed materials count.
  6. The system validates that the actor has completed at least 1 lesson.
  7. The system displays the write review form in the Reviews tab.
  8. Actor clicks the desired "star" icon to select a rating and enters comments in the review comment input.
  9. Actor clicks the "Submit Review" button.
  10. The system validates that a star rating is selected.
  11. The system validates that the comment content is not empty.
  12. The system checks if the actor's account has an active review posting restriction.
  13. The system saves the lesson review in the database.
  14. The system displays a success message.
  15. The system dynamically updates and reloads the reviews list, overall rating statistics, and lesson rating statistics.
  16. The system sends a new review notification to the course instructor.
Alternative Flows:

A.6 Alt Flow Course Not Completed

  1. The system hides the review form.
  2. The system displays a warning message stating the actor's current progress percentage and indicating that 100% completion is required to write a review.
  3. The flow terminates.

A.10 Alt Flow Star Rating Missing

  1. The system displays a validation warning message prompting the actor to select a rating.
  2. Back to step 8.

A.11 Alt Flow Comment Missing

  1. The system displays a validation warning message prompting the actor to write a comment.
  2. Back to step 8.

B.6 Alt Flow No Lesson Completed

  1. The system hides the review form.
  2. The system displays a warning message stating that at least 1 lesson must be completed to write a review.
  3. The flow terminates.

B.10 Alt Flow Star Rating Missing

  1. The system displays a validation warning message prompting the actor to select a rating.
  2. Back to step 8.

B.11 Alt Flow Comment Missing

  1. The system displays a validation warning message prompting the actor to write a comment.
  2. Back to step 8.
Exceptions:

EX-2 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-12 Restricted Account

  1. The system detects that the actor's account has an active review posting restriction.
  2. The system displays an error message indicating that their account is restricted from posting comments or reviews and states the restriction end time.
  3. The flow terminates.
Priority: High
Frequency of Use: High
Business Rules:
  1. Completion Check for Course Review: Non-owner students must have 100% course progress (completed all materials) to write a review on the Course Details page.
  2. Minimal Progress for Lesson Review: Non-owner students must have completed at least 1 lesson/material to write a review on the Course Player page.
  3. Instructor Bypass: Instructors are automatically considered to have completed the course (progress at 100%) and can review their own courses without progress restrictions.
  4. Rating Restrictions: Star ratings must be between 1 and 5 stars (inclusive).
  5. Account Restrictions (Lockout): Users with active comment/review restrictions are blocked from submitting new reviews. The system checks this validation at submission time.
  6. Instructor Notification: Upon successful submission, a notification is sent to the course instructor with a link pointing directly to the review card.
Other Information: N/A
Assumptions:

User, Instructor has stable internet connection.
