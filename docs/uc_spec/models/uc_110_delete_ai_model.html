UC ID and Name UC-110 - Delete AI Model
Created By: An Date Created: 11/06/2026
Primary Actor: Admin Secondary Actors: N/A
Trigger: Admin clicks the status toggle button on a model entry in the AI model list.
Description: Allows an Admin to soft-delete an AI model by toggling its status between active and inactive. No data is permanently removed; an active model is set to inactive and an inactive model is restored to active.
Preconditions:

PRE-1. Admin must be successfully authenticated.
Post-conditions:

POST-1. The selected AI model's status is updated in the database.

POST-2. The model list on the UI reflects the updated status after page reload.
Normal Flow:

A. Toggle AI Model Status Successfully

  1. Admin accesses the AI Service management page.
  2. The system validates the Admin's access authorization.
  3. Admin clicks the status toggle button on a model entry.
  4. The system displays a confirmation dialog.
  5. Admin confirms the action.
  6. The system updates the model status in the database.
  7. The system displays a success notification.
  8. The system reloads the page.
Alternative Flows:

A.5 Admin Cancels Confirmation

  1. Admin clicks the cancel button on the confirmation dialog.
  2. Back to step 3.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Unauthorized Access

  1. The system detects the current user does not have Admin authorization.
  2. The system displays an unauthorized access error message.
  3. The system redirects the user to the login page.

EX-6.1 Model Not Found

  1. The system cannot locate the target model record during the status update operation.
  2. The system displays a model not found error message.
  3. Back to step 3.

EX-6.2 Database Save Failed

  1. The system fails to persist the status change to the database.
  2. The system displays an error message regarding the database save failure.
  3. Back to step 3.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-25, BR-26, BR-31
Other Information: N/A
Assumptions:

Admin has stable internet connection.
