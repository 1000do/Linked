UC ID and Name UC-113 - Set Confidence Threshold
Created By: An Date Created: 11/06/2026
Primary Actor: Admin Secondary Actors: N/A
Trigger: Admin modifies one or more threshold values and clicks the "Save Changes" button in the threshold configuration form.
Description: Allows an Admin to configure the spam confidence, toxicity confidence, and similarity score thresholds used by the automated AI moderation system. The current threshold values are pre-filled from the database on page load. If no configuration exists, system defaults are applied.
Preconditions:

PRE-1. Admin must be successfully authenticated.
Post-conditions:

POST-1. The confidence threshold configuration is updated in the database.

POST-2. The updated threshold values are displayed in the configuration form in place of the old values.
Normal Flow:

A. Set Confidence Threshold Successfully

  1. Admin accesses the AI Service management page.
  2. The system validates the Admin's access authorization.
  3. The system retrieves and displays the current threshold configuration values in the configuration form.
  4. Admin modifies one or more threshold values in the configuration form.
  5. Admin clicks the "Save Changes" button.
  6. The system displays a confirmation dialog.
  7. Admin confirms the action.
  8. The system validates the form inputs.
  9. The system updates the threshold configuration in the database.
  10. The system displays a success notification.
Alternative Flows:

A.3 No Threshold Configuration Found

  1. The system detects no threshold configuration exists in the database.
  2. The system populates the form with system default threshold values.
  3. Back to step 4.

A.6 Admin Cancels Confirmation

  1. Admin clicks the cancel button on the confirmation dialog.
  2. Back to step 4.

A.8 Form Validation Failed

  1. The system detects one or more submitted values fail validation constraints.
  2. The system displays a validation error message.
  3. Back to step 4.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Unauthorized Access

  1. The system detects the current user does not have Admin authorization.
  2. The system displays an unauthorized access error message.
  3. The system redirects the user to the login page.

EX-9 Database Save Failed

  1. The system fails to persist the threshold configuration to the database.
  2. The system displays an error message regarding the database save failure.
  3. Back to step 4.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-1, BR-25
Other Information: N/A
Assumptions:

Admin has stable internet connection.
