UC ID and Name UC-12-View purchased course list
Created By: Truong Phuoc Tai Date Created: 12/06/2026
Primary Actor: User, Instructor Secondary Actors: N/A
Trigger: Actor accesses the enrolled courses page.
Description: Displays all courses successfully purchased or enrolled in by the user.
Preconditions:

PRE-1. Actor must be authenticated.
Post-conditions:

POST-1. Detailed list of enrolled courses is rendered successfully showing the thumbnail, title, instructor name, progress bar, completion status, and learning actions.
Normal Flow:

A. View Purchased Course List Successfully

  1. Actor accesses the enrolled courses page.
  2. The system checks the actor's session authentication.
  3. The system retrieves the list of courses the actor is enrolled in.
  4. For each course, the system retrieves the total material count and the completed material count.
  5. For each course, the system calculates the progress percentage.
  6. The system displays the enrolled courses page showing the course title, instructor, thumbnail, progress percentage, completion status, and action buttons.
Alternative Flows:

A.3 Empty Course List

  1. The system detects the enrolled course list is empty.
  2. The system displays the empty state message.
  3. Use case ends.

A.6.1 Navigate to Learning Player (Incomplete Course)

  1. Actor clicks the "Continue learning" button on an incomplete course card.
  2. The system redirects the actor to the course player page.
  3. Use case ends.

A.6.2 Review Course (Completed Course)

  1. Actor clicks the "Review" button on a completed course card.
  2. The system redirects the actor to the course player page.
  3. Use case ends.
Exceptions:

EX-2 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.
Priority: High
Frequency of Use: High
Business Rules: BR-87, BR-88, BR-89
Other Information: N/A
Assumptions:

User, Instructor has stable internet connection.
