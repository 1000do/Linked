UC ID and Name UC-92-Approve Course Submission
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Actor clicks the "Approve" button on a course row in the moderation list.
Description: Approves a submitted course, marking it as published and visible on the marketplace.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must have staff or admin role.

PRE-3. The course status must be pending review (or rejected, archived, permanently locked if re-approving).
Post-conditions:

POST-1. The course status is set to "published" (Approved) in the database.

POST-2. The instructor receives a notification of the approval.

POST-3. Existing enrolled students (if any) receive content update notifications.
Normal Flow:

A. Approve Course Submission Successfully

  1. Actor accesses the Course Moderation Page.
  2. Actor locates the pending course in the moderation list.
  3. Actor clicks the "Approve" button on the corresponding course row.
  4. The system displays an approval confirmation dialog containing a text field for optional feedback.
  5. Actor enters optional feedback.
  6. Actor clicks the "Approve & Publish" button.
  7. The system updates the course status to "published" and clears existing moderation feedback in the database.
  8. The system updates the status of all lessons and materials under the course to "active" in the database.
  9. The system sends an approval notification message to the instructor.
  10. The system sends content update notifications to all enrolled students.
  11. The system clears the course details cache.
  12. The system updates the moderation page stats and dims the approved course row.
Alternative Flows:

A.6.1 Cancel Approval

  1. Actor clicks the "Cancel" button on the confirmation dialog.
  2. The system closes the confirmation dialog.
  3. Back to step 2.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-2 Unauthorized Access

  1. The system detects that the actor does not have staff or admin roles.
  2. The system displays an access denied page or error message.
  3. The flow terminates.

EX-7 Database Save Failure

  1. The database fails to commit changes.
  2. The system displays a system error message.
  3. The flow terminates.
Priority: High
Frequency of Use: Medium
Business Rules:
  1. Re-approval Rule: If the course status is rejected, archived, or permanently locked, the action button is labeled "Re-approve" but follows the same approval logic.
  2. Notification Content: The approval notification sent to the instructor includes the optional feedback entered by the actor.
  3. Cache Eviction: The Redis cache for the course details must be evicted immediately upon approval to ensure users see the updated content.
Other Information: N/A
Assumptions:

Staff has stable internet connection.

Admin has stable internet connection.
