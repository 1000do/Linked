UC-AddMaterial - Add Learning Materials

Primary Actor:
Instructor

Secondary Actors:
Cloud Storage Service

Trigger:
Instructor clicks the upload area (for Video) or "Choose File" button (for Resource) in the active lesson editor.

Description:
Allows an instructor to add learning materials (videos or documents) to a specific lesson. The system handles direct-to-cloud file uploads, extracts file metadata, and enforces limits for instructors without an active Stripe account.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The lesson must exist and belong to a course owned by the active instructor.

Post-conditions:

POST-1. The new learning material record is saved in the database and linked to the lesson.

POST-2. The physical file is stored in the Cloud Storage Service.

POST-3. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

Normal Flow

A. Add Material Successfully

  1. Instructor accesses the course edit page.
  2. Instructor selects a specific lesson from the curriculum list.
  3. Instructor selects a file (video or document) to upload.
  4. The system validates the file format.
  5. The system uploads the file to the Cloud Storage Service.
  6. The system sends the uploaded file URL and metadata to the backend.
  7. The system validates the instructor's permission.
  8. The system validates the instructor's lockout status.
  9. The system validates the course status.
  10. The system validates the maximum material limits for unlinked Stripe accounts.
  11. The system creates or updates the learning material record in the database.
  12. The system updates the course status to draft and clears moderation feedback in the database.
  13. The system invalidates cached course data.
  14. The system updates the UI to display the new material.

Alternative Flows

A.4.1 Invalid File Format

  1. The system detects an invalid file format (e.g., video in the document section).
  2. The system halts the upload process and displays an invalid format error message.
  3. Back to step 3.

A.4.2 File is too large (Video) - Continue

  1. The system detects a video larger than 50MB and displays a warning prompt recommending compression.
  2. Instructor clicks "Yes, upload anyway".
  3. Back to step 5.

A.4.3 File is too large (Video) - Cancelled

  1. The system detects a video larger than 50MB and displays a warning prompt recommending compression.
  2. Instructor clicks "I will compress it first".
  3. The system halts the upload process and clears the file selection.
  4. Back to step 3.

Exceptions

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-5 Cloud Storage Service Upload Failure

  1. The system fails to upload the file to the Cloud Storage Service.
  2. The system displays an upload failure error message.
  3. Back to step 2.

EX-7 Permission Denied

  1. The system halts execution and displays a permission denied error message.
  2. Back to step 2.

EX-8 Instructor Account Locked

  1. The system halts execution and displays an account locked error message.
  2. Back to step 2.

EX-9 Course Pending Review

  1. The system halts execution and displays a course pending review error message.
  2. Back to step 2.

EX-10 Stripe Limit Exceeded

  1. The system detects the instructor has no active Stripe account and has reached the document limit.
  2. The system halts execution and displays a limit exceeded error message.
  3. Back to step 2.

EX-11 Database update failed

  1. The system fails to save the material changes.
  2. The system displays a database transaction failure message.
  3. Back to step 2.

Business Rules

  1. Material Limits for Unlinked Stripe Accounts: Instructors who have not linked an active Stripe account are only allowed to attach up to 1 document (resource) per lesson.
  2. Pending Restriction: Materials cannot be added to a course while it is currently in 'Pending' review status.
  3. Status Reversion: Adding material to a 'Published' course automatically changes its status back to 'Draft' and clears any existing moderation feedback.
  4. File Format Constraints: The document/resource section strictly blocks video files from being uploaded.

Assumptions

Instructor has stable internet connection.
Cloud Storage Service is working properly.
