UC ID and Name UC-89-Search Courses
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Actor enters search keywords in the "search" input box on the Course Moderation Page.
Description: Searches course list by title or instructor name for moderation purposes.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must have staff or admin role.

PRE-3. Actor must be on the Course Moderation Page.
Post-conditions:

POST-1. The system displays the course moderation list matching the search keywords and any active filters.
Normal Flow:

A. Search Courses Successfully

  1. Actor enters search keywords in the "search" input box.
  2. The system retrieves the courses list filtered by the search keywords and other active filters.
  3. The system updates the course moderation table and statistics cards.
  4. The system displays the filtered course list.
Alternative Flows:

A.1.1 Clear Search Keyword

  1. Actor clears the search keywords in the "search" input box.
  2. The system retrieves the courses list using default parameters (and active filters).
  3. The system updates the course moderation table and statistics cards.
  4. The system displays the default list of courses.
  5. Back to step 1.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-2 Unauthorized Access

  1. The system detects that the actor does not have staff or admin roles.
  2. The system displays an access denied page or error message.
  3. The flow terminates.

EX-3 API Call Failed

  1. The system fails to retrieve the filtered course list or stats from the API.
  2. The system displays an error message regarding search failure.
  3. The system renders the Course Moderation page with an empty list.
  4. The flow terminates.
Priority: High
Frequency of Use: High
Business Rules:
  1. Exclusion of Drafts: Searches are only performed on non-draft courses (pending, approved, rejected, archived, permanently locked).
  2. Search Criteria: Search keywords are matched (case-insensitively, substring match) against:
    • Course Title
    • Instructor Name (Full Name)
  3. Preservation of Filters: Searching preserves all other active filters (Category, Status, Sort Option).
  4. Debounce Time: To prevent excessive server requests, the search execution is debounced by 300 milliseconds from the last keystroke.
Other Information: N/A
Assumptions:

Staff has stable internet connection.

Admin has stable internet connection.
