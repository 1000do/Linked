UC ID and Name UC-83-View Chat List
Created By: Antigravity Date Created: 20/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Actor accesses the Chat page under the Admin Panel dashboard.
Description: Allows the Actor to view their list of active support or moderation conversations, view pending support request tickets (Staff only), search for specific conversations, filter by unread status, and sort the conversation list.
Preconditions:

PRE-1. Actor must be successfully authenticated.

PRE-2. Actor's role must be Staff or Admin.
Post-conditions:

POST-1. The system renders the list of active support/moderation conversations in the chat sidebar.

POST-2. The system displays the list of pending support requests in the sidebar (Staff only).
Normal Flow:

A. View Chat List Successfully

  1. Actor accesses the Chat page under the Admin Panel dashboard.
  2. The system checks the Actor's authentication status.
  3. The system validates that the Actor's role is Staff or Admin.
  4. The system requests the list of active support/moderation conversations.
  5. The system requests the list of pending support requests (if the Actor is Staff).
  6. The system retrieves the list of active support/moderation conversations from the database.
  7. The system retrieves the list of pending support requests from the database (if the Actor is Staff).
  8. The system displays the Chat page with the chat sidebar containing the active conversation list, pending requests list (Staff only), search input, filter dropdown, and sort dropdown.
Alternative Flows:

A.8.1 Filter by Unread Messages

  1. Actor selects "Unread" from the `"chat-filter-status"` dropdown.
  2. The system filters the active support/moderation chat list to display only conversations with an unread message count greater than zero.
  3. Back to step 8.

A.8.2 Sort by Name (A-Z)

  1. Actor selects "Name (A-Z)" from the `"chat-sort"` dropdown.
  2. The system sorts the list of active conversations alphabetically by the partner's name.
  3. Back to step 8.

A.8.3 Sort by Unread First

  1. Actor selects "Unread First" from the `"chat-sort"` dropdown.
  2. The system sorts the list of active conversations to display those with unread messages at the top.
  3. Back to step 8.

A.8.4 Search Users/Conversations

  1. Actor enters a search term in the `"search-chats"` input box.
  2. The system triggers a query after a brief debounce delay (300ms).
  3. The system retrieves the filtered conversation list matching the search term.
  4. Back to step 8.

A.8.5 Clear Search Input

  1. Actor clears the text in the `"search-chats"` input box.
  2. The system retrieves the default active conversation list.
  3. Back to step 8.

A.8.6 Contact Admin (Only for Staff)

  1. Actor (Staff) clicks the `"Contact Admin"` button.
  2. The system requests the Admin account details.
  3. The system creates a direct chat channel with the Admin.
  4. The system updates the conversation list and opens the chat room.
  5. Back to step 8.

A.8.7 Accept Support Request (Only for Staff)

  1. Actor (Staff) clicks the `"Accept Request"` button next to a pending support ticket.
  2. The system updates the support ticket status to accepted.
  3. The system creates a support chat room between the Actor (Staff) and the requester.
  4. The system removes the support ticket from the pending requests list.
  5. The system updates the conversation list and opens the support chat room.
  6. Back to step 8.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-3 Unauthorized Access

  1. The system detects that the Actor is neither Staff nor Admin.
  2. The system displays an access denied error message.
  3. The system redirects the Actor to the dashboard/home page.

EX-6 Retrieval Failed

  1. The system fails to retrieve the conversation list or pending requests.
  2. The system displays a bad request error notification.
  3. Back to step 8.
Priority: High
Frequency of Use: High
Business Rules:

1. Support Request Assignment: Pending support tickets are visible only to Staff members. When a Staff member clicks `"Accept Request"`, the ticket is assigned to them and removed from the public pending list. Admin members do not receive or view pending support requests from users.

2. Staff-Admin Escalation: Staff members can use the `"Contact Admin"` button to establish a direct communication channel with an Admin for escalation or administrative assistance. Admin members can view and reply to active conversations initiated by Staff members, but cannot initiate support chats or receive user tickets.

3. Real-Time Synchronization: Both active conversations and pending requests list are synchronized in real-time using SignalR hub notifications.
Other Information: N/A
Assumptions:

Staff, Admin has stable internet connection.
