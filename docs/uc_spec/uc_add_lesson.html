UC-AddLesson - Add Lesson

Primary Actor:
Instructor

Secondary Actors:
None

Trigger:
Instructor clicks the "Create New Lesson" button or the "Add Lesson" icon button on the course syllabus sidebar.

Description:
Allows an instructor to add a new lesson to an existing course. The system enforces business rules such as lesson limits, uniqueness of lesson titles, and restricts modifications if the course is currently pending review or the instructor is locked out.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The course must exist and belong to the active instructor.

Post-conditions:

POST-1. A new lesson is added to the course curriculum.

POST-2. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

Normal Flow

A. Add New Lesson Successfully

  1. Instructor accesses the course edit page.
  2. Instructor clicks the "Create New Lesson" button or the "Add Lesson" icon button on the course syllabus sidebar.
  3. The system displays the Add New Lesson modal.
  4. Instructor enters the lesson title.
  5. Instructor clicks the "Create Lesson" button.
  6. The system validates the instructor's permission.
  7. The system validates the instructor's lockout status.
  8. The system validates the course status.
  9. The system validates the maximum lesson limits.
  10. The system validates the lesson title for duplicates.
  11. The system creates the new lesson in the database.
  12. The system updates the course status to draft and clears moderation feedback in the database.
  13. The system adds the newly created lesson to the curriculum list.
  14. The system closes the Add New Lesson modal.

Alternative Flows

A.12 Course is not published

  1. The system skips updating the course status and moderation feedback.
  2. Back to step 13.

Exceptions

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-5 Permission Denied

  1. The system halts execution and displays a permission denied error message.
  2. Back to step 3.

EX-6 Instructor Account Locked

  1. The system halts execution and displays an account locked error message.
  2. Back to step 3.

EX-7 Course Pending Review

  1. The system halts execution and displays a course pending review error message.
  2. Back to step 3.

EX-8 Max Lessons Reached

  1. The system halts execution and displays a maximum lessons reached error message.
  2. Back to step 3.

EX-9 Duplicate Lesson Title

  1. The system halts execution and displays a duplicate title error message.
  2. Back to step 3.

EX-10 Database update failed

  1. The system fails to save the lesson changes.
  2. The system displays a database transaction failure message.
  3. Back to step 3.

Business Rules

  1. Lesson Limit for Unlinked Stripe Accounts: Instructors who have not linked an active Stripe account are only allowed to create up to 5 lessons per course.
  2. Duplicate Title Check: Lesson titles must be unique within a course (case-insensitive).
  3. Pending Restriction: Lessons cannot be added to a course while it is currently in 'Pending' review status.
  4. Status Reversion: Adding a lesson to a 'Published' course automatically changes its status back to 'Draft' and clears any existing moderation feedback.

Assumptions

Instructor has stable internet connection.
