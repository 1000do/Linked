UC ID and Name UC-65-Permanently delete learning materials
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Instructor Secondary Actors: N/A
Trigger: Instructor clicks the "Delete Forever" button next to a removed material on the Trash Bin page.
Description: Permanently deletes files from the storage server.
Preconditions:

PRE-1. Instructor must be authenticated.

PRE-2. Instructor's status must be "Approved".

PRE-3. The material must be currently in the trash bin (soft-deleted status is "removed").
Post-conditions:

POST-1. The learning material record is permanently deleted from the database.

POST-2. The material file is permanently deleted from cloud storage (Cloudinary).

POST-3. The material is no longer retrievable by the instructor or any other user.
Normal Flow:

A. Permanently Delete Learning Material Successfully

  1. Instructor accesses the Trash Bin page (displays the list of removed learning materials).
  2. Instructor locates the removed material they wish to delete forever.
  3. Instructor clicks the "Delete Forever" button on the corresponding material row.
  4. The system displays a confirmation popup dialog warning of irreversible deletion.
  5. Instructor clicks the "Yes, delete forever" button.
  6. The system checks the instructor's session authentication.
  7. The system checks if the instructor's rights are locked.
  8. The system checks if the parent course containing the lesson is pending review.
  9. The system checks if the parent course is permanently discontinued.
  10. The system checks the instructor's ownership of the material.
  11. The system deletes the material file from cloud storage.
  12. The system deletes the material record from the database.
  13. The system clears the course details cache.
  14. The system displays a success confirmation message.
  15. The system dynamically removes the deleted material row from the list.
Alternative Flows:

A.5.1 Cancel Deletion

  1. Instructor clicks the "Cancel" button on the confirmation popup dialog.
  2. The system closes the confirmation popup dialog.
  3. Back to step 2.
Exceptions:

EX-6 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-7 Instructor Rights Locked

  1. The system detects that the instructor's account rights are locked due to policy violations.
  2. The system displays an error message indicating their rights are locked and changes cannot be saved.
  3. The flow terminates.

EX-8 Course Pending Review

  1. The system detects the course containing the lesson is currently pending review.
  2. The system displays an error message indicating curriculum changes are blocked during review.
  3. The flow terminates.

EX-9 Course Permanently Discontinued

  1. The system detects the course is permanently discontinued due to policy violations (3 or more flags).
  2. The system displays an error message indicating modifications are blocked.
  3. The flow terminates.

EX-10 Unauthorized Access

  1. The system detects that the instructor does not own the course/material.
  2. The system displays an error message indicating permission denied.
  3. The flow terminates.

EX-14 Database Save Failure

  1. The database fails to commit changes.
  2. The system displays a system error message.
  3. The flow terminates.
Priority: High
Frequency of Use: Medium
Business Rules: BR-118, BR-119, BR-116
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
