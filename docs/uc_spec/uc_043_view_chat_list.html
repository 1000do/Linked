UC ID and Name UC-43-View Chat List
Created By: Antigravity Date Created: 20/06/2026
Primary Actor: User, Instructor Secondary Actors: N/A
Trigger: Actor accesses the Chat page.
Description: Allows the Actor to view their list of active conversations, search for specific conversations, filter by unread status, sort the conversation list, and interact with the compact chat list through the floating mini-chat widget.
Preconditions:

PRE-1. Actor must be successfully authenticated.

PRE-2. If the Actor is an Instructor, their application status must be approved.
Post-conditions:

POST-1. The system renders the list of active conversations in the main chat sidebar.

POST-2. The system renders the compact list of active conversations in the floating mini-chat widget dropdown.
Normal Flow:

A. View Chat List Successfully

  1. Actor accesses the Chat page.
  2. The system checks the Actor's authentication status.
  3. The system checks the Actor's role.
  4. The system validates the Actor's access permission (if Instructor, checks that the approval status is approved).
  5. The system requests the list of active conversations.
  6. The system retrieves the list of active conversations from the database.
  7. The system displays the Chat page with the chat sidebar containing the list of active conversations, search input, filter dropdown, and sort dropdown.
Alternative Flows:

A.7.1 Filter by Unread Messages

  1. Actor selects "Unread" from the `"chat-filter-status"` dropdown.
  2. The system filters the chat list to display only conversations with an unread message count greater than zero.
  3. Back to step 7.

A.7.2 Sort by Name (A-Z)

  1. Actor selects "Name (A-Z)" from the `"chat-sort"` dropdown.
  2. The system sorts the list of conversations alphabetically by the partner's name.
  3. Back to step 7.

A.7.3 Sort by Unread First

  1. Actor selects "Unread First" from the `"chat-sort"` dropdown.
  2. The system sorts the list of conversations to display those with unread messages at the top.
  3. Back to step 7.

A.7.4 Search Users/Conversations

  1. Actor enters a search term in the `"search-chats"` input box.
  2. The system triggers a query after a brief debounce delay (300ms).
  3. The system retrieves the filtered conversation list matching the search term.
  4. Back to step 7.

A.7.5 Clear Search Input

  1. Actor clears the text in the `"search-chats"` input box.
  2. The system retrieves the default active conversation list.
  3. Back to step 7.

A.7.6 Access Chat via Floating Mini-Chat Widget

  1. Actor accesses the Home page.
  2. Actor clicks the Floating Chat Button ("support_agent" icon).
  3. The system checks the Actor's authentication status.
  4. The system requests the list of active conversations.
  5. The system retrieves the list of active conversations from the database.
  6. The system displays the support chat popup, showing the active conversation details or the support welcome screen.
  7. Actor clicks the `"mini-chat-selector-btn"` dropdown selector.
  8. The system displays the `"mini-chat-dropdown"` menu containing a compact list of active conversations.
  9. Actor clicks a conversation card from the compact list.
  10. The system updates the active chat context, loads the conversation history, and closes the dropdown menu.
  11. Back to step 7.

A.7.7 Redirect to Main Chat Page from Mini-Chat

  1. Actor clicks the `"View All"` link in the `"mini-chat-dropdown"` menu.
  2. The system redirects the Actor to the main Chat page (`/Chat`).
  3. Back to step 1.

A.7.8 Request Support via Mini-Chat

  1. Actor clicks the `"Contact Support"` button or selects a quick reply option (e.g., "Ask about course").
  2. The system checks if there is any pending support request.
  3. If there is no pending request, the system shows the welcome text and waits for message input.
  4. Actor enters the support request content in the `"mini-chat-input"` box and clicks the `"send"` button.
  5. The system saves the support request as a pending ticket in the database.
  6. The system displays a waiting state UI ("Your request is in queue. Waiting for support staff...") in the widget.
  7. Back to step 7.

A.7.9 View New Message Notification via Unread Preview Bubble

  1. The system receives a new chat message via SignalR.
  2. The system displays the `"unread-preview-bubble"` containing a preview of the message next to the floating chat button.
  3. Actor clicks the `"unread-preview-bubble"`.
  4. The system opens the support chat popup, displays the conversation details, and hides the preview bubble.
  5. Back to step 7.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-4 Instructor Status Unapproved

  1. The system detects the Instructor's approval status is not approved.
  2. The system redirects the Instructor to the application status page.

EX-6 Retrieval Failed

  1. The system fails to retrieve the conversation list.
  2. The system displays a bad request error notification.
  3. Back to step 7.
Priority: High
Frequency of Use: High
Business Rules:

1. Unread Count Display: The unread message count is updated dynamically and displayed as a badge on both the main conversation card and the floating widget icon.

2. Online Status Indicator: A green dot is displayed on the conversation card if the partner is currently online.

3. Debounced Search: The search query is debounced by 300ms before sending a request to the backend.

4. Widget Home-only Visibility: The floating chat widget is visible and accessible only on the Home page for logged-in Users.

5. Support Ticket Queueing: Support requests submitted via the mini-chat widget create a pending request queue entry and prevent further messages until a staff member accepts the request.
Other Information: N/A
Assumptions:

User, Instructor has stable internet connection.
