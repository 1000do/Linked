UC ID and Name UC-111 - View AI Model Details
Created By: An Date Created: 11/06/2026
Primary Actor: Admin Secondary Actors: N/A
Trigger: Admin clicks the "Details" icon button on a model entry in the AI model list.
Description: Displays the full configuration details of a selected AI model in a details dialog. The detail data is pre-loaded with the page and presented client-side without an additional server request.
Preconditions:

PRE-1. Admin must be successfully authenticated.
Post-conditions:

POST-1. The selected AI model's details are displayed to the Admin.
Normal Flow:

A. View AI Model Details Successfully

  1. Admin accesses the AI Service management page.
  2. The system validates the Admin's access authorization.
  3. The system retrieves and displays the list of AI models.
  4. Admin clicks the "Details" icon button on a model entry.
  5. The system displays the model details in a popup.
  6. Admin reviews the model details.
  7. Admin clicks the "Close" button.
  8. The system closes the popup.
Alternative Flows: N/A
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Unauthorized Access

  1. The system detects the current user does not have Admin authorization.
  2. The system displays an unauthorized access error message.
  3. The system redirects the user to the login page.

EX-3 Database Query Failed

  1. The system fails to retrieve AI model data from the database.
  2. The system displays an error message regarding the retrieval failure.
  3. Back to step 1.
Priority: Low
Frequency of Use: Low
Business Rules: BR-25, BR-27
Other Information: N/A
Assumptions:

Admin has stable internet connection.
