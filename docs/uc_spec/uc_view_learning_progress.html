UC ID and Name UC-10-View Learning Progress
Created By: Truong Phuoc Tai Date Created: 13/05/2026
Primary Actor: User, Instructor Secondary Actors: N/A
Trigger: Actor accesses the enrolled courses page or accesses the course player/learning page.
Description: Allows Users and Instructors to track learning progress for enrolled or owned courses. Actors can view progress percentages on their enrolled courses dashboard, continue learning in the course player, view videos, read text documents, download learning materials, and automatically update their progress checks. When all materials are completed, the course is marked as completed.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must be enrolled in the course, or be the owner (Instructor) of the course.
Post-conditions:

POST-1. Detailed course list with progress percentage, completion status, and navigation controls is displayed.

POST-2. Overall progress percentage and completion bar are rendered in the course player header.

POST-3. Checkmark icons are displayed next to completed learning materials in the player's curriculum sidebar.
Normal Flow:

A. View Course Progress Successfully

  1. Actor accesses the "My Enrolled Courses" page.
  2. The system checks the actor's session authentication.
  3. The system retrieves the list of courses the actor is enrolled in.
  4. For each course, the system retrieves the total material count and the completed material count.
  5. For each course, the system calculates the progress percentage.
  6. The system displays the enrolled courses page showing the course title, instructor, thumbnail, progress percentage, completion status, and action buttons.
  7. Actor clicks the "Continue learning" button (or "Review" button if the course is completed) on a course card.
  8. The system retrieves the course details (lessons and learning materials).
  9. The system validates the actor's enrollment status or course ownership.
  10. The system retrieves the actor's learning progress for this course, including completed material IDs.
  11. The system calculates the overall completion percentage.
  12. The system displays the course player showing the media viewer, the overall progress percentage and progress bar in the header, and checkmarks next to completed items in the syllabus sidebar.
Alternative Flows:

A.3 Empty Enrollment List

  1. The system detects the actor has no enrolled courses.
  2. The system displays the empty state message.
  3. The system renders the "Browse Courses" button.
  4. Actor clicks the "Browse Courses" button.
  5. The system redirects the actor to the course exploration page.
  6. Use case ends.

A.12.1 Complete Video Material

  1. Actor plays a video material in the media player.
  2. The video playback reaches the end.
  3. The system saves the completed material ID to the database.
  4. The system recalculates the progress percentage.
  5. The system updates the progress bar and syllabus checkmarks.
  6. Back to step 12.

A.12.2 Complete Text Material

  1. Actor selects a text-based material.
  2. The system displays the text content.
  3. The system saves the completed material ID to the database.
  4. The system recalculates the progress percentage.
  5. The system updates the progress bar and syllabus checkmarks.
  6. Back to step 12.

A.12.3 Complete Downloadable Material

  1. Actor clicks the "Download Resource" button for a non-viewable material.
  2. The system saves the completed material ID to the database.
  3. The system recalculates the progress percentage.
  4. The system updates the progress bar and syllabus checkmarks.
  5. The browser downloads the file.
  6. Back to step 12.

A.12.4 Course Completion

  1. The system detects the completed material count equals the total material count.
  2. The system updates the enrollment status to completed in the database.
  3. The system records the completion date in the database.
  4. The system updates the course player UI to display a completion state.
  5. Back to step 12.
Exceptions:

EX-2 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-9 Access Denied (Not Enrolled)

  1. The system detects the actor is not enrolled in the course and is not the course owner.
  2. The system redirects the actor to the error page.
Priority: High
Frequency of Use: High
Business Rules:
  1. Progress Calculation: Course progress percentage is calculated as (Completed Materials / Total Materials) * 100, rounded to 1 decimal place.
  2. Video Completion Trigger: A video material is marked as completed only when the video playback reaches the end ("onended" event).
  3. Document/Text Completion Trigger: A text document (".txt") is marked as completed immediately upon viewing. Other downloadable resources are marked as completed when the User clicks the download button.
  4. Course Completion State: When all learning materials in a course are marked as completed, the system automatically sets the enrollment status IsCompleted = true and updates the completion date in the database.
  5. Instructor Bypass: Course owners/instructors can access the Course Player ("Learn" page) for preview, and their progress is treated as 100% completed virtual progress without writing completions to the database.
Other Information: N/A
Assumptions:

User, Instructor has stable internet connection.
