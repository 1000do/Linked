UC ID and Name UC-107 - Resolve Report
Created By: System Date Created: 09/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Staff or Admin clicks the "Moderate" button on a pending, under review, or escalated report row.
Description: Resolves a report by taking moderation action (e.g., removing content) and updating status.
Preconditions:

PRE-1. The actor must be authenticated.

PRE-2. The actor must have Staff or Admin role.

PRE-3. The selected report must be in a resolvable state ("Pending", "Under Review", or "Escalated").
Post-conditions:

POST-1. The system updates the report with the selected status and resolution note.

POST-2. If content removal was checked, the system increments the content's violation flag count and applies penalties (e.g., warning notifications, hiding content).

POST-3. If a course accumulates 3 strikes (flags), the system applies permanent course discontinuation and instructor lockouts as defined in the business rules.

POST-4. The system sends relevant notifications to the reporter regarding the report outcome, and to the content owner regarding policy violations or lockouts.

POST-5. If a review is removed, the system soft-deletes the review, marks it as violating, increments the author's account flag count, and applies progressing account lockouts as defined in the business rules.
Normal Flow:

A. Resolve Report Successfully

  1. Staff or Admin accesses the Report Center page.
  2. The system validates the actor's authentication and authorization.
  3. Staff or Admin clicks the "Moderate" button on a report row (course report, course review report, or lesson review report).
  4. The system displays the Moderate Report popup containing report details, resolution status options, content removal checkbox, and note textarea.
  5. The actor selects the "Accept Report (Valid / Policy Violation)" status.
  6. The actor enters moderation details.
  7. The actor clicks "Submit Decision".
  8. The system validates the actor's authentication and authorization.
  9. The system processes the moderation decision, updates the report, and applies any penalties or notifications.
  10. The system displays a success confirmation message.
  11. The system refreshes the report list and overall statistics.
Alternative Flows:

A.5 Select Non-resolved Status

  1. The actor selects a status other than "Accept Report" (e.g., Dismiss Report, Under Review, Escalate).
  2. The system automatically unchecks and disables the "Flag Course / Remove content" checkbox.
  3. Back to step 6.

A.6 Toggle Content Removal Checkbox

  1. The actor checks the "Flag Course / Remove content" checkbox.
  2. The system displays a secondary confirmation popup warning about the consequences of removing or flagging the content.
  3. The actor confirms the action.
  4. The system updates the checkbox state to checked.
  5. Back to step 6.

A.6.2 Cancel Content Removal

  1. The actor clicks "Cancel" on the secondary confirmation popup.
  2. The system keeps the checkbox unchecked.
  3. Back to step 6.
Exceptions:

EX-2.1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2.2 Unauthorized Access

  1. The system detects the actor does not have the Staff or Admin role.
  2. The system redirects the actor to the home page or access denied page.

EX-9.1 Moderation Action Rejected

  1. The server rejects the moderation action (e.g., business rule violation, lack of permissions to perform specific actions).
  2. The system displays an error popup with the failure message.
  3. Use case ends.

EX-9.2 Database Save Failure

  1. The system encounters an error while saving the moderation changes to the database.
  2. The system displays an error message regarding the data update failure.
  3. Use case ends.
