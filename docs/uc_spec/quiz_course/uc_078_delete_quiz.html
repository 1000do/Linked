UC ID and Name UC-78-Delete Quiz
Created By: Truong Phuoc Tai Date Created: 11/07/2026
Primary Actor: Instructor Secondary Actors: N/A
Trigger: Actor clicks the delete button for a specific quiz.
Description: Allows an Instructor to soft-delete a quiz they own, removing it from active lists.
Preconditions:

PRE-1. Actor must be the owner of the quiz.
Post-conditions:

POST-1. The quiz is marked as removed in the database.

POST-2. The actor is presented with a success message.
Normal Flow:

A. Delete Quiz Successfully

  1. Actor clicks the delete button for a specific quiz.
  2. The system validates the quiz ownership and deletion conditions.
  3. The system updates the quiz record to mark it as removed.
  4. The system saves the changes to the database.
  5. The system displays a success message that the quiz was deleted successfully.
Alternative Flows:

A.1 Cancel Deletion

  1. Actor clicks the cancel button on the deletion confirmation modal.
  2. The system closes the modal.
  3. Back to step 1.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Validation Failed

  1. The system detects the actor is unauthorized or the quiz is in an invalid state for deletion.
  2. The system displays a validation error message.
  3. Back to step 1.

EX-4 Database Save Failure

  1. The system encounters an error while saving to the database.
  2. The system displays a server error message.
  3. Back to step 1.
Priority: Medium
Frequency of Use: Low
Business Rules: N/A
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
