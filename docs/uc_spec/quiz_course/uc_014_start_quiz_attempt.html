UC ID and Name UC-14-Start Quiz Attempt
Created By: Truong Phuoc Tai Date Created: 11/07/2026
Primary Actor: User Secondary Actors: N/A
Trigger: Actor clicks the "Start Quiz" button.
Description: Allows an enrolled User to start a new attempt for a quiz, randomly fetching questions according to the quiz distribution settings and logging the attempt start time.
Preconditions:

PRE-1. Actor must be enrolled in the course containing the quiz.

PRE-2. The quiz must be available (e.g., active, within allowable attempts limits).
Post-conditions:

POST-1. A new quiz attempt record is saved to the database.

POST-2. The actor is presented with the quiz questions interface.
Normal Flow:

A. Start Quiz Attempt Successfully

  1. Actor accesses the quiz learning page.
  2. Actor clicks the "Start Quiz" button.
  3. The system retrieves the quiz details and questions.
  4. The system validates the quiz availability and the actor's enrollment status.
  5. The system randomly selects questions based on the question distribution settings.
  6. The system creates a new quiz attempt record.
  7. The system saves the attempt record to the database.
  8. The system displays the quiz questions for the actor to answer.
Alternative Flows: N/A
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-3 Quiz Not Found

  1. The system detects the requested quiz does not exist.
  2. The system displays a quiz not found error message.
  3. Back to step 1.

EX-4 Unauthorized Access or Quiz Unavailable

  1. The system detects the actor is not enrolled or the quiz is unavailable.
  2. The system displays an error message regarding unauthorized access.
  3. Back to step 1.

EX-7 Database Save Failure

  1. The system encounters an error while saving to the database.
  2. The system displays a server error message.
  3. Back to step 1.
Priority: High
Frequency of Use: High
Business Rules: BR-276
Other Information: N/A
Assumptions:

User has stable internet connection.
