UC ID and Name UC-16-View Attempt Detail
Created By: Truong Phuoc Tai Date Created: 11/07/2026
Primary Actor: User Secondary Actors: N/A
Trigger: Actor clicks on a specific quiz attempt from the history list.
Description: Allows a User to view the detailed results of a specific past quiz attempt, including the submitted answers and scores.
Preconditions:

PRE-1. The quiz attempt must exist and be owned by the actor.
Post-conditions:

POST-1. Detailed attempt results are displayed to the actor.
Normal Flow:

A. View Attempt Detail Successfully

  1. Actor clicks on a specific quiz attempt from the history list.
  2. The system retrieves the quiz attempt details and submitted answers.
  3. The system validates that the actor is the owner of the attempt.
  4. The system maps the attempt data to the detailed response format.
  5. The system displays the detailed attempt results to the actor.
Alternative Flows: N/A
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Attempt Not Found

  1. The system detects the requested attempt does not exist.
  2. The system displays an attempt not found error message.
  3. Back to step 1.

EX-3 Unauthorized Access

  1. The system detects the actor is not the owner of the attempt.
  2. The system displays an unauthorized access error message.
  3. Back to step 1.
Priority: High
Frequency of Use: High
Business Rules: N/A
Other Information: N/A
Assumptions:

User has stable internet connection.
