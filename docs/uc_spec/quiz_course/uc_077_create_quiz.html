UC ID and Name UC-77-Create Quiz
Created By: Truong Phuoc Tai Date Created: 11/07/2026
Primary Actor: Instructor Secondary Actors: N/A
Trigger: Actor fills quiz details and clicks Create.
Description: Allows an Instructor to create a new quiz by providing its settings, configurations, and defining question distributions.
Preconditions:

PRE-1. None.
Post-conditions:

POST-1. The new quiz and its distribution settings are saved to the database.

POST-2. The actor is presented with a success message.
Normal Flow:

A. Create Quiz Successfully

  1. Actor submits the quiz details and configurations in the editor page.
  2. The system validates the quiz options and title uniqueness.
  3. The system validates the quiz question distributions.
  4. The system maps the request to a new quiz entity.
  5. The system saves the new quiz and its distributions to the database.
  6. The system maps the created quiz to the response format.
  7. The system displays a success message that the quiz was created successfully.
Alternative Flows:

A.1 Invalid Form Submission

  1. Actor attempts to submit the form with invalid or missing data.
  2. The system validates the form inputs.
  3. The system detects validation errors.
  4. The system displays frontend validation error messages.
  5. Back to step 1.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Validation Failed

  1. The system detects the quiz options or title are invalid.
  2. The system displays a validation error message.
  3. Back to step 1.

EX-3 Distributions Invalid

  1. The system detects the configured question distributions are invalid.
  2. The system displays an invalid distribution error message.
  3. Back to step 1.

EX-6 Database Save Failure

  1. The system encounters an error while saving to the database.
  2. The system displays a server error message.
  3. Back to step 1.
Priority: High
Frequency of Use: Medium
Business Rules: N/A
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
