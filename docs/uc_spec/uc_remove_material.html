UC-RemoveMaterial - Remove Learning Materials

Primary Actor:
Instructor

Secondary Actors:
Cloud Storage Service

Trigger:
Instructor clicks the "Remove" icon button next to a specific resource material, or clicks the "Replace Video" button for an existing active video.

Description:
Allows an instructor to remove an existing learning material (such as a video or document) from a lesson. The system soft-deletes the material by moving the physical file to the cloud storage trash folder, updates the material's status to removed, and reverts the course status to draft if it was previously published. This use case also encompasses the automatic removal of an active video when a new video is uploaded to replace it.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The material must exist and belong to a course owned by the active instructor.

Post-conditions:

POST-1. The physical file is moved to the Cloud Storage Service's trash folder.

POST-2. The material record in the database is updated with a removed learning status.

POST-3. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

POST-4. Stale cache data is cleared.

Normal Flow

A. Remove Material Manually Successfully

  1. Instructor accesses the course edit page.
  2. Instructor selects a specific lesson from the curriculum list.
  3. Instructor clicks the "Remove" icon button next to a specific resource material.
  4. The system displays a removal confirmation dialog.
  5. Instructor confirms the removal.
  6. The system validates the instructor's rights.
  7. The system validates the course status.
  8. The system moves the physical file to the Cloud Storage Service's trash folder.
  9. The system updates the material record's learning status to removed in the database.
  10. The system updates the course status to draft and clears previous moderation feedback (if previously published).
  11. The system invalidates the cached course data.
  12. The system displays a success message.

Alternative Flows

A.3 Replace Existing Video Successfully

  1. Instructor clicks the "Replace Video" button for an existing active video.
  2. The system executes the flow defined in UC-AddMaterials to upload and save the new video.
  3. The system soft-deletes the previously active video by moving its physical file to the Cloud Storage Service's trash folder and updating its learning status to removed in the database.
  4. Use case ends.

A.5 Removal Cancelled

  1. Instructor clicks "Cancel" or dismisses the confirmation dialog.
  2. Back to step 3.

A.10 Course is not published

  1. The system skips updating the course status and moderation feedback.
  2. Back to step 11.

Exceptions

EX-1.1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-1.2 Material Not Found

  1. The system halts execution and displays a material not found error message.

EX-6 Invalid Instructor Rights

  1. The system detects the instructor is unauthorized to remove this material or has an active lockout status.
  2. The system displays an error message regarding invalid instructor rights.
  3. Back to step 3.

EX-7 Invalid Course Status

  1. The system detects the course is pending review or is permanently archived due to policy violations.
  2. The system displays an error message regarding invalid course status.
  3. Back to step 3.

EX-9 Database Save Failed

  1. The system fails to save the material changes to the database.
  2. The system displays a database transaction failure message.
  3. Back to step 3.

Business Rules

  1. Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from removing materials.
  2. Pending Restriction: Materials cannot be removed while the course is currently in 'Pending' review status.
  3. Archived Course Restriction: Courses that are permanently archived due to policy violations (flag count >= 3) cannot be modified.
  4. Soft Deletion: Removing a material does not permanently delete the file; the physical file is moved to the Cloud Storage Service's trash folder and its database status is updated to 'Removed'.
  5. Video Replacement: Each lesson can only have one active video. Uploading a new video automatically soft-deletes the previous active video by moving it to the trash and marking it as removed.
  6. Status Reversion: Removing a material from a 'Published' course automatically changes its status back to 'Draft' and clears any existing moderation feedback.

Assumptions

Instructor has stable internet connection.
Cloud Storage Service is working properly.
