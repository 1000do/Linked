UC-RemoveMaterial - Remove Learning Materials

Primary Actor:
Instructor

Secondary Actors:
Cloud Storage Service

Trigger:
Instructor clicks the "Remove" icon button on a resource material OR clicks the "Replace video" icon button for the video material in the active lesson editor.

Description:
Allows an instructor to remove an existing learning material (resource) or replace an existing video. The system handles soft-deleting the removed material (or replacing it), invalidating caches, and reverting the course status if published.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The material must exist and belong to a lesson in a course owned by the active instructor.

Post-conditions:

POST-1. The removed (or replaced) learning material is marked as removed (soft-deleted) in the database.

POST-2. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

POST-3. Stale cache data is cleared.

Normal Flow

A. Remove Learning Materials Successfully

  1. Instructor clicks the "Remove" icon button on a specific resource material.
  2. The system prompts the instructor to confirm the removal.
  3. Instructor confirms the removal by clicking "Remove".
  4. The system validates the instructor's permission.
  5. The system validates the instructor's lockout status.
  6. The system validates the course status.
  7. The system updates the learning material record's status to 'removed' in the database.
  8. The system updates the course status to draft and clears moderation feedback in the database.
  9. The system invalidates cached course data.
  10. The system updates the UI to reflect the removed resource.

Alternative Flows

A.1 Replace Video Material

  1. Instructor clicks the "Replace video" icon button for the video material.
  2. The system opens the file selection dialog.
  3. Instructor selects a new video file to upload.
  4. The system executes UC-AddMaterial - Add Learning Materials to handle the file validation and upload process.
  5. The system soft-deletes the old video record and links the newly uploaded video material to the lesson.
  6. Continue to step 8.

A.2 Cancel Removal

  1. The system prompts for confirmation.
  2. Instructor clicks "Cancel".
  3. The system halts the removal process.
  4. Back to step 1.

Exceptions

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-4 Instructor Unauthorized

  1. The system detects the instructor does not own the course.
  2. The system displays an authorization error message.
  3. Back to step 1.

EX-5 Instructor Locked Out

  1. The system detects the instructor is currently locked out.
  2. The system displays a lockout error message.
  3. Back to step 1.

EX-6 Course Pending Review

  1. The system detects the course is currently pending review.
  2. The system displays a restriction error message.
  3. Back to step 1.

Business Rules

  1. Soft Deletion: Removed materials are not permanently deleted immediately but are marked as 'removed' and sent to the instructor's trash bin.
  2. Course Status Reversion: Modifying or removing materials from a previously published course automatically forces the course back to draft status, requiring a re-publish and moderation cycle.

Assumptions

Instructor has stable internet connection.
Cloud Storage Service is working properly.
