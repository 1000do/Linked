UC ID and Name UC-58-Edit Learning Material Details
Created By: Truong Phuoc Tai Date Created: 12/06/2026
Primary Actor: Instructor Secondary Actors: N/A
Trigger: Instructor clicks the "Save" button after editing video details, or clicks the "Save Changes" button after editing a resource file's details in the lesson editor.
Description: Allows an instructor to edit the title and description of an existing learning material (video or resource file). The system saves the updated details to the database. Modifying material details automatically reverts the course status to draft if it was previously published.
Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The material must exist and belong to a lesson in a course owned by the active instructor.
Post-conditions:

POST-1. The material's title and description are updated in the database.

POST-2. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

POST-3. Stale cache data is cleared.
Normal Flow:

A. Edit Material Details Successfully

  1. Instructor accesses the course edit page.
  2. Instructor modifies the title or description of a video or resource file in the active lesson editor.
  3. Instructor clicks the save button.
  4. The system validates the instructor's rights.
  5. The system validates the course status.
  6. The system updates the learning material's details in the database.
  7. The system updates the course status to draft and clears moderation feedback in the database.
  8. The system invalidates cached course data.
  9. The system displays a success confirmation message and updates the UI to reflect the updated details.
Alternative Flows:

A.3.1 No Active Video Exists

  1. Instructor clicks the save button on the video details panel.
  2. The system detects no active video exists for the current lesson.
  3. The system displays a warning message.
  4. Back to step 2.

A.3.2 Invalid Form Input

  1. Instructor clicks the save button.
  2. The system detects one or more form inputs are invalid.
  3. The system displays a validation warning message.
  4. Back to step 3.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-4 Invalid Instructor Rights

  1. The system detects the instructor's rights are invalid.
  2. The system displays an error message.
  3. Back to step 1.

EX-5 Invalid Course Status

  1. The system detects the course status does not permit modifications.
  2. The system displays a restriction error message.
  3. Back to step 1.

EX-6 Database Update Failed

  1. The system fails to save the material changes.
  2. The system displays a save failure error message.
  3. Back to step 1.
Priority: High
Frequency of Use: High
Business Rules: BR-145, BR-108, BR-8
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
