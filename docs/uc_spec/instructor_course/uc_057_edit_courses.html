UC ID and Name UC-53-Edit Course
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Instructor Secondary Actors: Cloud Storage Service
Trigger: Instructor clicks the "edit" icon of a course from his/her course list.
Description: Allows instructors to modify details of their courses (title, description, category, price, requirements, and promotional thumbnails) with integrated automatic debounced saving and content security hashing.
Preconditions:

PRE-1. User must be successfully authenticated as an approved Instructor.

PRE-2. The course must exist in the database and belong to the active Instructor's Account ID.
Post-conditions:

POST-1. Course details are updated in the primary course tables.

POST-2. Content hashes are computed and updated in the secondary table.

POST-3. The stale Redis cache for this course is invalidated.
Normal Flow:

A. Edit Course Details Successfully

  1. Instructor accesses the course editor.
  2. Instructor modifies the course details.
  3. The system triggers a brief debounce delay.
  4. The system validates the form inputs.
  5. The system validates the instructor's rights.
  6. The system validates the course status.
  7. The system retrieves the thumbnail from the Cloud Storage Service.
  8. The system computes the course fingerprint hashes.
  9. The system saves the updated course details and fingerprint hashes in the database.
  10. The system invalidates the cached course data.
  11. The system displays a success message.
Alternative Flows:

A.4 Invalid Form Input

  1. The system detects one or more form inputs are invalid.
  2. The system displays a validation error message.
  3. Back to step 2.
Exceptions:

EX-1.1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-1.2 Course Not Found

  1. The system displays a course not found error message.
  2. The system redirects the user back to the course list page.

EX-5 Invalid Instructor Rights

  1. The system detects the instructor is unauthorized to edit this course or has an active lockout status.
  2. The system displays an error message regarding invalid instructor rights.
  3. Back to step 1.

EX-6 Invalid Course Status

  1. The system detects the course is pending review or is permanently discontinued due to policy violations.
  2. The system displays an error message regarding invalid course status.
  3. Back to step 1.

EX-7 File Upload Failed

  1. The system fails to upload the thumbnail file to the Cloud Storage Service.
  2. The system displays a file upload error message.
  3. Back to step 2.

EX-8 Duplicate Course Content Detected

  1. The system detects a duplication in course content using the computed fingerprint.
  2. The system displays an error message regarding content duplication.
  3. Back to step 2.

EX-9 Database Save Failed

  1. The system fails to save the course updates or fingerprint hashes to the database.
  2. The system displays a database transaction failure message.
  3. Back to step 2.
Priority: High
Frequency of Use: High
Business Rules: BR-139, BR-140, BR-104, BR-8, BR-141
Other Information: N/A
Assumptions:

Instructor has stable internet connection.

Cloud Storage Service is working properly.
