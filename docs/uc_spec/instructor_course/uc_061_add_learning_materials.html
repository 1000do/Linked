UC ID and Name UC-57-Add Learning Materials
Created By: Truong Phuoc Tai Date Created: 12/06/2026
Primary Actor: Instructor Secondary Actors: Cloud Storage Service
Trigger: Instructor clicks the "Upload Video" area for video content, or clicks the "Choose File" button for supplementary files in the lesson editor.
Description: Allows an instructor to add new learning materials (such as videos or document files) to an empty lesson. The system enforces business rules including Stripe account limits, handles file uploads to the cloud, and updates the course status to draft if it was previously published.
Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The lesson must exist and belong to a course owned by the active instructor.
Post-conditions:

POST-1. The new material is uploaded and linked to the lesson.

POST-2. The physical file is stored in the Cloud Storage Service.

POST-3. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

POST-4. Stale cache data is cleared.
Normal Flow:

A. Add Material Successfully

  1. Instructor accesses the course edit page.
  2. Instructor selects a specific lesson from the curriculum list.
  3. Instructor clicks the "Upload Video" area for video content, or clicks the "Choose File" button for supplementary files.
  4. The frontend system directly uploads the file to the Cloud Storage Service and submits the resulting URL to the backend.
  5. The system validates the instructor's rights.
  6. The system validates the course status.
  7. The system validates the document limits.
  8. The system creates the new material record in the database.
  9. The system updates the course status to draft and clears previous moderation feedback (if previously published).
  10. The system invalidates the cached course data.
  11. The system displays a success message.
Alternative Flows:

A.4 Upload cancelled or failed

  1. The system displays an upload failure message.
  2. Back to step 3.

A.9 Course is not published

  1. The system skips updating the course status and moderation feedback.
  2. Back to step 10.
Exceptions:

EX-1.1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Lesson Not Found

  1. The system halts execution and displays a lesson not found error message.

EX-5 Invalid Instructor Rights

  1. The system detects the instructor is unauthorized to add material to this lesson or has an active lockout status.
  2. The system displays an error message regarding invalid instructor rights.
  3. Back to step 3.

EX-6 Invalid Course Status

  1. The system detects the course is pending review or is permanently archived due to policy violations.
  2. The system displays an error message regarding invalid course status.
  3. Back to step 3.

EX-7 Document Limit Exceeded

  1. The system detects the instructor has not linked an active Stripe Connect profile and already reached the maximum of 2 documents for the lesson.
  2. The system displays an error message regarding the document limit.
  3. Back to step 3.

EX-8 Database Save Failed

  1. The system fails to save the material changes to the database.
  2. The system displays a database transaction failure message.
  3. Back to step 3.
Priority: High
Frequency of Use: High
Business Rules: BR-8, BR-144, BR-108, BR-147, BR-145
Other Information: N/A
Assumptions:

Instructor has stable internet connection.

Cloud Storage Service is working properly.
