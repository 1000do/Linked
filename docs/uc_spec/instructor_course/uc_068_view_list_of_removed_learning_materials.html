UC ID and Name UC-63-View list of removed learning materials
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Instructor Secondary Actors: N/A
Trigger: Instructor clicks the "Trash" button (or similar) on the Instructor Studio dashboard.
Description: Displays a list of deleted learning materials (instructor's trash bin).
Preconditions:

PRE-1. Instructor must be authenticated.

PRE-2. Instructor's status must be "Approved".
Post-conditions:

POST-1. The system displays a list of soft-deleted learning materials belonging to the instructor's courses.
Normal Flow:

A. View Removed Learning Materials Successfully

  1. Instructor accesses the Instructor Studio dashboard page.
  2. The system checks the instructor's session authentication.
  3. The system checks the instructor's approval status.
  4. Instructor clicks the "Trash" button.
  5. The system checks the instructor's session authentication.
  6. The system retrieves the list of soft-deleted learning materials for the instructor.
  7. The system calculates the remaining days before auto-deletion for each material.
  8. The system displays the trash bin page, showing the list of removed learning materials, their title, source lesson title, source course title, deleted timestamp, and remaining days.
Alternative Flows:

A.8.1 Restore Material

  1. Instructor locates the removed material they wish to restore.
  2. Instructor clicks the "Restore" button on the corresponding material row.
  3. The system displays a confirmation popup dialog.
  4. Instructor clicks the "Yes, restore" button.
  5. The system checks the instructor's session authentication.
  6. The system checks if the instructor's rights are locked.
  7. The system checks if the course containing the lesson is pending review.
  8. The system checks if the course is permanently discontinued.
  9. The system checks the instructor's ownership of the material.
  10. The system checks if the restored material is a video and the lesson already has an active video.
  11. The system moves the existing active video to the trash (if step 10 is true).
  12. The system restores the material file in cloud storage.
  13. The system updates the material status to active in the database.
  14. The system clears the course details cache.
  15. The system displays a success confirmation message.
  16. The system dynamically removes the restored material row from the list.
  17. The flow terminates.

A.8.2 Permanent Delete Material

  1. Instructor locates the removed material they wish to delete forever.
  2. Instructor clicks the "Delete Forever" button on the corresponding material row.
  3. The system displays a confirmation popup dialog warning of irreversible deletion.
  4. Instructor clicks the "Yes, delete forever" button.
  5. The system checks the instructor's session authentication.
  6. The system checks if the instructor's rights are locked.
  7. The system checks if the course containing the lesson is pending review.
  8. The system checks if the course is permanently discontinued.
  9. The system checks the instructor's ownership of the material.
  10. The system deletes the material file from cloud storage.
  11. The system deletes the material record from the database.
  12. The system clears the course details cache.
  13. The system displays a success confirmation message.
  14. The system dynamically removes the deleted material row from the list.
  15. The flow terminates.

A.8.3 Cancel Action

  1. Instructor clicks the "Cancel" button on any confirmation popup dialog.
  2. The system closes the confirmation popup dialog.
  3. Back to step 8.
Exceptions:

EX-2 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-3 Account Not Approved

  1. The system detects the actor's instructor approval status is not "Approved".
  2. The system redirects the actor to the application status or apply page.
  3. The flow terminates.

EX-5.1 Restore/Delete Rights Locked

  1. The system detects that the instructor's account rights are locked due to policy violations.
  2. The system displays an error message indicating their rights are locked and changes cannot be saved.
  3. The flow terminates.

EX-5.2 Course Pending Review

  1. The system detects the course containing the lesson is currently pending review.
  2. The system displays an error message indicating curriculum changes are blocked during review.
  3. The flow terminates.

EX-5.3 Course Permanently Discontinued

  1. The system detects the course is permanently discontinued due to policy violations (3 or more flags).
  2. The system displays an error message indicating modifications are blocked.
  3. The flow terminates.

EX-5.4 Unauthorized Access

  1. The system detects that the instructor does not own the course/material.
  2. The system displays an error message indicating permission denied.
  3. The flow terminates.

EX-5.5 Database Save Failure

  1. The database fails to commit changes.
  2. The system displays a system error message.
  3. The flow terminates.
