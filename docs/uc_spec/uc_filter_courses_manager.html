UC ID and Name UC-91-Filter Courses
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Actor selects a category from the "category" dropdown or a status from the "status" dropdown on the Course Moderation Page.
Description: Filters courses by categories or moderation status (Pending, Approved, Rejected).
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must have staff or admin role.

PRE-3. Actor must be on the Course Moderation Page.
Post-conditions:

POST-1. The system displays the course moderation list filtered by the selected criteria.
Normal Flow:

A. Filter Courses Successfully

  1. Actor selects a category from the "category" dropdown, or a status from the "status" dropdown.
  2. The system retrieves the courses list matching the selected criteria and other active search keywords or sorting options.
  3. The system updates the course moderation table and statistics cards.
  4. The system displays the filtered course list.
Alternative Flows:

A.1.1 Change Filter Selection

  1. Actor selects a different category from the "category" dropdown, or a different status from the "status" dropdown.
  2. The system retrieves the courses list matching the new criteria.
  3. The system updates the course moderation table and statistics cards.
  4. The system displays the updated filtered course list.
  5. Back to step 1.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-2 Unauthorized Access

  1. The system detects that the actor does not have staff or admin roles.
  2. The system displays an access denied page or error message.
  3. The flow terminates.

EX-3 API Call Failed

  1. The system fails to retrieve the filtered course list or stats from the API.
  2. The system displays an error message regarding filtering failure.
  3. The system renders the Course Moderation page with an empty list.
  4. The flow terminates.
Priority: High
Frequency of Use: High
Business Rules:
  1. Exclusion of Drafts: Filtered results never include draft courses.
  2. Category Filter: Matches the Category ID or Name against the course's category. If "All Categories" is selected, no category filtering is applied.
  3. Status Filters:
    • "All" ("all"): Shows all courses except draft status.
    • "Pending" ("pending"): Shows courses that are submitted and waiting for review.
    • "Approved" ("published"): Shows courses that have been approved.
    • "Rejected" ("rejected"): Shows courses that have been rejected.
    • "Archived" ("archived"): Shows archived courses (with violation flags less than 3).
    • "Permanently Locked" ("permanently_locked"): Shows archived courses that have 3 or more violation flags.
  4. Preservation of Other Criteria: Filtering preserves active search keywords and sorting options.
Other Information: N/A
Assumptions:

Staff has stable internet connection.

Admin has stable internet connection.
