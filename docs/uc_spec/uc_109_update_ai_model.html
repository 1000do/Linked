UC ID and Name UC-109 - Update AI Model
Created By: An Date Created: 11/06/2026
Primary Actor: Admin Secondary Actors: N/A
Trigger: Admin clicks the edit icon on a model entry in the AI model list.
Description: Allows an Admin to update an existing AI model's configuration details. The edit form allows modification of provider, version, path, and description. The system validates all required fields before persisting the changes to the database.
Preconditions:

PRE-1. Admin must be successfully authenticated.
Post-conditions:

POST-1. The selected AI model record is updated in the database.

POST-2. The model list on the UI reflects the updated information after page reload.
Normal Flow:

A. Edit AI Model Details Successfully

  1. Admin accesses the AI Service management page.
  2. The system validates the Admin's access authorization.
  3. Admin clicks the edit icon on a model entry.
  4. The system displays the edit modal pre-filled with the current model details.
  5. Admin modifies the model details.
  6. Admin clicks the "Save Changes" button.
  7. The system displays a confirmation dialog.
  8. Admin confirms the action.
  9. The system validates the form inputs.
  10. The system updates the model record in the database.
  11. The system displays a success notification.
  12. The system reloads the page.
Alternative Flows:

A.8 Admin Cancels Confirmation

  1. Admin clicks the cancel button on the confirmation dialog.
  2. Back to step 4.

A.9 Form Validation Failed

  1. The system detects one or more required fields are invalid.
  2. The system displays a validation error message.
  3. Back to step 4.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Unauthorized Access

  1. The system detects the current user does not have Admin authorization.
  2. The system displays an unauthorized access error message.
  3. The system redirects the user to the login page.

EX-10.1 Model Not Found

  1. The system cannot locate the target model record during the update operation.
  2. The system displays a model not found error message.
  3. Back to step 4.

EX-10.2 Database Save Failed

  1. The system fails to persist the model changes to the database.
  2. The system displays an error message regarding the database save failure.
  3. Back to step 4.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-1, BR-25, BR-26
Other Information: N/A
Assumptions:

Admin has stable internet connection.
