UC ID and Name UC-52-Create Courses
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Instructor Secondary Actors: N/A
Trigger: Instructor clicks the "Create New Course" button (or similar) on the Instructor Studio dashboard.
Description: Drafts basic details for a new course, including title, pricing, description, and curriculum outline.
Preconditions:

PRE-1. Instructor must be authenticated.

PRE-2. Instructor's application status must be "Approved".
Post-conditions:

POST-1. The draft course record is created in the database with status set to "Draft".

POST-2. The system redirects the Instructor to the Course Editor page.
Normal Flow:

A. Create Course Successfully

  1. Instructor clicks the "Create New Course" button on the Instructor Studio dashboard.
  2. The system checks the instructor's session authentication.
  3. The system checks the instructor's approval status.
  4. The system retrieves the available course categories.
  5. The system displays Step 1 (Core Identity) of the creation wizard, showing the "Course Title" input and "Category" dropdown.
  6. Instructor enters a title, selects a category, and clicks the "Next" button.
  7. The system validates the inputs for Step 1.
  8. The system displays Step 2 (Description & Curriculum) of the creation wizard, showing the rich text editors for Course Description, What You Will Learn, and Requirements.
  9. Instructor enters the description, learning achievements, and requirements, then clicks the "Next" button.
  10. The system validates the rich text fields.
  11. The system displays Step 3 (Course Thumbnail) of the creation wizard, showing the "drop-area" file upload zone.
  12. Instructor uploads a course thumbnail image.
  13. The system uploads the image to the cloud storage service.
  14. The system retrieves the returned image URL.
  15. The system stores the URL in the "CourseThumbnailUrl" variable.
  16. Instructor clicks the "Next" button.
  17. The system validates that the thumbnail URL is present.
  18. The system displays Step 4 (Access & Pricing) of the creation wizard, showing the "Price" input and "Apply coupon to this course" switch.
  19. Instructor enters the price and toggles the coupon switch.
  20. Instructor clicks the "Select System Voucher" button.
  21. The system displays the platform coupons list in the "coupon-modal".
  22. Instructor selects a platform coupon and clicks the "Create Course" button.
  23. The system validates Step 4 price inputs.
  24. The system sends the course creation payload to the backend server.
  25. The backend server validates the request model state.
  26. The backend server checks the instructor's creation rights.
  27. The backend server checks for duplicate course content.
  28. The backend server inserts the draft course record into the database.
  29. The backend server saves the changes.
  30. The system displays a success confirmation popup dialog.
  31. The system redirects the instructor to the Course Editor page.
Alternative Flows:

A.6.1 Cancel Creation

  1. Instructor clicks the "Cancel" button on the floating action bar.
  2. The system redirects the instructor back to the Instructor Studio dashboard.
  3. The flow terminates.

A.7.1 Step 1 Validation Failure

  1. The system detects one or more inputs in Step 1 are invalid (Title is empty, Title is under 5 characters, Title exceeds 100 characters, or Category is not selected).
  2. The system displays a validation warning alert box detailing the issues.
  3. Instructor resolves the issues.
  4. Back to step 6.

A.10.1 Step 2 Validation Failure

  1. The system detects one or more inputs in Step 2 are invalid (Description is empty, What You Will Learn is empty/under 20 characters, or Requirements is empty/under 20 characters).
  2. The system displays a validation warning alert box detailing the issues.
  3. Instructor resolves the issues.
  4. Back to step 9.

A.17.1 Thumbnail Missing

  1. The system detects that the course thumbnail has not been uploaded.
  2. The system displays a validation warning alert box.
  3. Instructor uploads a thumbnail.
  4. Back to step 16.

A.18.1 Stripe Payments Not Active

  1. The system detects that the instructor has not connected Stripe payments.
  2. The system disables premium pricing tier inputs, setting the price default to "0.00 — Free".
  3. The system displays a warning message advising that connecting Stripe is required to unlock premium paid tiers, along with a link to connect Stripe.
  4. Back to step 19.

A.19.1 Disable Platform Vouchers

  1. Instructor disables the "coupon-toggle" switch.
  2. The system hides the voucher selection container and sets the Coupon ID to null.
  3. Back to step 22.

A.23.1 Step 4 Validation Failure

  1. The system detects that the price is invalid (Price is empty, negative, or exceeds 10000 USD).
  2. The system displays a validation warning alert box.
  3. Instructor resolves the price.
  4. Back to step 19.
Exceptions:

EX-2 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-3 Account Not Approved

  1. The system detects the actor's instructor approval status is not "Approved".
  2. The system redirects the actor to the application status or apply page.
  3. The flow terminates.

EX-24 Backend Connection Failure / System Error

  1. The backend server returns a connection failure or server error.
  2. The system displays a system error alert box detailing the failure.
  3. The flow terminates.

EX-25 Invalid Backend Request State

  1. The backend server returns a bad request status code (400).
  2. The system displays a validation warning alert box with the validation details.
  3. Back to step 22.

EX-26 Unauthorized Instructor Rights

  1. The backend server returns a bad request status code (400) due to invalid instructor rights.
  2. The system displays an error message indicating unauthorized creation.
  3. The flow terminates.

EX-27 Duplicate Course Content

  1. The database throws a uniqueness/duplicate content exception.
  2. The backend server handles the database error and returns a bad request status code (400) with a duplicate content warning.
  3. The system displays a system error alert box indicating duplicate course content is detected.
  4. Back to step 6.

EX-29 Database Insertion Failure

  1. The backend server throws an invalid operation exception.
  2. The backend server returns a bad request status code (400).
  3. The system displays a system error alert box detailing the insertion failure.
  4. The flow terminates.
Priority: High
Frequency of Use: Medium
Business Rules:
  1. Title Length Constraints: The course title must be between 5 and 100 characters in length.
  2. Text Constraints: What You Will Learn and Requirements descriptions must each be at least 20 characters in length (excluding HTML tags).
  3. Stripe Payout Connectivity: Instructors must connect and activate their Stripe accounts to set premium prices. If Stripe is disconnected or inactive, the course price is locked to 0.00 (Free).
  4. Voucher Validation: Platform sponsored discount vouchers cannot be applied to free courses.
  5. Draft Initial State: All newly created courses are saved with a status of "Draft" and must go through moderation before being published.
Other Information: N/A
Assumptions:

Instructor has stable internet connection.
