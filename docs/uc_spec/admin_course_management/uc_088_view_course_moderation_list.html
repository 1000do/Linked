UC ID and Name UC-88-View Course Moderation List
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Actor accesses the Course Moderation Page.
Description: Displays all courses submitted by instructors that are pending review.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must have staff or admin role.
Post-conditions:

POST-1. The system displays the pending courses list and moderation stats.
Normal Flow:

A. View Course Moderation List Successfully

  1. Actor accesses the Course Moderation Page.
  2. The system retrieves all categories.
  3. The system retrieves the pending/under-review courses list matching current filters, sorting, and pagination parameters.
  4. The system calculates the urgency level and urgency color for each pending course.
  5. The system retrieves the course moderation statistics.
  6. The system displays the moderation page containing stats cards, filters, and the course moderation table.
Alternative Flows:

A.6.1 Search Courses

  1. Actor enters search keywords in the search input box.
  2. The system retrieves the courses list filtered by search keywords.
  3. The system updates the course moderation table and statistics cards.
  4. Back to step 6.

A.6.2 Filter Courses by Category

  1. Actor selects a category from the "category" dropdown.
  2. The system retrieves the courses list filtered by category.
  3. The system updates the course moderation table and statistics cards.
  4. Back to step 6.

A.6.3 Filter Courses by Status

  1. Actor selects a status from the "status" dropdown.
  2. The system retrieves the courses list filtered by status.
  3. The system updates the course moderation table and statistics cards.
  4. Back to step 6.

A.6.4 Sort Courses

  1. Actor selects a sorting option from the "sortBy" dropdown.
  2. The system retrieves the courses list sorted accordingly.
  3. The system updates the course moderation table and statistics cards.
  4. Back to step 6.

A.6.5 Clear Filters

  1. Actor clicks the "refresh" icon button to clear filters.
  2. The system resets the filters and sorting options to their default values.
  3. The system retrieves the courses list using the default parameters.
  4. The system updates the course moderation table and statistics cards.
  5. Back to step 6.

A.6.6 Preview Course Details

  1. Actor clicks the "visibility" icon button on a course row.
  2. The system opens the course details page in a new browser tab.
  3. Back to step 6.

A.6.7 Navigate Pagination

  1. Actor clicks a page number, or the "chevron_left" / "chevron_right" icon button.
  2. The system retrieves the courses list for the selected page.
  3. The system updates the course moderation table and statistics cards.
  4. Back to step 6.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-2 Unauthorized Access

  1. The system detects that the actor does not have staff or admin roles.
  2. The system displays an access denied page or error message.
  3. The flow terminates.

EX-3 API Call Failed

  1. The system fails to retrieve categories, pending courses list, or stats from the API.
  2. The system displays an error message regarding API call failure.
  3. The system renders the Course Moderation page with an empty list and stats reset to zero.
  4. The flow terminates.
Priority: High
Frequency of Use: Medium
Business Rules: BR-120, BR-121, BR-122, BR-123
Other Information: N/A
Assumptions:

Staff has stable internet connection.

Admin has stable internet connection.
