UC ID and Name UC-93-Reject Course Submission
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Staff, Admin Secondary Actors: N/A
Trigger: Actor clicks the "Reject" button on a course row in the moderation list.
Description: Rejects a submitted course with reasons and feedback for the instructor.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must have staff or admin role.

PRE-3. The course status must be pending review (or approved/published if rejecting an active course).
Post-conditions:

POST-1. The course status is set to "rejected" in the database.

POST-2. Tagged lessons and materials (if any) are set to "rejected" in the database.

POST-3. The instructor receives a notification of the rejection.
Normal Flow:

A. Reject Course Submission Successfully

  1. Actor accesses the Course Moderation Page.
  2. Actor locates the pending course in the moderation list.
  3. Actor clicks the "Reject" button on the corresponding course row.
  4. The system retrieves the course details (lessons and materials).
  5. The system displays a detailed reject modal containing tagged element selectors and reason text fields.
  6. Actor clicks the "Add Rejection Item" button to add a new violation tag row.
  7. Actor selects a violating element from the dropdown (e.g. course title, course description, course thumbnail, specific lesson, or specific material file).
  8. Actor enters a rejection reason for the selected element.
  9. Actor clicks the "Submit Rejection" button.
  10. The system updates the status of tagged lessons and materials to "rejected" and saves their specific rejection feedback in the database.
  11. The system updates the course status to "rejected" and saves the combined moderation feedback in the database.
  12. The system sends a rejection notification message to the instructor.
  13. The system clears the course details cache.
  14. The system updates the moderation page stats and dims the rejected course row.
Alternative Flows:

A.5.1 Cancel Rejection

  1. Actor clicks the "Cancel" button on the reject modal.
  2. The system closes the reject modal.
  3. Back to step 2.

A.6.1 Remove Rejection Item Row

  1. Actor clicks the "delete" icon button next to a rejection item row.
  2. The system removes the rejection item row from the modal.
  3. Back to step 7.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-2 Unauthorized Access

  1. The system detects that the actor does not have staff or admin roles.
  2. The system displays an access denied page or error message.
  3. The flow terminates.

EX-4.1 Course Metadata Fetch Failed

  1. The system fails to retrieve course details from the API.
  2. The system displays an error message indicating failure to load course details.
  3. The reject modal shows a connection failure message.
  4. The flow terminates.

EX-9.1 Form Incomplete Validation

  1. The system detects that a tagged element dropdown is unselected or a reason text field is empty.
  2. The system displays a warning message indicating information is missing.
  3. The system halts submission.
  4. Back to step 7.

EX-10 Database Save Failure

  1. The database fails to commit changes.
  2. The system displays a system error message.
  3. The flow terminates.
Priority: High
Frequency of Use: Medium
Business Rules: BR-133, BR-135, BR-136
Other Information: N/A
Assumptions:

Staff has stable internet connection.

Admin has stable internet connection.
