UC ID and Name UC-ReportReview - Report a review
Created By: Huynh Kien An Date Created: 09/06/2026
Primary Actor: User Secondary Actors: N/A
Trigger: User clicks the "Report" button on a course review in the course details page.
Description: Allows a user to report a course or lesson review for violations such as spam, offensive content, or fake reviews. The system prompts the user to select a reason and optionally provide a description before submitting the report for moderation.
Preconditions:

PRE-1. User must be successfully authenticated.

PRE-2. User must have completed the course or lesson to be able to report the review.
Post-conditions:

POST-1. The review report is recorded in the database for moderation.
Normal Flow:

A. Report Course Review Successfully

  1. User accesses the course details page.
  2. User locates a course review and clicks the Report button.
  3. The system checks authentication.
  4. The system displays the report review form modal.
  5. User selects a violation reason from the dropdown.
  6. User optionally inputs additional description details.
  7. User clicks the Submit Report button.
  8. The system validates the report details and checks for duplicate pending reports.
  9. The system submits the course review report data to the server.
  10. The system saves the report in the database.
  11. The system displays a success message regarding report submission.
Alternative Flows:

A.1 Report Lesson Review Successfully

  1. User accesses the course learning page.
  2. User locates a lesson review and clicks the Report button.
  3. Back to step 3.

A.7 Cancel reporting

  1. User clicks the Cancel button.
  2. The system closes the report review modal.
  3. Back to step 2.
Exceptions:

EX-3 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-8 Duplicate Pending Report

  1. The system detects that the user has already reported this review with the exact same reason and the report is still pending.
  2. The system displays a duplicate report error message preventing submission.
  3. Back to step 4.

EX-9 Database Save Failed

  1. The system fails to save the report to the database.
  2. The system displays an error message regarding report submission failure.
  3. Back to step 4.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-3, BR-4
Other Information: N/A
Assumptions:

User has stable internet connection.
