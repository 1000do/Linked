UC ID and Name UC-06-Sort Courses
Created By: Truong Phuoc Tai Date Created: 10/06/2026
Primary Actor: User, Guest, Instructor, Staff, Admin Secondary Actors: N/A
Trigger: Actor selects a sorting option from the sort by dropdown list on the explore courses page.
Description: Allows Guests and Users to order the displayed courses by Newest, Highest Rated, Price: Low to High, or Price: High to Low.
Preconditions:

PRE-1. None.
Post-conditions:

POST-1. List of courses is displayed ordered by the selected sorting parameter.
Normal Flow:

A. Sort Courses Successfully

  1. Actor accesses the explore courses page.
  2. Actor clicks the "Sort By" dropdown.
  3. Actor selects a sorting parameter (e.g., "Newest", "Highest Rated", "Price: Low to High", or "Price: High to Low").
  4. The system updates the sort parameter.
  5. The system retrieves the paginated list of courses ordered by the selected sort parameter.
  6. The system displays the sorted courses in the catalog grid.
Alternative Flows:

A.5 Empty Sorted Results

  1. The system detects the course list is empty.
  2. The system displays the "No courses found" empty state block.
  3. Use case ends.
Exceptions: N/A
Priority: High
Frequency of Use: High
Business Rules: BR-67, BR-68, BR-69, BR-70, BR-71
Other Information: N/A
Assumptions:

User, Guest, Instructor, Staff, Admin has stable internet connection.
