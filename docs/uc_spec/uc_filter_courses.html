UC ID and Name UC-05-Filter Courses
Created By: Truong Phuoc Tai Date Created: 10/06/2026
Primary Actor: User, Guest, Instructor, Staff, Admin Secondary Actors: N/A
Trigger: Actor clicks a category pill, category option, price range radio, or rating option.
Description: Allows Guests and Users to filter the list of courses by category, rating stars, and price (Free vs. Paid). The filters can be applied singly or in combination.
Preconditions:

PRE-1. None.
Post-conditions:

POST-1. Paginated list of courses matching all active category, price, and rating filters is displayed successfully in the catalog grid.
Normal Flow:

A. Filter Courses Successfully

  1. Actor accesses the explore courses page.
  2. Actor selects a category filter (e.g., clicking a category pill).
  3. The system updates the active category filter.
  4. Actor selects a rating filter radio option under the "Rating" accordion.
  5. The system updates the active rating filter.
  6. Actor selects a price filter radio option under the "Price" accordion.
  7. The system updates the active price filter.
  8. The system retrieves the paginated list of courses matching all selected filter parameters.
  9. The system displays the filtered results in the catalog grid.
Alternative Flows:

A.2.1 Select Category Pill

  1. Actor clicks on a category pill (e.g., a specific category name or the "All Categories" pill).
  2. The system updates the category filter parameter.
  3. Back to step 8.

A.4 Toggle Accordion

  1. Actor clicks the toggle icon next to the "Rating" or "Price" accordion header.
  2. The system toggles the visibility of the accordion filter options.
  3. Back to step 9.

A.8 Empty Filter Results

  1. The system detects no courses match the filter criteria.
  2. The system displays a "No courses found" empty state message.
  3. The system renders a "Clear All Filters" button.
  4. Back to step 9.

A.9 Clear Filters via Sidebar Button

  1. Actor clicks the "Clear Filters" button in the sidebar.
  2. The system resets the search keyword, category, price, and rating filters.
  3. Back to step 8.

A.9.1 Remove Active Filter Badge

  1. Actor clicks the close icon on an active filter badge.
  2. The system resets the corresponding filter.
  3. Back to step 8.
Exceptions: N/A
Priority: High
Frequency of Use: High
Business Rules:
  1. Filter Combinability: Multiple filters (category, price, rating, keyword) can be applied simultaneously, narrowing the results conjunctively (AND operation).
  2. Rating Filter Bounds: Rating filters select courses with average rating greater than or equal to the selected threshold (e.g., >= 4.5, >= 4.0, >= 3.5).
  3. Price Filter Bounds: Price filter options classify courses as "Free" (Price == 0) or "Paid" (Price > 0).
Other Information: N/A
Assumptions:

User, Guest, Instructor, Staff, Admin has stable internet connection.
