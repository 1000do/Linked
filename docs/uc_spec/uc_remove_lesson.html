UC-RemoveLesson - Remove Lesson

Primary Actor:
Instructor

Secondary Actors:
Cloud Storage Service

Trigger:
Instructor clicks the "Remove Lesson" icon button on a specific lesson in the course syllabus sidebar.

Description:
Allows an instructor to remove an existing lesson and all of its associated materials from a course. The system ensures the instructor has ownership and the course is not currently pending review. It soft-deletes the records and moves associated material files to a cloud trash folder.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The lesson must exist and belong to a course owned by the active instructor.

Post-conditions:

POST-1. The lesson and its materials are marked as removed in the database.

POST-2. The material files are moved to the cloud trash folder.

POST-3. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

Normal Flow

A. Remove Lesson Successfully

  1. Instructor accesses the course edit page.
  2. Instructor clicks the "Remove Lesson" icon button.
  3. The system displays a confirmation modal.
  4. Instructor clicks the "Delete" button.
  5. The system validates the instructor's permission.
  6. The system validates the instructor's lockout status.
  7. The system validates the course status.
  8. The system requests the Cloud Storage Service to move all lesson materials to the trash.
  9. The system updates the lesson and its materials to removed status in the database.
  10. The system updates the course status to draft and clears moderation feedback in the database.
  11. The system invalidates cached course data.
  12. The system removes the lesson from the curriculum list.
  13. The system hides the active lesson editor and displays the empty state.
  14. The system displays a success confirmation message.

Alternative Flows

A.4 Instructor cancels removal

  1. Instructor clicks the "Cancel" button on the confirmation modal.
  2. The system closes the modal without making any changes.
  3. Back to step 2.

A.8 No materials with URLs

  1. The system skips requesting the Cloud Storage Service.
  2. Back to step 9.

A.10 Course is not published

  1. The system skips updating the course status and moderation feedback.
  2. Back to step 11.

Exceptions

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-5 Lesson Not Found

  1. The system halts execution and displays a lesson not found error message.
  2. Back to step 2.

EX-5.1 Permission Denied

  1. The system halts execution and displays a permission denied error message.
  2. Back to step 2.

EX-6 Instructor Account Locked

  1. The system halts execution and displays an account locked error message.
  2. Back to step 2.

EX-7 Course Pending Review

  1. The system halts execution and displays a course pending review error message.
  2. Back to step 2.

EX-8 Cloud Storage Service failure

  1. The system fails to move the materials to the trash.
  2. The system halts execution and displays a server error message.
  3. Back to step 2.

EX-9 Database update failed

  1. The system fails to save the lesson changes.
  2. The system displays a database transaction failure message.
  3. Back to step 2.

Business Rules

  1. Pending Restriction: Lessons cannot be deleted while the course is currently in 'Pending' review status.
  2. Status Reversion: Deleting a lesson from a 'Published' course automatically changes its status back to 'Draft' and clears any existing moderation feedback.
  3. Soft Deletion: Removing a lesson only marks it and its materials as "removed" (soft delete) in the database. Physical files are moved to the cloud trash folder rather than being permanently destroyed immediately.

Assumptions

Instructor has stable internet connection.
Cloud Storage Service is working properly.
