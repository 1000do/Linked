UC ID and Name UC-04-Search Courses
Created By: Truong Phuoc Tai Date Created: 10/06/2026
Primary Actor: User, Guest, Instructor, Staff, Admin Secondary Actors: N/A
Trigger: Actor enters a search term in the keyword input field on the explore courses page.
Description: Allows Guests and Users to search for courses by entering keywords that match titles, descriptions, and skills. The search is executed in real-time with a brief debounce delay, presenting matching results in the course catalog.
Preconditions:

PRE-1. None.
Post-conditions:

POST-1. Paginated list of courses matching the keyword search criteria is displayed successfully in the catalog grid.
Normal Flow:

A. Search Courses Successfully

  1. Actor accesses the explore courses page.
  2. Actor enters a search term in the "Search Keyword" input field.
  3. The system triggers the search query after a 350ms debounce delay.
  4. The system retrieves the paginated list of courses matching the keyword.
  5. The system displays the search results in the catalog grid.
Alternative Flows:

A.2.1 Clear Search via Close Button

  1. Actor clicks the "close" icon next to the search input.
  2. The system clears the search input field.
  3. Back to step 3.

A.4 Empty Search Results

  1. The system detects no courses match the search keyword.
  2. The system displays a "No courses found" empty state message.
  3. The system renders a "Clear All Filters" button.
  4. Back to step 5.

A.4.1 Clear All Filters from Empty State

  1. Actor clicks the "Clear All Filters" button in the empty state block.
  2. The system resets the search keyword, category, price, and rating filters.
  3. Back to step 3.

A.5 Navigate Pagination

  1. Actor clicks a page link or page navigation control (e.g., page number, left chevron "chevron_left", or right chevron "chevron_right").
  2. The system updates the page index.
  3. The system scrolls the viewport smoothly back to the top of the course catalog.
  4. Back to step 4.
Exceptions: N/A
Priority: High
Frequency of Use: High
Business Rules:
  1. Debounced Query Execution: Real-time keyword filtering is debounced by 350ms to minimize API calls during continuous typing.
  2. Keyword Match Scope: Search query checks against course titles, instructor names, short descriptions, and categories.
  3. Published Status Only: Only courses with "published" status are returned in the search results.
Other Information: N/A
Assumptions:

User, Guest, Instructor, Staff, Admin has stable internet connection.
