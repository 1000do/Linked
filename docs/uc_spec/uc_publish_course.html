UC-54 - Publish courses

Primary Actor:
Instructor

Secondary Actors:
AI Service

Trigger:
Instructor clicks the "Submit for Review" button in course edit page.

Description:
Allows instructors to request system approval to publish draft courses. The system validates the submission immediately, returns a confirmation status, and triggers AI-assisted moderation asynchronously.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The course must exist and belong to the active instructor.

Post-conditions:

POST-1. The course is updated with a pending status in the database.

POST-2. AI moderation runs in the background and saves resolved results to the database upon completion.

POST-3. Stale cache data is cleared.

POST-4. The course is queued on the admin manual audit dashboard.

Normal Flow

A. Submit Course for Review Successfully

  1. Instructor accesses the course editor.
  2. Instructor clicks the "Submit for Review" button.
  3. The system validates the course landing page fields.
  4. The system validates the course curriculum.
  5. The system validates the instructor's rights.
  6. The system validates the course status.
  7. The system validates the course video durations against the business limits.
  8. The system updates the course status to pending, resets lesson and material statuses, and clears previous moderation feedback in the database.
  9. The system invalidates the cached course data.
  10. The system triggers the background AI moderation process:
    • 10a. The system requests moderation from the AI Service.
    • 10b. The system resolves the moderation results from the AI Service.
    • 10c. The system updates the course metadata and priority level in the database.
    • 10d. The system logs the moderation transaction details in the database.
    • 10e. The system sends a notification to the administrator requesting a manual audit.
  11. The system displays a success message.

Alternative Flows

A.3 Invalid Course Landing Page Fields

  1. The system detects one or more landing page fields (what you will learn, requirements, or thumbnail) are invalid.
  2. The system displays a validation error message.
  3. Back to step 2.

A.4 Invalid Course Curriculum

  1. The system detects the course contains no lessons or a lesson is missing required video content.
  2. The system displays a curriculum validation error message.
  3. Back to step 2.

A.7 Video Duration Limits Exceeded

  1. The system detects the total video duration exceeds the allowed limit based on Stripe active status or free course status.
  2. The system displays an error message regarding video duration limits.
  3. Back to step 2.

Exceptions

EX-1.1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-1.2 Course Not Found

  1. The system displays a course not found error message.
  2. The system redirects the user back to the course list page.

EX-5 Invalid Instructor Rights

  1. The system detects the instructor is unauthorized to submit this course or has an active lockout status.
  2. The system displays an error message regarding invalid instructor rights.
  3. Back to step 1.

EX-6 Invalid Course Status

  1. The system detects the course status cannot be modified because it is pending review or permanently archived.
  2. The system displays an error message regarding invalid course status.
  3. Back to step 1.

EX-8 Database Save Failed

  1. The system fails to save the course status changes or reset lesson and material states to the database.
  2. The system displays a database transaction failure message.
  3. Back to step 2.

EX-10a.1 AI Service unhealthy or offline

  1. The system bypasses automated moderation and flags the course directly for admin manual review.
  2. Back to step 10e.

EX-10a.2 AI Moderation request timeout or pipeline error

  1. The system logs the failure and flags the course directly for admin manual review.
  2. Back to step 10e.

EX-10c Database update failure

  1. The system logs the database write error.
  2. Back to step 10e.

EX-10d Database logging failure

  1. The system logs the database write error.
  2. Back to step 10e.

Business Rules

  1. Stripe Link Limits: Instructors who have not fully onboarded with an active Stripe Connect profile are only allowed to publish up to 2 courses and are restricted to a maximum video content threshold of 30 minutes.
  2. Free Course Video Ceiling: Any course with a price parameter of 0 has a strict video content cap of 60 minutes. Paid courses require an active Stripe Connect configuration.
  3. Content Re-activation: When a resubmitted course is set to pending, the system automatically resets any rejected lesson and material statuses back to active for the admin moderator's review.
  4. Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from changing course status or submitting courses for review.

Assumptions

Instructor has stable internet connection.
AI Service is working properly.
