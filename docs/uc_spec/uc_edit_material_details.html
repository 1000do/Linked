UC-EditMaterialDetails - Edit Learning Materials Details

Primary Actor:
Instructor

Secondary Actors:
None

Trigger:
Instructor modifies the details of a video or resource file and clicks outside the input field (onblur) or triggers an auto-save.

Description:
Allows an instructor to edit the details of an existing learning material (video or resource). The system updates the details inline and automatically saves the changes to the database. Modifying these details automatically resets the course status to draft if it was previously published.

Preconditions:

PRE-1. User must be successfully authenticated as an approved instructor.

PRE-2. The material must exist and belong to a lesson in a course owned by the active instructor.

Post-conditions:

POST-1. The material's details are updated in the database.

POST-2. If the course was previously published, its status is reverted to draft and moderation feedback is cleared.

POST-3. Stale cache data is cleared.

Normal Flow

A. Edit Material Details Successfully

  1. Instructor accesses the course edit page.
  2. Instructor modifies the details of a video or resource file in the active lesson editor.
  3. Instructor triggers the save action (by clicking outside the input field, pressing Enter, or waiting for auto-save).
  4. The system validates the instructor's permission.
  5. The system validates the instructor's lockout status.
  6. The system validates the course status.
  7. The system updates the learning material's details in the database.
  8. The system updates the course status to draft and clears moderation feedback in the database.
  9. The system invalidates cached course data.
  10. The system updates the UI to reflect the updated details.

Alternative Flows

A.3 No Changes Made

  1. Instructor triggers the save action.
  2. The system detects the new details are identical to the current values.
  3. The system halts the update process.
  4. Back to step 2.

Exceptions

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-4 Instructor Unauthorized

  1. The system detects the instructor does not own the course.
  2. The system displays an authorization error message.
  3. Back to step 1.

EX-5 Instructor Locked Out

  1. The system detects the instructor is currently locked out.
  2. The system displays a lockout error message.
  3. Back to step 1.

EX-6 Course Pending Review

  1. The system detects the course is currently pending review.
  2. The system displays a restriction error message.
  3. Back to step 1.

EX-7 Database update failed

  1. The system fails to save the material changes.
  2. The system displays a save failure error message.
  3. Back to step 1.

Business Rules

  1. Course Status Reversion: Modifying the details of materials in a previously published course automatically forces the course back to draft status, requiring a re-publish and moderation cycle.

Assumptions

Instructor has stable internet connection.
