UC ID and Name UC-108 - Add AI Model
Created By: An Date Created: 11/06/2026
Primary Actor: Admin Secondary Actors: N/A
Trigger: Admin clicks the "Add AI Model" button on the AI Service management page.
Description: Allows an Admin to register a new AI moderation model configuration in the system by filling out the model registration form. The system validates all required fields before persisting the new model record to the database.
Preconditions:

PRE-1. Admin must be successfully authenticated.
Post-conditions:

POST-1. A new AI model configuration record is persisted to the database.

POST-2. The model list on the UI is refreshed with the added model as its first entry.
Normal Flow:

A. Add AI Model Successfully

  1. Admin accesses the AI Service management page.
  2. The system validates the Admin's access authorization.
  3. Admin clicks the "Add AI Model" button.
  4. The system displays the model registration modal.
  5. Admin fills in the model details.
  6. Admin clicks the "Add Model" button.
  7. The system displays a confirmation dialog.
  8. Admin confirms the action.
  9. The system validates the form inputs.
  10. The system persists the new AI model record to the database.
  11. The system displays a success notification.
  12. The system reloads the page.
Alternative Flows:

A.8 Admin Cancels Confirmation

  1. Admin clicks the cancel button on the confirmation dialog.
  2. Back to step 4.

A.9 Form Validation Failed

  1. The system detects one or more required fields are invalid.
  2. The system displays a validation error message.
  3. Back to step 4.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Unauthorized Access

  1. The system detects the current user does not have admin-level authorization.
  2. The system displays an unauthorized access error message.
  3. The system redirects the user to the login page.

EX-10 Database Save Failed

  1. The system fails to persist the model record to the database.
  2. The system displays an error message regarding the database save failure.
  3. Back to step 4.
Priority: Medium
Frequency of Use: Low
Business Rules: BR-1, BR-25, BR-26, BR-30
Other Information: N/A
Assumptions:

Admin has stable internet connection.
