UC ID and Name UC-11-Go To Learning
Created By: Truong Phuoc Tai Date Created: 12/06/2026
Primary Actor: User, Instructor Secondary Actors: N/A
Trigger: Actor clicks the "Go to learning" button (or "Preview your course" button) on the course details page, or clicks the "Continue learning" button (or "Review" button) on the enrolled courses page.
Description: Opens the course player to watch video lectures, read text content, and view course slides.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must be enrolled in the course, or be the owner (Instructor) of the course.
Post-conditions:

POST-1. Course player page is displayed successfully, rendering the video player or document viewer, the list of lessons, and the overall course progress bar.
Normal Flow:

A. Go To Learning Successfully

  1. Actor accesses the Course Details page or the "My Enrolled Courses" page.
  2. Actor clicks the "Go to learning" button (or "Preview your course", "Continue learning", "Review" buttons).
  3. The system checks the actor's session authentication.
  4. The system validates the actor's enrollment status or course ownership.
  5. The system retrieves the course structure, including lessons, lectures, videos, text content, and downloadable slides.
  6. The system retrieves the actor's learning progress (completed materials list).
  7. The system retrieves the rating average and total reviews count for each lesson section.
  8. The system displays the course player view, showing the media area (with the first or last-viewed video lecture loaded), overall progress bar, and the curriculum list.
Alternative Flows:

A.8.1 Play Video Lecture

  1. Actor clicks on a video-based material in the curriculum sidebar.
  2. The system pauses the current media playback.
  3. The system retrieves the video file source URL.
  4. The system renders the video player.
  5. The system automatically plays the video lecture.
  6. Back to step 8.

A.8.2 Read Text Document

  1. Actor clicks on a text-based material in the curriculum sidebar.
  2. The system pauses the current media playback.
  3. The system retrieves the text file content.
  4. The system renders the text content in the document viewer.
  5. Back to step 8.

A.8.3 Download Course Slides / Resources

  1. Actor clicks on a non-viewable resource or slide in the curriculum sidebar.
  2. The system retrieves the attachment download URL.
  3. The browser triggers the file download.
  4. Back to step 8.

A.8.4 View Course Warning Banner

  1. The system detects the course status is not published.
  2. The system displays a warning banner indicating the course's unpublished state.
  3. Back to step 8.
Exceptions:

EX-3 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-4 Access Denied (Not Enrolled)

  1. The system detects the actor is not enrolled in the course and is not the course owner.
  2. The system redirects the actor to the error page.
Priority: High
Frequency of Use: High
Business Rules: BR-84, BR-10, BR-85, BR-86
Other Information: N/A
Assumptions:

User, Instructor has stable internet connection.
