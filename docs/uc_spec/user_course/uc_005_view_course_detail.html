UC ID and Name UC-03-View Course Details
Created By: Truong Phuoc Tai Date Created: 10/06/2026
Primary Actor: User, Guest, Instructor, Staff, Admin Secondary Actors: N/A
Trigger: Actor clicks on a course from the public course list, category page, or search results.
Description: Allows Guests and Users to view the detailed information of a course, including its description, curriculum, instructor profile, and student reviews. The system determines the visibility of course content and available actions (e.g., "Add to cart", "Enroll Now", "Go to learning") based on the actor's authentication and enrollment status.
Preconditions:

PRE-1. None.
Post-conditions:

POST-1. Detailed course details, including landing section, pricing actions, curriculum accordion, instructor bio, and rating statistics are displayed matching the selected course.
Normal Flow:

A. View Course Details Successfully

  1. Actor accesses the course details page.
  2. The system retrieves the course details.
  3. The system retrieves the active instructor's statistics and biography.
  4. The system retrieves the default course review statistics.
  5. The system retrieves the first page of course reviews.
  6. The system checks the actor's authentication status.
  7. The system retrieves the actor's wishlist status for this course (if authenticated).
  8. The system retrieves the actor's enrollment status for this course (if authenticated).
  9. The system retrieves the actor's review eligibility (if authenticated).
  10. The system displays the course details page, showing the categories, metadata, curriculum, and reviews.
Alternative Flows:

A.2.1 View Unpublished Course

  1. The system detects the course status is not "published".
  2. The system checks if the authenticated actor is the course owner, a manager (admin/staff), or an enrolled user. (If not, trigger EX-3).
  3. The system displays a warning banner indicating the course's unpublished state.
  4. Back to step 4.

A.10.1 Open Preview Modal

  1. Actor clicks on the course thumbnail or play icon.
  2. The system retrieves the first video material of the course.
  3. The system displays the video preview modal and auto-plays the video.
  4. Back to step 10.

A.10.2 Close Preview Modal

  1. Actor clicks the close button ("close" icon) or clicks outside the modal.
  2. The system pauses the video playback and hides the modal.
  3. Back to step 10.

A.10.3 Enroll in Free Course

  1. Actor clicks the "Enroll Now" button.
  2. The system validates that the actor is an authenticated User. (If not, trigger EX-1).
  3. The system processes the free enrollment in the database.
  4. The system displays a success message.
  5. The system updates the details page layout dynamically (unlocks all lectures, replaces "Enroll Now" with "Go to learning" button, enables reporting).
  6. Back to step 10.

A.10.4 Add Course to Cart

  1. Actor clicks the "Add to cart" button.
  2. The system validates that the actor is an authenticated User. (If not, trigger EX-1).
  3. The system adds the course to the user's shopping cart.
  4. The system displays a success message and updates the cart badge.
  5. Back to step 10.

A.10.5 Toggle Wishlist

  1. Actor clicks the "Add to Wishlist" or "Saved to Wishlist" button.
  2. The system validates that the actor is an authenticated User. (If not, trigger EX-1).
  3. The system toggles the course's wishlist status in the database.
  4. The system updates the heart button icon and text dynamically.
  5. Back to step 10.

A.10.6 Expand/Collapse All Curriculum Sections

  1. Actor clicks the "Expand all sections" or "Collapse all sections" button.
  2. The system toggles the visibility of all lesson contents in the curriculum grid.
  3. Back to step 10.

A.10.7 Toggle Individual Lesson Accordion

  1. Actor clicks on a lesson accordion header.
  2. The system toggles the visibility of the clicked lesson's materials.
  3. Back to step 10.

A.10.8 Preview First Lecture

  1. Actor clicks the preview link or "Preview" button on the first lecture of the first lesson.
  2. The system opens the preview modal and plays the first lecture.
  3. Back to step 10.

A.10.9 Access Enrolled Lesson

  1. Actor clicks on any unlocked material link.
  2. The system redirects the actor to the learning page.

A.10.10 Contact Instructor

  1. Actor clicks the "Message Instructor" button.
  2. The system validates that the actor is an authenticated User. (If not, trigger EX-1).
  3. The system creates or retrieves a chat session with the instructor.
  4. The system redirects the actor to the chat interface.

A.10.11 Write and Submit Review

  1. Actor selects a star rating (1 to 5) and enters a comment in the review form.
  2. Actor clicks the "Submit Review" button.
  3. The system validates the input. (If invalid, trigger A.10.11.1).
  4. The system saves the review in the database.
  5. The system updates the course's rating average and reviews count.
  6. The system reloads the reviews list and stats display.
  7. Back to step 10.

A.10.11.1 Review Validation Error

  1. The system detects the rating is not selected or the comment is empty.
  2. The system displays a validation error message.
  3. Back to step 10.

A.10.12 Edit Own Review

  1. Actor clicks the "Edit" link on their own review card.
  2. The system displays the edit review inline form, populated with current rating and comment.
  3. Actor modifies the rating/comment and clicks the "Save" button.
  4. The system saves the changes in the database.
  5. The system updates the review card content and re-calculates statistics.
  6. Back to step 10.

A.10.13 Cancel Review Edit

  1. Actor clicks the "Cancel" button in the edit review form.
  2. The system hides the edit form and restores the original review display.
  3. Back to step 10.

A.10.14 Delete Own Review

  1. Actor clicks the "Delete" link on their own review card.
  2. The system displays a deletion confirmation dialog.
  3. Actor confirms the deletion by clicking "Yes, delete it".
  4. The system soft-deletes the review in the database. (If database reject due to moderation, trigger EX-10.14.1).
  5. The system removes/updates the review on the UI and updates stats.
  6. Back to step 10.

A.10.15 Filter Reviews by Star

  1. Actor selects a star rating option from the rating filter dropdown.
  2. The system retrieves and displays only reviews matching the selected star rating.
  3. Back to step 10.

A.10.16 Report Course

  1. Actor clicks the "Report this course" flag icon.
  2. The system validates that the actor is an authenticated User. (If not, trigger EX-1).
  3. The system displays the report course modal.
  4. Actor selects one or more reasons, writes descriptions (min 20 chars), and clicks the "Submit Report" button.
  5. The system saves the course report in the database.
  6. The system displays a success notification.
  7. Back to step 10.

A.10.17 Report Review

  1. Actor clicks the "Report" button under the three-dot dropdown on another user's review.
  2. The system validates that the actor is an authenticated User. (If not, trigger EX-1).
  3. The system displays the report review modal.
  4. Actor selects a reason, enters a description (optional), and clicks the "Submit Report" button.
  5. The system saves the review report in the database.
  6. The system displays a success notification.
  7. Back to step 10.
Exceptions:

EX-1 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the user to the login page.

EX-2 Course Not Found

  1. The system detects the course does not exist.
  2. The system redirects the actor to the Not Found page.

EX-3 Unauthorized Access (Unpublished Course)

  1. The system detects the course is not published.
  2. The system detects the actor is not the course owner, a system manager, or an enrolled student.
  3. The system redirects the actor to the Error page.

EX-10.14.1 Delete Review Blocked (Under Moderation)

  1. The system detects the review is under moderation.
  2. The system displays a warning message stating the review cannot be deleted.
  3. Back to step 10.
