UC ID and Name UC-13-Enroll courses
Created By: Truong Phuoc Tai Date Created: 12/06/2026
Primary Actor: User, Instructor Secondary Actors: N/A
Trigger: Actor clicks the "Enroll Now" button on the course details page (for free courses), or completes checkout payment (for paid courses).
Description: Grants immediate course access (for free courses) or updates status after payment confirmation (for paid courses).
Preconditions:

PRE-1. Actor must be authenticated.
Post-conditions:

POST-1. Actor's active enrollment record is created and saved in the database.

POST-2. Actor's shopping cart is cleared (if checked out from cart).

POST-3. Course content access is unlocked for the actor.
Normal Flow:

A. Enroll in Free Course Successfully

  1. Actor accesses the Course Details page of a free course.
  2. Actor clicks the "Enroll Now" button.
  3. The system checks the actor's session authentication.
  4. The system validates that the course price is zero.
  5. The system validates that the actor is not the course owner.
  6. The system checks if the actor is already enrolled.
  7. The system creates a new active enrollment record for the actor.
  8. The system displays a success message.
  9. The system dynamically updates the course details page layout (replaces "Enroll Now" with "Go to learning" button and unlocks lectures).

B. Enroll in Paid Course Successfully

  1. Actor accesses the Course Details page of a paid course.
  2. Actor clicks the "Add to cart" button.
  3. The system checks the actor's session authentication.
  4. The system validates that the actor is not the course owner.
  5. The system validates that the actor is not already enrolled.
  6. The system adds the course to the actor's shopping cart.
  7. Actor clicks the "Proceed to Checkout" button in the shopping cart.
  8. The system retrieves a secure checkout secret from the payment gateway.
  9. The system displays the secure payment form.
  10. Actor enters card details and clicks the "Pay" button.
  11. The payment gateway confirms the successful payment.
  12. The system creates the transaction, order, and active enrollment records.
  13. The system clears the actor's shopping cart.
  14. The system redirects the actor to the enrolled courses page with a success message.
Alternative Flows:

B.2.1 Add to Cart from Course Card

  1. From the Home page, Course Catalog page, or Wishlist page, the Actor clicks the "add_shopping_cart" icon on a course card.
  2. The system checks the actor's session authentication.
  3. The system validates that the actor is not the course owner.
  4. The system validates that the actor is not already enrolled.
  5. The system adds the course to the actor's shopping cart.
  6. The system displays a success notification indicating the course has been added.
  7. Back to step 7 of Normal Flow B.
Exceptions:

EX-3 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-5 Own Course Enrollment

  1. The system detects the actor is the course owner (Instructor).
  2. The system displays an error message indicating that instructors cannot enroll in their own courses.
  3. Back to step 1.

EX-6 Already Enrolled

  1. The system detects the actor is already enrolled.
  2. The system displays an error message indicating that the actor is already enrolled.
  3. Back to step 1.
Priority: High
Frequency of Use: High
Business Rules: BR-90, BR-91, BR-92
Other Information: N/A
Assumptions:

User, Instructor has stable internet connection.
