UC ID and Name UC-08-View Review List
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: Guest, User, Instructor Secondary Actors: N/A
Trigger: Actor accesses the Course Details page and scrolls to the reviews section.
Description: Displays the list of public course reviews, star ratings, and statistics, allowing sorting/filtering by star rating and pagination.
Preconditions:

PRE-1. Course details must be accessible.
Post-conditions:

POST-1. Rating statistics and reviews list are displayed successfully.
Normal Flow:

A. View Review List Successfully

  1. Actor accesses the Course Details page of a course.
  2. The system retrieves the overall review statistics.
  3. The system retrieves the first page of course reviews.
  4. The system displays the average rating, star rating icons, total review count, and percentage distribution bars.
  5. The system displays the list of reviews, showing each reviewer's avatar/initial, name, role badge, review date, star rating, course/lesson badge, and comment content.
  6. The system displays the pagination controls containing "Prev" and "Next" buttons.
Alternative Flows:

A.4.1 Filter Reviews by Star Rating

  1. Actor selects a star count (1 to 5 stars) from the "review-star-filter" drop-down selector.
  2. The system retrieves the course reviews filtered by the selected star rating.
  3. The system renders the filtered reviews list starting at page 1.
  4. Back to step 5.

A.4.2 No Reviews Found

  1. The system detects that there are no reviews matching the selected star rating (or no reviews for the course yet).
  2. The system displays a message indicating that no reviews are found.
  3. The flow terminates.

A.6.1 Paginate Reviews

  1. Actor clicks the "Next" (or "Prev") pagination button.
  2. The system retrieves the next (or previous) page of reviews.
  3. The system renders the corresponding page of reviews.
  4. Back to step 5.
Exceptions:

EX-1 System Error Loading Reviews

  1. The system displays an error message indicating that reviews could not be loaded.
  2. The flow terminates.
Priority: High
Frequency of Use: High

Guest, User, Instructor has stable internet connection.
