UC ID and Name UC-16-Delete Personal Reviews
Created By: Truong Phuoc Tai Date Created: 13/06/2026
Primary Actor: User, Instructor Secondary Actors: N/A
Trigger: Actor clicks the "Delete" button on a review they previously submitted.
Description: Allows users to delete the course reviews they have previously submitted.
Preconditions:

PRE-1. Actor must be authenticated.

PRE-2. Actor must be the author of the review.
Post-conditions:

POST-1. The review is soft-deleted in the database.

POST-2. The reviews list and rating statistics are updated dynamically.
Normal Flow:

A. Delete Personal Review Successfully

  1. Actor accesses the Course Details page (or Course Player page) where their review is displayed.
  2. Actor clicks the "Delete" button on their own review card.
  3. The system displays a confirmation popup dialog.
  4. Actor clicks the "Yes, delete it" button.
  5. The system checks the actor's session authentication.
  6. The system checks if the review is currently under moderation review.
  7. The system checks if the actor is the owner of the review.
  8. The system soft-deletes the review in the database.
  9. The system displays a success notification indicating the review has been deleted.
  10. The system dynamically reloads the reviews list and updates rating statistics.
Alternative Flows:

A.4.1 Cancel Deletion

  1. Actor clicks the "Cancel" button on the confirmation popup dialog.
  2. The system closes the popup dialog.
  3. Back to step 1.
Exceptions:

EX-5 Session Invalid (Not Logged In)

  1. The system displays an authentication error message.
  2. The system redirects the actor to the login page.

EX-6 Review Under Moderation

  1. The system detects that the review has an active pending moderation report.
  2. The system displays an alert message indicating that the review is currently under moderation review and cannot be deleted.
  3. The flow terminates.

EX-7 Unauthorized Access

  1. The system detects that the actor is not the author of the review.
  2. The system displays an error message indicating that they can only delete their own reviews.
  3. The flow terminates.
Priority: High
Frequency of Use: Medium
Business Rules: BR-98, BR-99, BR-100, BR-101
Other Information: N/A
Assumptions:

User, Instructor has stable internet connection.
