4.4.6. Reject Course Submission

Function Trigger: Actor clicks the "Reject" button on a course row in the moderation list.

Function Description: Rejects a submitted course with reasons and feedback for the instructor.

Function Details

  • Preconditions
    • Actor must be authenticated.
    • Actor must have staff or admin role.
    • The course status must be pending review (or approved/published if rejecting an active course).
  • Process
    • Reject Course Submission Successfully:
      • Actor accesses the Course Moderation Page.
      • Actor locates the pending course in the moderation list.
      • Actor clicks the "Reject" button on the corresponding course row.
      • The system retrieves the course details (lessons and materials).
      • The system displays a detailed reject modal containing tagged element selectors and reason text fields.
      • Actor clicks the "Add Rejection Item" button to add a new violation tag row.
      • Actor selects a violating element from the dropdown (e.g. course title, course description, course thumbnail, specific lesson, or specific material file).
      • Actor enters a rejection reason for the selected element.
      • Actor clicks the "Submit Rejection" button.
      • The system updates the status of tagged lessons and materials to "rejected" and saves their specific rejection feedback in the database.
      • The system updates the course status to "rejected" and saves the combined moderation feedback in the database.
      • The system sends a rejection notification message to the instructor.
      • The system clears the course details cache.
      • The system updates the moderation page stats and dims the rejected course row.
  • Postconditions
    • The course status is set to "rejected" in the database.
    • Tagged lessons and materials (if any) are set to "rejected" in the database.
    • The instructor receives a notification of the rejection.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Unauthorized Access: The system detects that the actor does not have staff or admin roles. The system displays an access denied page or error message. The flow terminates.
    • .1 Course Metadata Fetch Failed: The system fails to retrieve course details from the API. The system displays an error message indicating failure to load course details. The reject modal shows a connection failure message. The flow terminates.
    • .1 Form Incomplete Validation: The system detects that a tagged element dropdown is unselected or a reason text field is empty. The system displays a warning message indicating information is missing. The system halts submission. Back to step 7.
    • Database Save Failure: The database fails to commit changes. The system displays a system error message. The flow terminates.
  • Security Measures
    • Violation Element Tagging:
      • Course-level targets correspond to: title ("course.title"), description ("course.description"), thumbnail ("course.thumbnail"), what you will learn ("course.what_you_will_learn"), requirements ("course.requirements").
      • Lesson-level targets correspond to "lesson.title".
      • Material-level targets correspond to "file".
    • Combined Feedback: The course's "ModerationFeedback" is updated by combining all tagged element reasons (e.g. "[@Title] Reason" or "[Lesson: Title] Reason" separated by newlines).
    • Notification Link: The rejection notification sent to the instructor contains a link redirecting to the course editor "/InstructorCourse/Editor?id={courseId}".
