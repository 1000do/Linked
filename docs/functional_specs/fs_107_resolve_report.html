UC-107 - Resolve Report

Function Trigger: Staff or Admin clicks the "Moderate" button on a pending, under review, or escalated report row.

Function Description: Resolves a report by taking moderation action (e.g., removing content) and updating status.

Function Details

  • Preconditions
    • The actor must be authenticated.
    • The actor must have Staff or Admin role.
    • The selected report must be in a resolvable state ("Pending", "Under Review", or "Escalated").
  • Process
    • Resolve Report Successfully:
      • Staff or Admin accesses the Report Center page.
      • The system validates the actor's authentication and authorization.
      • Staff or Admin clicks the "Moderate" button on a report row (course report, course review report, or lesson review report).
      • The system displays the Moderate Report popup containing report details, resolution status options, content removal checkbox, and note textarea.
      • The actor selects the "Accept Report (Valid / Policy Violation)" status.
      • The actor enters internal moderation notes.
      • The actor clicks "Submit Decision".
      • The system validates the actor's authentication and authorization.
      • The system processes the moderation decision, updates the report, and applies any penalties or notifications.
      • The system displays a success confirmation message.
      • The system refreshes the report list and overall statistics.
  • Alternative Flow (Select Non-resolved Status)
    • If the user selects this alternative flow:
      • The actor selects a status other than "Accept Report" (e.g., Dismiss Report, Under Review, Escalate).
      • The system automatically unchecks and disables the "Flag Course / Remove content" checkbox.
      • Back to step 6.
  • Alternative Flow (Toggle Content Removal Checkbox)
    • If the user selects this alternative flow:
      • The actor checks the "Flag Course / Remove content" checkbox.
      • The system displays a secondary confirmation popup warning about the consequences of removing or flagging the content.
      • The actor confirms the action.
      • The system updates the checkbox state to checked.
      • Back to step 6.
  • Alternative Flow (Cancel Content Removal)
    • If the user selects this alternative flow:
      • The actor clicks "Cancel" on the secondary confirmation popup.
      • The system keeps the checkbox unchecked.
      • Back to step 6.
  • Postconditions
    • The system updates the report with the selected status and resolution note.
    • If content removal was checked, the system increments the content's violation flag count and applies penalties (e.g., warning notifications, hiding content).
    • If a course accumulates 3 strikes (flags), the system applies permanent course discontinuation and instructor lockouts as defined in the business rules.
    • The system sends relevant notifications to the reporter regarding the report outcome, and to the content owner regarding policy violations or lockouts.
    • If a review is removed, the system soft-deletes the review, marks it as violating, increments the author's account flag count, and applies progressing account lockouts as defined in the business rules.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the actor does not have the Staff or Admin role. The system redirects the actor to the home page or access denied page.
    • Moderation Action Rejected: The server rejects the moderation action (e.g., business rule violation, lack of permissions to perform specific actions). The system displays an error popup with the failure message. Use case ends.
    • Database Save Failure: The system encounters an error while saving the moderation changes to the database. The system displays an error message regarding the data update failure. Use case ends.
  • Security Measures
    • Adaptive Moderation/Detail Popup UI: The moderation and detail popups dynamically adjust titles, fields, labels, and action terminologies (e.g., "Flag Course" vs "Remove content") depending on whether a course or a review is being viewed/moderated.
    • Escalation Restriction: Staff may select the "Escalate to Senior Admin" option. Admins do not see this option, as they are the highest escalation tier.
    • Action Limitations: The "Flag Course / Remove content" checkbox is enabled exclusively when the Resolution Status is set to "Accept Report". Changing the status automatically clears and disables the checkbox.
    • Course Strike System: When a course report is accepted with the "Flag Course" toggle checked: Strike 1–2 sends a warning to the instructor; Strike 3 permanently archives the course and applies a 30-day severe instructor lockout.
    • Review Penalty System: When a review report is accepted with "Remove content" checked: Strike 1 sends a warning; Strike 2 applies a 7-day moderate review lockout; Strike 3 applies a 30-day severe full account ban and terminates active sessions.
    • Implicit Refresh: A successful moderation action automatically triggers a silent refresh of the current report list and overall statistics.
