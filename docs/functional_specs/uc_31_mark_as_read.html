UC-31. Mark As Read

Function Trigger: User or Instructor clicks the "Mark as read" button (check circle icon) next to an unread notification OR clicks on an unread notification to view its details.

Function Description: Allows users and instructors to mark their received notifications as read. The system updates the notification status in the database, notifies active client sessions in real-time, and updates the unread count in the UI.

Function Details

  • Preconditions
    • User or Instructor must be successfully authenticated.
    • User or Instructor has a stable internet connection.
  • Process
    • Mark Single Notification:
      • User or Instructor accesses the notifications page.
      • User or Instructor clicks the "Mark as read" button on a specific unread notification.
      • The system validates the user's session.
      • The system validates the notification ownership.
      • The system updates the notification's read status to 'read' in the database.
      • The system sends a real-time update event to the user's active client connections.
      • The system updates the UI to display the notification as read.
      • The system updates the unread notification count in the navigation bar.
  • Alternative Flow (Click notification to view details)
    • If the user clicks to view details:
      • User or Instructor clicks on the unread notification item.
      • The system displays the notification detail modal.
      • The system proceeds to validate the user's session.
  • Alternative Flow (Mark all notifications as read)
    • If the user marks all as read:
      • User or Instructor clicks the "Mark All as Read" button.
      • The system validates if there are unread notifications.
      • The system validates the user's session.
      • The system updates all unread notifications for the user to read status in the database.
      • The system sends a real-time update event to the user's active client connections.
      • The system updates the UI to display all notifications as read.
      • The system updates the unread notification count in the navigation bar.
      • The system displays a success confirmation message.
  • Alternative Flow (No unread notifications)
    • If there are no unread notifications when marking all:
      • The system displays a message indicating all notifications are already read.
  • Postconditions
    • The selected notification (or all notifications) is updated with a read status in the database.
    • The unread notification count is updated in the UI.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Notification Not Found or Access Denied: The system halts execution and displays a notification not found or access denied error message.
    • Database update failed: The system fails to save the notification read status. The system displays a database transaction failure message.
  • Security Measures
    • Ownership Constraint: A user or instructor can only mark their own received notifications as read.
    • Real-time Synchronization: Marking a notification as read immediately syncs the status across all open tabs/devices of the current user via WebSockets/SignalR and decrements the unread count.
