UC-117. Delete Account (Toggle Ban)

Function Trigger: Admin clicks the "Ban" (or "Delete") button next to an account.

Function Description: Allows an Admin to deactivate (ban) or reactivate a registered account. Banning an account places a severe 100-year lockout suspension on it. Reactivating the account deletes all active lockout entries, immediately restoring access. The Super Admin account is protected from deletion/banning.

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • The target account must exist in the system.
    • Admin has stable internet connection.
  • Process
    • Toggle Ban Successfully:
      • Admin accesses the Account Management page.
      • Admin clicks the Ban (or Toggle Ban) button next to a specific account.
      • The system checks if the Admin is authenticated.
      • The system retrieves the target account details.
      • The system verifies that the target account is not the Super Admin.
      • The system toggles the account status. If the account was active, the system changes the status to banned and creates a 100-year lockout record. If the account was banned, the system changes the status to active and deletes all active lockout records.
      • The system saves the changes to the database.
      • The system displays a success message showing the new status and refreshes the account list.
  • Postconditions
    • The account's status is toggled in the database between active and banned.
    • Lockout records are created (if banned) or deleted (if unbanned) in the database.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
    • Account Not Found: The system detects that the target account ID does not exist in the database. The system displays the error message: "Account not found."
    • Super Admin Ban Blocked: The system detects that the target account is the Super Admin account. The system displays the error message: "Cannot ban the Super Admin account."
    • Database Save Failure: The system detects that saving the status changes failed (rows affected <= 0). The system displays an error message.
  • Security Measures
    • Super Admin Protection: The Super Admin account ("admin" role) is protected and cannot be banned or locked out.
    • Lockout Duration: Banning an account creates a lockout record with a duration of 100 years to effectively disable login permissions permanently.
    • Staff Start Date: For Staff accounts, the account creation timestamp (AccountCreatedAt) serves as the official start date of the staff member.
    • Immediate Recovery: Unbanning an account deletes all active lockout entries immediately, restoring login access.
