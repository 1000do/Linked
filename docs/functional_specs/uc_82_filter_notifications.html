UC-82. Filter Notifications

Function Trigger: Actor clicks a recipient role tab or selects a status option from the filter dropdown on the System Notifications page.

Function Description: Allows the Actor to filter the system notifications log by recipient role and/or read status.

[Placeholder for UI mockup]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • Actor is on the System Notifications page.
  • Process
    • Filter Options Selection:
      • Actor accesses the System Notifications page.
      • Actor clicks a recipient role tab button (such as "Students").
      • Actor selects a status filter option (such as "Unread") from the status dropdown.
    • Filter List:
      • The system filters the notifications list in real time.
      • The system displays the filtered notifications list in the table, preserving any active search keywords or sorting orders.
  • Alternative Flow (Clear Filters)
    • If the Actor resets all filters:
      • Actor clicks the Reset button.
      • The system resets the active role tab to "All Notifications" and the status dropdown to "All Statuses".
      • The system displays the unfiltered notifications list in the table.
  • Postconditions
    • The system displays the notifications in the table filtered according to the selected filter criteria.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to filter system notifications is restricted strictly to authenticated users with Staff or Admin roles.
