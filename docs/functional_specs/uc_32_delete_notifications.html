UC-32. Delete Notifications

Function Trigger: User or Instructor clicks the delete icon next to a notification.

Function Description: Allows users and instructors to permanently delete a notification. The system displays a confirmation dialog before sending the request to the database.

Function Details

  • Preconditions
    • User or Instructor must be successfully authenticated.
    • The notification must exist in the database and belong to the active user.
    • User or Instructor has a stable internet connection.
  • Process
    • Confirm and Delete:
      • User or Instructor clicks the delete icon next to a notification.
      • The system displays a confirmation dialog.
      • User or Instructor clicks the "Delete" button.
      • The system sends a delete request to the backend.
      • The system deletes the notification from the database.
      • The system displays a deletion success notification.
      • The system updates the UI to remove the notification.
  • Alternative Flow (Cancel deletion)
    • If the user cancels the deletion:
      • User or Instructor clicks the "Cancel" button on the confirmation dialog.
      • The system closes the confirmation dialog without making any changes.
      • The process returns to the initial state (list view).
  • Postconditions
    • The notification is permanently deleted from the database.
    • The notification is removed from the UI.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Delete request failed or access denied: The system displays an error message regarding missing notification or permission.
    • Database update failed: The system fails to delete the notification from the database and displays a database transaction failure message.
  • Security Measures
    • Confirmation Prompt: Deleting a notification requires confirmation via a dialog to prevent accidental deletion.
    • Ownership Validation: The backend must rigorously verify that the authenticated user is the legitimate owner of the notification being deleted to prevent unauthorized data removal.
