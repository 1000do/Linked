4.10.22. View list of removed learning materials

Function Trigger: Instructor clicks the "Trash" button (or similar) on the Instructor Studio dashboard.

Function Description: Displays a list of deleted learning materials (instructor's trash bin).

Function Details

  • Preconditions
    • Instructor must be authenticated.
    • Instructor's status must be "Approved".
  • Process
    • View Removed Learning Materials Successfully:
      • Instructor accesses the Instructor Studio dashboard page.
      • The system checks the instructor's session authentication.
      • The system checks the instructor's approval status.
      • Instructor clicks the "Trash" button.
      • The system checks the instructor's session authentication.
      • The system retrieves the list of soft-deleted learning materials for the instructor.
      • The system calculates the remaining days before auto-deletion for each material.
      • The system displays the trash bin page, showing the list of removed learning materials, their title, source lesson title, source course title, deleted timestamp, and remaining days.
  • Postconditions
    • The system displays a list of soft-deleted learning materials belonging to the instructor's courses.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Account Not Approved: The system detects the actor's instructor approval status is not "Approved". The system redirects the actor to the application status or apply page. The flow terminates.
    • .1 Restore/Delete Rights Locked: The system detects that the instructor's account rights are locked due to policy violations. The system displays an error message indicating their rights are locked and changes cannot be saved. The flow terminates.
    • .2 Course Pending Review: The system detects the course containing the lesson is currently pending review. The system displays an error message indicating curriculum changes are blocked during review. The flow terminates.
    • .3 Course Permanently Discontinued: The system detects the course is permanently discontinued due to policy violations (3 or more flags). The system displays an error message indicating modifications are blocked. The flow terminates.
    • .4 Unauthorized Access: The system detects that the instructor does not own the course/material. The system displays an error message indicating permission denied. The flow terminates.
    • .5 Database Save Failure: The database fails to commit changes. The system displays a system error message. The flow terminates.
  • Security Measures
    • Auto-Purge Policy: Learning materials placed in the trash bin are automatically and permanently deleted after 30 days by a background cleanup service.
    • Cloud Storage Virtual Directory: Moving a file to the trash bin renames the file prefix on Cloudinary to `trash/` and updates its status to "removed" in the database, hiding it from the public display while retaining the database record.
    • Video Swap Constraint: Restoring a video material to a lesson that already contains an active video will automatically move the current active video to the trash bin, replacing it with the restored video.
    • Discontinued & Pending Locking: Learning materials cannot be restored or permanently deleted if the parent course is pending review or is permanently discontinued due to policy violations.
    • Unpublish Trigger: Restoring a learning material to a published course will automatically update the course status back to "Draft" to force re-moderation of the updated curriculum.
