102. View List of User Reports

Function Trigger: Actor navigates to the Report Center page.

Function Description: Displays reported courses, reported course reviews, and reported lesson reviews submitted by users. All category reports are accessible via tabs and subtabs; Actor can switch between category subtabs without a page reload.

Function Details

  • Preconditions
    • The actor must be authenticated.
    • The actor must have Staff or Admin role.
  • Process
    • View List of User Reports Successfully:
      • Actor accesses the Report Center page.
      • The system validates the actor's authentication and authorization.
      • The system retrieves the overall report statistics and the paginated lists of pending course reports, course review reports, and lesson review reports.
      • The system displays the overall report statistics and main navigation tabs.
      • The system displays the list of pending course reports by default.
      • Actor views the list of user reports.
  • Alternative Flow (Empty Report List)
    • The system detects no reports exist for one or more report categories.
    • The system displays an empty state message in the affected category's report list.
    • Back to step 6.
  • Alternative Flow (Actor Switches Category Subtab)
    • Actor clicks a different category tab or subtab.
    • The system displays the report list for the selected category.
    • Back to step 6.
  • Alternative Flow (Actor Navigates to Another Page)
    • Actor clicks a pagination button.
    • The system reloads the page with the requested page of reports for the active category.
    • Back to step 1.
  • Postconditions
    • The system displays the paginated list of reports for the selected category.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Unauthorized Access: The system detects the actor does not have the Staff or Admin role and redirects the actor to the home page or access denied page.
    • Database Query Failure: The system encounters an error while retrieving the reports from the database, displays an error message regarding the data retrieval failure, and returns to step 1.
  • Security Measures
    • The system enforces role-based access control, allowing only Staff or Admin to view user reports.
