4.10.10. Enroll courses

Function Trigger: Actor clicks the "Enroll Now" button on the course details page (for free courses), or completes checkout payment (for paid courses).

Function Description: Grants immediate course access (for free courses) or updates status after payment confirmation (for paid courses).

Function Details

  • Preconditions
    • Actor must be authenticated.
  • Process
    • Enroll in Free Course Successfully:
      • Actor accesses the Course Details page of a free course.
      • Actor clicks the "Enroll Now" button.
      • The system checks the actor's session authentication.
      • The system validates that the course price is zero.
      • The system validates that the actor is not the course owner.
      • The system checks if the actor is already enrolled.
      • The system creates a new active enrollment record for the actor.
      • The system displays a success message.
      • The system dynamically updates the course details page layout (replaces "Enroll Now" with "Go to learning" button and unlocks lectures).
      • Actor accesses the Course Details page of a paid course.
      • Actor clicks the "Add to cart" button.
      • The system checks the actor's session authentication.
      • The system validates that the actor is not the course owner.
      • The system validates that the actor is not already enrolled.
      • The system adds the course to the actor's shopping cart.
      • Actor clicks the "Proceed to Checkout" button in the shopping cart.
      • The system retrieves a secure checkout secret from the payment gateway.
      • The system displays the secure payment form.
      • Actor enters card details and clicks the "Pay" button.
      • The payment gateway confirms the successful payment.
      • The system creates the transaction, order, and active enrollment records.
      • The system clears the actor's shopping cart.
      • The system redirects the actor to the enrolled courses page with a success message.
  • Postconditions
    • Actor's active enrollment record is created and saved in the database.
    • Actor's shopping cart is cleared (if checked out from cart).
    • Course content access is unlocked for the actor.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Own Course Enrollment: The system detects the actor is the course owner (Instructor). The system displays an error message indicating that instructors cannot enroll in their own courses. Back to step 1.
    • Already Enrolled: The system detects the actor is already enrolled. The system displays an error message indicating that the actor is already enrolled. Back to step 1.
  • Security Measures
    • No Self-Enrollment: Instructors are blocked from enrolling in their own courses (both free and paid). They bypass enrollment checks when accessing the player for their own courses.
    • Double Purchase Prevention: The system blocks purchase/checkout of courses that the user is already enrolled in, prompting them to remove it from their cart.
    • Stripe Integration: Paid course enrollment is completed asynchronously or upon return redirect validation after Stripe gateway processes the successful payment.
