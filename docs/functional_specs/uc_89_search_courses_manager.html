4.4.2. Search Courses

Function Trigger: Actor enters search keywords in the "search" input box on the Course Moderation Page.

Function Description: Searches course list by title or instructor name for moderation purposes.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be authenticated.
    • Actor must have staff or admin role.
    • Actor must be on the Course Moderation Page.
  • Process
    • Search Courses Successfully:
      • Actor enters search keywords in the "search" input box.
      • The system retrieves the courses list filtered by the search keywords and other active filters.
      • The system updates the course moderation table and statistics cards.
      • The system displays the filtered course list.
  • Postconditions
    • The system displays the course moderation list matching the search keywords and any active filters.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Unauthorized Access: The system detects that the actor does not have staff or admin roles. The system displays an access denied page or error message. The flow terminates.
    • API Call Failed: The system fails to retrieve the filtered course list or stats from the API. The system displays an error message regarding search failure. The system renders the Course Moderation page with an empty list. The flow terminates.
  • Security Measures
    • Exclusion of Drafts: Courses with status "Draft" are never retrieved or displayed in the moderation queues.
    • Search Criteria: Search keywords are matched (case-insensitively, substring match) against:
      • Course Title
      • Instructor Name (Full Name)
    • Preservation of Filters: Searching preserves all other active filters (Category, Status, Sort Option).
    • Debounce Time: To prevent excessive server requests, the search execution is debounced by 300 milliseconds from the last keystroke.
