2.2.3.130. View transaction list (For System)

Function Trigger: Administrator accesses the Platform Finance Dashboard page (or clicks the "Transactions" tab within the dashboard).

Function Description: Allows administrators to track all course purchase transactions across the entire platform. Provides search, filtering by status, sorting, and direct links to view the details of each transaction.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • - The administrator must be successfully logged into the system.
    • - The administrator account must have platform-wide financial management permissions.
  • Process
    • View Transaction History Successfully:
      • 1. Administrator accesses the Admin Finance dashboard page.
      • 2. The system retrieves the list of course transactions from the database.
      • 3. The system displays the Transactions tab showing the transaction table with the following columns:
        • - ID: Sequential transaction ID prefixed with #.
        • - Stripe Tx ID: The secure Stripe transaction reference ID.
        • - Course: The title of the purchased course.
        • - Purchased By: The name of the student who made the purchase.
        • - Instructor: The name of the instructor teaching the course.
        • - Amount: The purchase price.
        • - Date: The transaction date and time (formatted as dd/MM/yyyy HH:mm).
        • - Status: The colored badge representing the transaction state (Succeeded, Failed, Pending, Refund Pending, Refunded).
      • 4. Administrator reviews the list of transactions.
  • Postconditions
    • - The administrator views the platform-wide transaction history list including the Stripe Tx ID, course title, buyer, instructor, amount, date, and status.
  • Error Handling
    • - Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
  • Security Measures
    • - Negative Payout Display for Refunded Status: If a transaction has a status of "refunded", the Amount column must display as a negative value prefixed with a minus - sign in red (e.g., - USD 19.99).
    • - Multi-Criteria Search: The search bar matches the keyword against course title, purchaser name, and instructor name in a case-insensitive manner.
