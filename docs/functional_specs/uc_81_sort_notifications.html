UC-81. Sort Notifications

Function Trigger: Actor selects a sorting option from the sort selector on the System Notifications page.

Function Description: Allows the Actor to sort the system notifications log by timestamp, title, or recipient name.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • Actor is on the System Notifications page.
  • Process
    • Select Sorting Option:
      • Actor accesses the System Notifications page.
      • Actor selects a sorting option (such as "Title (A - Z)") from the sort dropdown list.
    • Sort List:
      • The system sorts the notification list in real time as the selection changes.
      • The system displays the sorted notifications list in the table, preserving any active role tab filters, status filters, or search keywords.
  • Alternative Flow (Clear Sorting)
    • If the Actor resets the sorting selector:
      • Actor clicks the Reset button.
      • The system resets the sort selector to its default ("Newest First").
      • The system displays the sorted notifications list in the table.
  • Postconditions
    • The system displays the notifications in the table sorted according to the selected sorting option.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to sort system notifications is restricted strictly to authenticated users with Staff or Admin roles.
