UC-AddLesson. Add Lesson

Function Trigger: Instructor clicks the "Create New Lesson" button or "Add Lesson" icon on the course syllabus.

Function Description: Allows an instructor to add a new lesson to an existing course. The system enforces business rules such as lesson limits, uniqueness of lesson titles, and restricts modifications if the course is currently pending review or the instructor is locked out.

Function Details

  • Preconditions
    • User must be successfully authenticated as an approved instructor.
    • The course must exist and belong to the active instructor.
  • Process
    • Add New Lesson Successfully:
      • Instructor accesses the course editor.
      • Instructor clicks the "Create New Lesson" button or "Add Lesson" icon.
      • The system displays the "Add New Lesson" modal.
      • Instructor enters the lesson title in the "Lesson Title" input field.
      • Instructor clicks the "Create Lesson" button.
      • The system validates the instructor's rights.
      • The system validates the course status.
      • The system validates the maximum lesson limits.
      • The system validates the lesson title for uniqueness.
      • The system creates the new lesson, updates the course status to draft, and clears previous moderation feedback in the database.
      • The system invalidates the cached course data.
      • The system adds the newly created lesson to the curriculum list.
      • The system closes the "Add New Lesson" modal.
  • Alternative Flow (Course is not published)
    • At step 10, if the course is not published:
      • The system skips updating the course status and moderation feedback.
      • Back to step 11.
  • Postconditions
    • A new lesson is added to the course curriculum.
    • If the course was previously published, its status is reverted to draft and moderation feedback is cleared.
    • Stale cache data is cleared.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Course Not Found: The system displays a course not found error message and redirects the user back to the course list page.
    • Invalid Instructor Rights: The system detects the instructor is unauthorized to add a lesson to this course or has an active lockout status. The system displays an error message regarding invalid instructor rights and returns to step 3.
    • Invalid Course Status: The system detects the course is pending review or is permanently archived due to policy violations. The system displays an error message regarding invalid course status and returns to step 3.
    • Max Lessons Reached: The system detects the course already contains 5 lessons and the instructor has not linked an active Stripe Connect profile. The system displays an error message regarding maximum lesson limits and returns to step 3.
    • Duplicate Lesson Title: The system detects a lesson with the same title already exists in the course (case-insensitive). The system displays an error message regarding duplicate lesson titles and returns to step 3.
    • Database Save Failed: The system fails to save the lesson changes to the database. The system displays a database transaction failure message and returns to step 3.
  • Security Measures
    • Instructors who have not linked an active Stripe Connect profile are only allowed to create up to 5 lessons per course.
    • Lesson titles must be unique within a course (case-insensitive).
    • Lessons cannot be added to a course while it is currently in 'Pending' review status.
    • Adding a lesson to a 'Published' course automatically changes its status back to 'Draft' and clears any existing moderation feedback.
    • Instructors with an active lockout status are prohibited from creating lessons.
