UC-113 - Set Confidence Threshold

Function Trigger: Admin modifies one or more threshold values and clicks the "Save Changes" button in the threshold configuration form.

Function Description: Allows an Admin to configure the spam confidence, toxicity confidence, and similarity score thresholds used by the automated AI moderation system. The current threshold values are pre-filled from the database on page load. If no configuration exists, system defaults are applied.

Function Details

  • Preconditions
    • Admin must be successfully authenticated.
  • Process
    • Set Confidence Threshold Successfully:
      • Admin accesses the AI Service management page.
      • The system validates the Admin's access authorization.
      • The system retrieves and displays the current threshold configuration values in the configuration form.
      • Admin modifies one or more threshold values in the configuration form.
      • Admin clicks the "Save Changes" button.
      • The system displays a confirmation dialog.
      • Admin confirms the action.
      • The system validates the form inputs.
      • The system updates the threshold configuration in the database.
      • The system displays a success notification.
  • Alternative Flow (No Threshold Configuration Found)
    • If the user selects this alternative flow:
      • The system detects no threshold configuration exists in the database.
      • The system populates the form with system default threshold values.
      • Back to step 4.
  • Alternative Flow (Admin Cancels Confirmation)
    • If the user selects this alternative flow:
      • Admin clicks the cancel button on the confirmation dialog.
      • Back to step 4.
  • Alternative Flow (Form Validation Failed)
    • If the user selects this alternative flow:
      • The system detects one or more submitted values fail validation constraints.
      • The system displays a validation error message.
      • Back to step 4.
  • Postconditions
    • The confidence threshold configuration is updated in the database.
    • The updated threshold values are displayed in the configuration form in place of the old values.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the current user does not have Admin authorization. The system displays an unauthorized access error message. The system redirects the user to the login page.
    • Database Save Failed: The system fails to persist the threshold configuration to the database. The system displays an error message regarding the database save failure. Back to step 4.
  • Security Measures
    • Universal Form Data Validation: All submitted form fields must pass strict data type, format, length, and value constraints (e.g., report descriptions must be at least 20 characters, and confidence thresholds must be decimals between 0.01 and 1.0) before the system processes the submission.
    • Admin Authorization for AI Services: Only authenticated Admin actors are authorized to access and perform actions within the AI Service and Model Management module (including adding/updating models, toggling status, viewing details/list, configuring thresholds, and viewing activity logs).
