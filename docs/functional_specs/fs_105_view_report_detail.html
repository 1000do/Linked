UC-105 - View Report Detail

Function Trigger: Staff or Admin clicks on a specific report row in the Report Center (applies to course reports, course review reports, and lesson review reports).

Function Description: Displays details and evidence of a reported violation.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • The actor must be authenticated.
    • The actor must have Staff or Admin role.
  • Process
    • View Report Detail Successfully:
      • Staff or Admin accesses the Report Center page.
      • The system validates the actor's authentication and authorization.
      • Staff or Admin clicks on a report row (course report, course review report, or lesson review report) in the report list.
      • The system retrieves the selected report details.
      • The system determines the report status does not contain resolution details.
      • The system displays a popup containing the report details (Reporter, Submitted At, Status, Reported Content, Reason, and Description).
  • Alternative Flow (View Resolved/Rejected Report Detail)
    • If the user selects this alternative flow:
      • The system determines the report status contains resolution details (e.g., resolved, rejected).
      • The system appends the Resolution Details (Moderator, Resolved At, Resolution Note) to the report details.
      • Back to step 6.
  • Postconditions
    • The system displays a popup containing the detailed information of the selected report.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the actor does not have the Staff or Admin role. The system redirects the actor to the home page or access denied page.
    • Database Query Failure: The system encounters an error while retrieving the report details from the database. The system displays an error message regarding the data retrieval failure. Use case ends.
  • Security Measures
    • Adaptive Moderation/Detail Popup UI: The moderation and detail popups dynamically adjust titles, fields, labels, and action terminologies (e.g., "Flag Course" vs "Remove content") depending on whether a course or a review is being viewed/moderated.
    • Resolution Visibility: Resolution details (Moderator, Resolved At, Resolution Note) are visible only for reports with a resolved or rejected status.
