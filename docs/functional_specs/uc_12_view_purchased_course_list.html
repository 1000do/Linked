4.10.9. View purchased course list

Function Trigger: Actor accesses the enrolled courses page.

Function Description: Displays all courses successfully purchased or enrolled in by the user.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be authenticated.
  • Process
    • View Purchased Course List Successfully:
      • Actor accesses the enrolled courses page.
      • The system checks the actor's session authentication.
      • The system retrieves the list of courses the actor is enrolled in.
      • For each course, the system retrieves the total material count and the completed material count.
      • For each course, the system calculates the progress percentage.
      • The system displays the enrolled courses page showing the course title, instructor, thumbnail, progress percentage, completion status, and action buttons.
  • Postconditions
    • Detailed list of enrolled courses is rendered successfully showing the thumbnail, title, instructor name, progress bar, completion status, and learning actions.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
  • Security Measures
    • Completed Badge Display: A completed course card displays a distinct completed overlay badge with the text "Completed" and a "check_circle" icon.
    • Progress Bar Color Coding: The progress bar is colored emerald green if the course is 100% completed, and the primary color (teal) otherwise.
    • Dynamic Actions based on Completion: The action button text is set to "Review" with a "replay" icon for completed courses, and "Continue learning" with a "play_arrow" icon for incomplete courses.
