4.10.11. Create Courses

Function Trigger: Instructor clicks the "Create New Course" button (or similar) on the Instructor Studio dashboard.

Function Description: Drafts basic details for a new course, including title, pricing, description, and curriculum outline.

Function Details

  • Preconditions
    • Instructor must be authenticated.
    • Instructor's application status must be "Approved".
  • Process
    • Create Course Successfully:
      • Instructor clicks the "Create New Course" button on the Instructor Studio dashboard.
      • The system checks the instructor's session authentication.
      • The system checks the instructor's approval status.
      • The system retrieves the available course categories.
      • The system displays Step 1 (Core Identity) of the creation wizard, showing the "Course Title" input and "Category" dropdown.
      • Instructor enters a title, selects a category, and clicks the "Next" button.
      • The system validates the inputs for Step 1.
      • The system displays Step 2 (Description & Curriculum) of the creation wizard, showing the rich text editors for Course Description, What You Will Learn, and Requirements.
      • Instructor enters the description, learning achievements, and requirements, then clicks the "Next" button.
      • The system validates the rich text fields.
      • The system displays Step 3 (Course Thumbnail) of the creation wizard, showing the "drop-area" file upload zone.
      • Instructor uploads a course thumbnail image.
      • The system uploads the image to the cloud storage service.
      • The system retrieves the returned image URL.
      • The system stores the URL in the "CourseThumbnailUrl" variable.
      • Instructor clicks the "Next" button.
      • The system validates that the thumbnail URL is present.
      • The system displays Step 4 (Access & Pricing) of the creation wizard, showing the "Price" input and "Apply coupon to this course" switch.
      • Instructor enters the price and toggles the coupon switch.
      • Instructor clicks the "Select System Voucher" button.
      • The system displays the platform coupons list in the "coupon-modal".
      • Instructor selects a platform coupon and clicks the "Create Course" button.
      • The system validates Step 4 price inputs.
      • The system sends the course creation payload to the backend server.
      • The backend server validates the request model state.
      • The backend server checks the instructor's creation rights.
      • The backend server checks for duplicate course content.
      • The backend server inserts the draft course record into the database.
      • The backend server saves the changes.
      • The system displays a success confirmation popup dialog.
      • The system redirects the instructor to the Course Editor page.
  • Postconditions
    • The draft course record is created in the database with status set to "Draft".
    • The system redirects the Instructor to the Course Editor page.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Account Not Approved: The system detects the actor's instructor approval status is not "Approved". The system redirects the actor to the application status or apply page. The flow terminates.
    • Backend Connection Failure / System Error: The backend server returns a connection failure or server error. The system displays a system error alert box detailing the failure. The flow terminates.
    • Invalid Backend Request State: The backend server returns a bad request status code (400). The system displays a validation warning alert box with the validation details. Back to step 22.
    • Unauthorized Instructor Rights: The backend server returns a bad request status code (400) due to invalid instructor rights. The system displays an error message indicating unauthorized creation. The flow terminates.
    • Duplicate Course Content: The database throws a uniqueness/duplicate content exception. The backend server handles the database error and returns a bad request status code (400) with a duplicate content warning. The system displays a system error alert box indicating duplicate course content is detected. Back to step 6.
    • Database Insertion Failure: The backend server throws an invalid operation exception. The backend server returns a bad request status code (400). The system displays a system error alert box detailing the insertion failure. The flow terminates.
  • Security Measures
    • Title Length Constraints: The course title must be between 5 and 100 characters in length.
    • Text Constraints: What You Will Learn and Requirements descriptions must each be at least 20 characters in length (excluding HTML tags).
    • Stripe Payout Connectivity: Instructors must connect and activate their Stripe accounts to set premium prices. If Stripe is disconnected or inactive, the course price is locked to 0.00 (Free).
    • Voucher Validation: Platform sponsored discount vouchers cannot be applied to free courses.
    • Draft Initial State: All newly created courses are saved with a status of "Draft" and must go through moderation before being published.
