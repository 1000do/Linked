UC-100. Filter Instructor Applications

Function Trigger: Actor selects a specialization option from the filter dropdown.

Function Description: Allows the Actor to filter the pending instructor applications list by applicant expertise or specialization.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • The instructor applications list is loaded on the page.
  • Process
    • Load Filter Options:
      • Actor accesses the instructor registration requests page.
      • The system dynamically populates the filter dropdown with unique expertise categories from the loaded pending applications.
    • Select Filter:
      • Actor selects an expertise specialization option from the dropdown menu.
      • The system filters the pending applications list to display only those containing the selected expertise specialization.
      • The system updates the table rows to show only matching applicants.
  • Alternative Flow (Select All Specializations)
    • If the Actor resets the filter:
      • Actor selects the "All Specializations" option from the dropdown.
      • The system resets the expertise filter and displays all applications.
  • Alternative Flow (No Matching Applications)
    • If filtering results in no matches:
      • Actor combines the expertise filter with a search query that has no match.
      • The system displays an empty state message indicating no matching registration requests were found.
  • Postconditions
    • The system displays only those applications that match the selected specialization.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to filter instructor applications is restricted strictly to authenticated users with Staff or Admin roles.
    • Data Integrity: The filter options are derived dynamically from existing applications to ensure consistency.
