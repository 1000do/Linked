4.4.1. View Course Moderation List

Function Trigger: Actor accesses the Course Moderation Page.

Function Description: Displays all courses submitted by instructors that are pending review.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be authenticated.
    • Actor must have staff or admin role.
  • Process
    • View Course Moderation List Successfully:
      • Actor accesses the Course Moderation Page.
      • The system retrieves all categories.
      • The system retrieves the pending/under-review courses list matching current filters, sorting, and pagination parameters.
      • The system calculates the urgency level and urgency color for each pending course.
      • The system retrieves the course moderation statistics.
      • The system displays the moderation page containing stats cards, filters, and the course moderation table.
  • Postconditions
    • The system displays the pending courses list and moderation stats.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Unauthorized Access: The system detects that the actor does not have staff or admin roles. The system displays an access denied page or error message. The flow terminates.
    • API Call Failed: The system fails to retrieve categories, pending courses list, or stats from the API. The system displays an error message regarding API call failure. The system renders the Course Moderation page with an empty list and stats reset to zero. The flow terminates.
  • Security Measures
    • Exclusion of Drafts: Courses with status "Draft" are never retrieved or displayed in the moderation queues.
    • Urgency Level Calculation: For pending courses, the urgency color is calculated based on creation date:
      • Red / "High" urgency: pending > 72 hours.
      • Amber / "Medium" urgency: pending > 24 hours.
      • Slate / "Normal" urgency: pending <= 24 hours.
    • Threat Levels: Threat levels correspond to AI check feedback:
      • Critical Threat (FlaggedOrRejected): course hits critical policy violations.
      • High Threat (ManualAudit): course needs manual audit.
      • Clean (Approved): course is approved by automated check.
      • Pending / None: default.
    • Infraction Flags: A course tracks up to 3 flags. If flag count reaches 3, status is changed to "Archived" and course status maps to "Permanently Locked".
