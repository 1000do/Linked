UC-54. Publish courses

Function Trigger: Instructor clicks the "Submit for Review" button in course edit page.

Function Description: Allows instructors to request system approval to publish draft courses. The system validates the submission immediately, returns a confirmation status, and triggers AI-assisted moderation asynchronously.

Function Details

  • Preconditions
    • User must be successfully authenticated as an approved instructor.
    • The course must exist and belong to the active instructor.
  • Process
    • Submit Course for Review Successfully:
      • Instructor accesses the course editor.
      • Instructor clicks the "Submit for Review" button.
      • The system validates the course landing page fields.
      • The system validates the course curriculum.
      • The system validates the instructor's rights.
      • The system validates the course status.
      • The system validates the course video durations against the business limits.
      • The system updates the course status to pending, resets lesson and material statuses, and clears previous moderation feedback in the database.
      • The system invalidates the cached course data.
      • The system triggers the background AI moderation process:
        • 10a. The system requests moderation from the AI Service.
        • 10b. The system resolves the moderation results from the AI Service.
        • 10c. The system updates the course metadata and priority level in the database.
        • 10d. The system logs the moderation transaction details in the database.
        • 10e. The system sends a notification to the administrator requesting a manual audit.
      • The system displays a success message.
  • Alternative Flow (Invalid Course Landing Page Fields)
    • If the user selects this alternative flow:
      • The system detects one or more landing page fields (what you will learn, requirements, or thumbnail) are invalid.
      • The system displays a validation error message.
      • Back to step 2.
  • Alternative Flow (Invalid Course Curriculum)
    • If the user selects this alternative flow:
      • The system detects the course contains no lessons or a lesson is missing required video content.
      • The system displays a curriculum validation error message.
      • Back to step 2.
  • Alternative Flow (Video Duration Limits Exceeded)
    • If the user selects this alternative flow:
      • The system detects the total video duration exceeds the allowed limit based on Stripe active status or free course status.
      • The system displays an error message regarding video duration limits.
      • Back to step 2.
  • Postconditions
    • The course is updated with a pending status in the database.
    • AI moderation runs in the background and saves resolved results to the database upon completion.
    • Stale cache data is cleared.
    • The course is queued on the admin manual audit dashboard.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Course Not Found: The system displays a course not found error message. The system redirects the user back to the course list page.
    • Invalid Instructor Rights: The system detects the instructor is unauthorized to submit this course or has an active lockout status. The system displays an error message regarding invalid instructor rights. Back to step 1.
    • Invalid Course Status: The system detects the course status cannot be modified because it is pending review or permanently archived. The system displays an error message regarding invalid course status. Back to step 1.
    • Database Save Failed: The system fails to save the course status changes or reset lesson and material states to the database. The system displays a database transaction failure message. Back to step 2.
    • AI Service unhealthy or offline: The system bypasses automated moderation and flags the course directly for admin manual review. Back to step 10e.
    • AI Moderation request timeout or pipeline error: The system logs the failure and flags the course directly for admin manual review. Back to step 10e.
    • Database update failure: The system logs the database write error. Back to step 10e.
    • Database logging failure: The system logs the database write error. Back to step 10e.
  • Security Measures
    • Stripe Link Limits: Instructors without an active Stripe Connect profile may publish up to 2 courses with a maximum of 30 minutes of video content.
    • Free Course Video Ceiling: Courses priced at 0 are capped at 60 minutes of video content. Paid courses require an active Stripe Connect configuration.
    • Content Re-activation: On resubmission, the system automatically resets any previously rejected lesson and material statuses back to active for admin review.
    • Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from submitting courses for review or changing course status.
