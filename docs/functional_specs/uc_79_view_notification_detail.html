UC-79. View Notification Detail

Function Trigger: Actor clicks on a notification row in the System Notifications table.

Function Description: Allows the Actor to view the detailed recipient, title, content, creation timestamp, and read status of a notification. If a Staff member views an unread notification addressed to themselves, the system automatically marks it as read.

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • The target notification must exist.
  • Process
    • Open Modal:
      • Actor accesses the System Notifications page.
      • Actor locates the desired notification in the list.
      • Actor clicks the notification row.
      • The system displays the notification detail modal, showing the recipient name and email, message title, full message details, creation timestamp, and read status.
    • Close Modal:
      • Actor clicks the "Close" button.
      • The system closes the detail modal.
  • Alternative Flow (Staff Views Own Unread Notification)
    • If the Actor views their own unread notification:
      • The system detects that the viewer is a Staff member, the notification recipient matches the viewer's user ID, and the notification is currently unread.
      • The system requests the backend to mark the notification as read.
      • The server validates that the active user is the recipient of the notification.
      • The server updates the notification's status to read in the database.
      • The system updates the detail status badge to Read and refreshes the background table view dynamically.
  • Postconditions
    • The system displays the notification detail modal.
    • If the viewer is the recipient and the notification was unread, the read status is updated to read in the database and updated in the UI.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to view notification details is restricted strictly to authenticated users with Staff or Admin roles.
    • Recipient Ownership Verification: The server validates that the active user is the recipient of the notification before updating its status to read.
