UC-116. Update Account

Function Trigger: Admin clicks the "Edit" button next to a staff account, modifies details, and saves.

Function Description: Allows an Admin to update profile details, change status, and reset the password of an existing Staff account. The system validates entries and manages lockout records if the account status is updated to or from banned.

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • The target account must exist and belong to a Staff member.
    • Admin has stable internet connection.
  • Process
    • Update Account Successfully:
      • Admin accesses the Account Management page.
      • Admin clicks the Edit button on a specific staff account.
      • The system displays the edit form populated with current staff details.
      • Admin modifies the staff details.
      • Admin clicks the Save button.
      • The system checks that the Display Name field is provided.
      • The system retrieves the target account and verifies it is a staff account.
      • The system updates the staff profile fields.
      • The system hashes the new password using BCrypt (if a new password is provided).
      • The system adjusts the account status and creates or deletes lockout records in the database based on the status change.
      • The system saves all changes to the database.
      • The system displays a success message and redirects to the account list.
  • Postconditions
    • The updated account profile details are saved to the database.
    • Lockout records are updated in the database if the status is changed to or from banned.
    • The system displays a success message and redirects to the account list.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
    • Mandatory Fields Missing: The system detects that the Display Name is empty. The system displays a validation error message.
    • Staff Account Not Found: The system detects that the account does not exist or does not have a staff role. The system displays the error message: "Staff account not found."
    • Database Save Failure: The system detects that saving the updated staff details to the database failed. The system displays an error message.
  • Security Measures
    • Role Scope Restriction: This update endpoint is restricted solely to accounts carrying the "staff" role.
    • Optional Password Reset: If the password input is empty, the current password remains unchanged. If a new password is provided, it is hashed using BCrypt and overrides the old hash.
    • Ban/Lockout Logic:
      • Banning: Transitioning the status to "Banned" automatically adds a 100-year lockout record.
      • Unbanning: Transitioning from "Banned" to active/etc. automatically deletes all active lockout entries to restore login access.
