UC-106 - Filter Reports

Function Trigger: Staff or Admin selects a filter criteria (report type tab, subtab, or status dropdown) in the Report Center.

Function Description: Filters reports by type (Course, Review, Lesson) or status.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • The actor must be authenticated.
    • The actor must have Staff or Admin role.
  • Process
    • Filter Reports Successfully:
      • Staff or Admin accesses the Report Center page.
      • The system validates the actor's authentication and authorization.
      • The actor selects a filter criteria (report type tab, review subtab, or status dropdown).
      • The system retrieves the paginated list of reports matching the active combination of report type, subtype (if applicable), and status.
      • The system displays the filtered list of reports.
  • Alternative Flow (Empty Filter Result)
    • If the user selects this alternative flow:
      • The system detects the retrieved list of reports is empty.
      • The system displays an empty state message indicating that no reports were found.
      • Use case ends.
  • Postconditions
    • The system displays the paginated list of reports matching the selected filter criteria.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the actor does not have the Staff or Admin role. The system redirects the actor to the home page or access denied page.
    • Database Query Failure: The system encounters an error while retrieving the reports from the database. The system displays an error message regarding the data retrieval failure. Use case ends.
  • Security Measures
    • Independent Status Filters: The status filter in the Course Reports tab maintains its own independent state from the status filter in the Review Reports tab.
    • Subtab Filter Retention: When switching between the "Course Reviews" and "Lesson Reviews" subtabs, the currently active status filter is retained and applied to the newly selected subtype.
    • Universal Empty State Display: Whenever a list retrieval operation yields no records (either due to an empty database or active filter criteria), the system displays a user-friendly empty state message instead of a blank table or list container.
