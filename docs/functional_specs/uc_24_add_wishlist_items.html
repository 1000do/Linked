1. Add Wishlist Items

Function Trigger: User clicks the "Add to Wishlist" button (Heart Icon) on the Course Details page.

Function Description: Allows an authenticated User to add a course to their personal wishlist. The system checks if the course is already in the wishlist, creates a wishlist record, saves it to the database, and updates the heart icon state dynamically via AJAX.

Function Details

  • Preconditions
    • User must be logged in.
    • The target course must exist in the system and not already be in the User's wishlist.
  • Process
    • Add Wishlist Item Successfully:
      • User accesses the Course Details page.
      • User clicks the Add to Wishlist button (Heart Icon).
      • The system checks if the course is already in the User's wishlist.
      • The system detects that the course is not in the wishlist.
      • The system extracts the user ID from the session claims.
      • The system verifies that the course is unique in the User's wishlist.
      • The system instantiates a new wishlist item.
      • The system saves the wishlist item to the database.
      • The system displays a success message and updates the wishlist indicator on the screen.
  • Postconditions
    • A new wishlist item record is successfully saved in the database.
    • The system updates the wishlist heart icon to an active state on the frontend.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the User is not authenticated and redirects the User to the Login page.
    • Course Already in Wishlist: The system detects that the course is already in the User's wishlist, displays an error message, and returns the user to the initial step.
    • Database Save Failure: The system detects that saving the changes failed, displays an error message, and returns the user to the initial step.
  • Security Measures
    • User must be logged in.
