UC-118. View Account Detail

Function Trigger: Admin clicks on a specific account in the account list.

Function Description: Allows an Admin to view detailed profile information of any account in the system. The details retrieved dynamically vary depending on the account's role (User, Instructor, Staff, or Admin), showing custom personal, academic, and financial/Stripe statistics.

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • Admin has stable internet connection.
  • Process
    • View Account Detail Successfully:
      • Admin accesses the Account Management page.
      • Admin clicks on a specific account in the list.
      • The system verifies that the Admin is authenticated.
      • The system queries the database to retrieve the account, associated manager, user, instructor, and lockout details.
      • The system determines the role of the account.
      • For User (learner) accounts, the system calculates total spent on courses and total enrollment counts.
      • For Instructor accounts, the system retrieves title, expertise categories, Stripe credentials, and calculates total revenue, processed payouts, and available balance.
      • The system checks and retrieves the most recent active lockout timestamps.
      • The system renders and displays the account details page.
  • Postconditions
    • The account details are successfully retrieved and displayed on the screen.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
    • Account Not Found: The system detects that the target account ID does not exist in the database. The system displays the error message: "Account not found."
  • Security Measures
    • Role-Specific Data Aggregation:
      • Users/Students: Shows total purchase sum from successful payment transactions and enrolled course counts.
      • Instructors: Displays Stripe integration details, title, and calculates total revenues from platform split transactions minus completed withdrawals to yield the available balance.
      • Managers/Staff: Displays display name, role, and manager bio.
    • Lockout Detection: The system displays active lockout periods (lockout start and end timestamps) if the account has been banned.
