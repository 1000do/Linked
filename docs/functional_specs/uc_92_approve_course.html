4.4.5. Approve Course Submission

Function Trigger: Actor clicks the "Approve" button on a course row in the moderation list.

Function Description: Approves a submitted course, marking it as published and visible on the marketplace.

Function Details

  • Preconditions
    • Actor must be authenticated.
    • Actor must have staff or admin role.
    • The course status must be pending review (or rejected, archived, permanently locked if re-approving).
  • Process
    • Approve Course Submission Successfully:
      • Actor accesses the Course Moderation Page.
      • Actor locates the pending course in the moderation list.
      • Actor clicks the "Approve" button on the corresponding course row.
      • The system displays an approval confirmation dialog containing a text field for optional feedback.
      • Actor enters optional feedback.
      • Actor clicks the "Approve & Publish" button.
      • The system updates the course status to "published" and clears existing moderation feedback in the database.
      • The system updates the status of all lessons and materials under the course to "active" in the database.
      • The system sends an approval notification message to the instructor.
      • The system sends content update notifications to all enrolled students.
      • The system clears the course details cache.
      • The system updates the moderation page stats and dims the approved course row.
  • Postconditions
    • The course status is set to "published" (Approved) in the database.
    • The instructor receives a notification of the approval.
    • Existing enrolled students (if any) receive content update notifications.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Unauthorized Access: The system detects that the actor does not have staff or admin roles. The system displays an access denied page or error message. The flow terminates.
    • Database Save Failure: The database fails to commit changes. The system displays a system error message. The flow terminates.
  • Security Measures
    • Re-approval Rule: If the course status is rejected, archived, or permanently locked, the action button is labeled "Re-approve" but follows the same approval logic.
    • Notification Content: The approval notification sent to the instructor includes the optional feedback entered by the actor.
    • Cache Eviction: The Redis cache for the course details must be evicted immediately upon approval to ensure users see the updated content.
