UC-101. Sort Instructor Applications

Function Trigger: Actor selects a sorting option from the sort dropdown.

Function Description: Allows the Actor to sort the pending instructor applications list by application date (newest/oldest) or alphabetical order of the applicant's name.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • The instructor applications list is loaded on the page.
  • Process
    • Select Sorting Option:
      • Actor accesses the instructor registration requests page.
      • Actor selects a sorting criteria option from the sort dropdown menu.
    • Sort List:
      • The system sorts the loaded pending applications list according to the selected criteria.
      • The system updates the table rows to display the sorted list of applicants.
  • Alternative Flow (No Matching Applications)
    • If sorting is combined with empty results:
      • Actor combines the sorting criteria with a search query that has no match.
      • The system displays an empty state message indicating no matching registration requests were found.
  • Postconditions
    • The system updates the applications table to display applications in the selected sort order.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to sort instructor applications is restricted strictly to authenticated users with Staff or Admin roles.
