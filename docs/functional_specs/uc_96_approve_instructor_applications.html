UC-96. Approve Instructor Applications

Function Trigger: Actor clicks the "Approve" button next to a pending application.

Function Description: Allows the Actor to approve an instructor's pending registration application, which promotes their role to Instructor in the database and updates their status.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • The instructor's application must exist in a pending status.
  • Process
    • Locate Application:
      • Actor accesses the instructor registration requests page.
      • Actor locates the desired pending application.
    • Click Approve:
      • Actor clicks the "Approve" button.
      • The system displays a confirmation dialog.
    • Confirm Approval:
      • Actor clicks the "Approve" button on the confirmation dialog.
      • The system updates both the instructor's application status to "Approved" and the user's role to "Instructor" in the database.
      • The system displays a success notification message.
      • The system removes the approved application row from the pending list dashboard.
  • Alternative Flow (Cancel Approval)
    • If the Actor cancels the approval:
      • Actor clicks the "Cancel" button on the confirmation dialog.
      • The system closes the confirmation dialog.
  • Postconditions
    • The instructor's status is updated to approved in the database.
    • The user's account role is updated to Instructor in the database.
    • The approved application is removed from the active pending list dashboard.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
    • Server Processing Error: The server fails to update the application status or user role in the database, displays an error message regarding server processing failure, and returns to displaying the pending applications list page.
  • Security Measures
    • Role-based Access Control: Access to approve applications is restricted strictly to authenticated users with Staff or Admin roles.
    • Status Validation: The system validates that the application is currently in a pending state before updating the database.
