UC-23. View Profile

Function Trigger: User accesses the profile page.

Function Description: Allows an authenticated User to view their personal profile information, such as full name, username, email, phone number, bio, date of birth, and verification status. The system also synchronizes the client-side verification cookie during this request.

Function Details

  • Preconditions
    • User must be logged in with a valid session.
    • User has stable internet connection.
  • Process
    • View Profile Successfully:
      • User accesses the profile page.
      • The system checks if the AccessToken cookie is present.
      • The system parses the user ID from the session.
      • The system queries the database to retrieve user details, including account navigation and instructor info.
      • The system constructs the user profile response object.
      • The system updates the local client IsVerified cookie with the actual status.
      • The system renders and displays the user profile information.
  • Postconditions
    • The user profile information is retrieved and displayed successfully.
    • The IsVerified cookie is synchronized with the latest account verification status.
  • Error Handling
    • No Access Token / Invalid Session: The system detects that the AccessToken cookie is missing or invalid. The system redirects the User to the Login page.
    • User Not Found: The system detects that the user profile details do not exist in the database (retrieved entity is null). The system redirects the User to the Login page.
  • Security Measures
    • Verification Syncing: During the profile retrieval process, the system must dynamically read the database verification flag and update the client-side IsVerified cookie to ensure consistent page behavior and menu permissions.
