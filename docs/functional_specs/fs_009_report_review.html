UC-ReportReview - Report a review

Function Trigger: User clicks the "Report" button on a course review in the course details page.

Function Description: Allows a user to report a course or lesson review for violations such as spam, offensive content, or fake reviews. The system prompts the user to select a reason and optionally provide a description before submitting the report for moderation.

Function Details

  • Preconditions
    • User must be successfully authenticated.
    • User must have completed the course or lesson to be able to report the review.
  • Process
    • Report Course Review Successfully:
      • User accesses the course details page.
      • User locates a course review and clicks the Report button.
      • The system checks authentication.
      • The system displays the report review form modal.
      • User selects a violation reason from the dropdown.
      • User optionally inputs additional description details.
      • User clicks the Submit Report button.
      • The system validates the report details and checks for duplicate pending reports.
      • The system submits the course review report data to the server.
      • The system saves the report in the database.
      • The system displays a success message regarding report submission.
  • Alternative Flow (Report Lesson Review Successfully)
    • If the user selects this alternative flow:
      • User accesses the course learning page.
      • User locates a lesson review and clicks the Report button.
      • Back to step 3.
  • Alternative Flow (Cancel reporting)
    • If the user selects this alternative flow:
      • User clicks the Cancel button.
      • The system closes the report review modal.
      • Back to step 2.
  • Postconditions
    • The review report is recorded in the database for moderation.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Duplicate Pending Report: The system detects that the user has already reported this review with the exact same reason and the report is still pending. The system displays a duplicate report error message preventing submission. Back to step 4.
    • Database Save Failed: The system fails to save the report to the database. The system displays an error message regarding report submission failure. Back to step 4.
  • Security Measures
    • Reason Selection: A valid violation reason (Spam, Offensive, Fake, Other) must be selected before submission.
    • Duplicate Report Prevention: A user cannot report the same review with the same reason if their previous report is still pending.
