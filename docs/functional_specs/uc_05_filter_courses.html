4.10.4. Filter Courses

Function Trigger: Actor clicks a category pill, category option, price range radio, or rating option.

Function Description: Allows Guests and Users to filter the list of courses by category, rating stars, and price (Free vs. Paid). The filters can be applied singly or in combination.

Function Details

  • Preconditions
    • None.
  • Process
    • Filter Courses Successfully:
      • Actor accesses the explore courses page.
      • Actor selects a category filter (e.g., clicking a category pill).
      • The system updates the active category filter.
      • Actor selects a rating filter radio option under the "Rating" accordion.
      • The system updates the active rating filter.
      • Actor selects a price filter radio option under the "Price" accordion.
      • The system updates the active price filter.
      • The system retrieves the paginated list of courses matching all selected filter parameters.
      • The system displays the filtered results in the catalog grid.
  • Postconditions
    • Paginated list of courses matching all active category, price, and rating filters is displayed successfully in the catalog grid.
  • Error Handling
    • None.
  • Security Measures
    • Filter Combinability: Multiple filters (category, price, rating, keyword) can be applied simultaneously, narrowing the results conjunctively (AND operation).
    • Rating Filter Bounds: Rating filters select courses with average rating greater than or equal to the selected threshold (e.g., >= 4.5, >= 4.0, >= 3.5).
    • Price Filter Bounds: Price filter options classify courses as "Free" (Price == 0) or "Paid" (Price > 0).
