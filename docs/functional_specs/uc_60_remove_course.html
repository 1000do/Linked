4.10.19. Remove Courses

Function Trigger: Instructor clicks the "Delete course" button for a course in the dashboard list view.

Function Description: Allows an instructor to remove an existing course. The system requires confirmation, validates instructor rights and status, verifies that the course is not pending review and has no active student enrollments, soft deletes the course and all associated lessons, and invalidates the cached course data.

Function Details

  • Preconditions
    • User must be successfully authenticated as an approved instructor.
    • The course must exist and belong to the active instructor.
  • Process
    • Remove Course Successfully:
      • Instructor accesses the instructor dashboard.
      • Instructor clicks the "Delete course" button for a course.
      • The system displays a deletion confirmation modal.
      • Instructor clicks the "Delete" button.
      • The system validates the instructor's rights.
      • The system validates the course status.
      • The system checks if the course has active student enrollments.
      • The system marks the course as removed in the database.
      • The system marks all associated lessons as removed in the database.
      • The system invalidates the cached course data.
      • The system reloads the page to update the course list.
  • Postconditions
    • The course is marked as removed in the database.
    • All lessons associated with the course are marked as removed in the database.
    • Cached course detail data is cleared.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Invalid Instructor Rights: The system detects the instructor is unauthorized to delete this course or has an active lockout status. The system displays an error message regarding invalid instructor rights. Back to step 2.
    • Invalid Course Status: The system detects the course does not exist, is pending review, or is permanently discontinued due to policy violations. The system displays an error message regarding invalid course status. Back to step 2.
    • Course Has Enrolled Students (Backend check): The system detects that the course has active student enrollments. The system displays a failure message stating the course cannot be deleted with active students. Back to step 2.
    • Database Save Failed: The system fails to save the course changes to the database. The system displays a database transaction failure message. Back to step 2.
  • Security Measures
    • Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from submitting courses for review or changing course status.
    • Permanently Discontinued Block: A course that is 'archived' and has 3 or more flags is considered permanently discontinued due to policy violations and cannot have its status changed or be edited.
    • Pending Review Modification Lock: Learning materials and lessons cannot be added, edited, or removed if the course is currently in 'Pending' review status.
    • Active Student Deletion Restriction: Courses with active student enrollments cannot be deleted and must be archived instead.
    • Course Soft Deletion: Deleting a course soft-deletes both the course and all its associated lessons and materials (IsRemoved = true) in the database, rather than permanently destroying the records.
    • Cache Eviction: The Redis cache for the course details must be evicted immediately upon approval to ensure users see the updated content.
