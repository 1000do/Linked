UC-87. Search Chat History

Function Trigger: Actor enters a search query in the search input field of the chat sidebar.

Function Description: Allows the Actor to search their conversation list by sending a query to the backend, which searches participant profiles and returns matching conversation rooms.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • Actor is on the Chat page under the Admin Panel dashboard.
  • Process
    • Enter Query:
      • Actor accesses the Chat page.
      • Actor types a search query in the "Search users..." input box.
    • Request Search:
      • The system detects the input change and waits for a typing pause.
      • The system requests the backend to search conversations matching the query.
    • Process Search:
      • The server validates that the active Staff or Admin is authenticated.
      • The server queries the database for chat rooms where the current Staff or Admin is a participant and the other participant's name matches the search query.
      • The server returns the matching conversation list.
    • Display Results:
      • The system displays the matching conversations in the sidebar.
  • Alternative Flow (Clear Search Query)
    • If the Actor clears the search query:
      • Actor clears the search query text box.
      • The system restores the full list of active conversations from the local cache.
      • The system displays the full active conversation list.
  • Alternative Flow (No Matching Results)
    • If no matching results are found:
      • The server returns an empty list.
      • The system displays a message in the sidebar indicating "No conversations found."
  • Postconditions
    • The sidebar chat list displays only conversations matching the search query.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, redirects the Actor to the access denied error page, and terminates the flow.
  • Security Measures
    • Role Verification: Access to search conversations is restricted strictly to authenticated Staff or Admin users.
    • Input Sanitization: The system sanitizes search input text to prevent injection vulnerabilities.
