4.4.7. Flag Course for Content Violation

Function Trigger: Actor clicks the "Flag" button on a course row in the moderation list.

Function Description: Flags a course for content or policy violations, hiding it from public display.

Function Details

  • Preconditions
    • Actor must be authenticated.
    • Actor must have staff or admin role.
    • The course status must be pending, approved/published, rejected, or archived.
    • The course's current flag count must be less than 3.
  • Process
    • Flag Course for Content Violation Successfully:
      • Actor accesses the Course Moderation Page.
      • Actor locates the course in the moderation list.
      • Actor clicks the "Flag" button on the corresponding course row.
      • The system displays a flag violation warning dialog containing a text field for the violation description.
      • Actor enters the violation description.
      • Actor clicks the "Confirm Flag" button.
      • The system checks the course's current flag count.
      • The system increments the flag count by 1 in the database.
      • The system sets the course's "ModerationFeedback" containing the flag count and the violation description.
      • The system updates the course's modified date.
      • The system checks if the new flag count has reached 3.
      • The system sends a policy warning notification (Strike 1 or 2) to the instructor.
      • The system saves the changes in the database.
      • The system clears the course details cache.
      • The system updates the moderation page stats and dims the flagged course row.
  • Postconditions
    • The course's flag count is incremented by 1 in the database.
    • The instructor receives a warning or discontinuation notification.
    • If flag count reaches 3, the course status is updated to "archived" (Permanently Locked).
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Unauthorized Access: The system detects that the actor does not have staff or admin roles. The system displays an access denied page or error message. The flow terminates.
    • .1 Empty Input Validation: The system detects that the violation description text field is empty. The system displays an error message indicating a violation reason is required. The system halts submission. Back to step 5.
    • Flag Lockout: The system detects that the course's flag count is already 3 or more. The system displays an error message indicating the course is already permanently locked. The flow terminates.
    • Database Save Failure: The database fails to commit changes. The system displays a system error message. The flow terminates.
  • Security Measures
    • Flag Count Limit: A course can have a maximum of 3 flags. Flag count cannot exceed 3.
    • Strike Warnings:
      • Strike 1 (1st Flag): Sends notification "Course Violation Reminder (1st Time)". The course is temporarily hidden. Instructor is reminded to review and comply.
      • Strike 2 (2nd Flag): Sends notification "Severe Violation Warning (2nd Time)". Course remains hidden. Warning that 3rd flag leads to permanent lock.
      • Strike 3 (3rd Flag): Sends notice "Permanent Course Discontinuation Notice (3rd Time)". Status changes to "archived" (Permanently Locked). Instructor is blocked from editing, and no new enrollments are allowed.
    • Notification Link: The rejection notification sent to the instructor contains a link redirecting to the course editor "/InstructorCourse/Editor?id={courseId}".
