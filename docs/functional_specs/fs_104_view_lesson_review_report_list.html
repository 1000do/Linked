UC-104 - View List of Lesson Review Report

Function Trigger: Staff or Admin accesses the Report Center page, navigates to the "Review Reports" tab, and selects the "Lesson Reviews" subtab.

Function Description: Displays reports submitted against lesson reviews.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • The actor must be authenticated.
    • The actor must have Staff or Admin role.
  • Process
    • View List of Lesson Review Report Successfully:
      • Staff or Admin accesses the Report Center page.
      • The system validates the actor's authentication and authorization.
      • The actor selects the "Review Reports" tab.
      • The system displays the review subtabs.
      • The actor selects the "Lesson Reviews" subtab.
      • The system retrieves the paginated list of pending lesson review reports.
      • The system displays the list of lesson review reports.
  • Alternative Flow (Empty Lesson Review Report List)
    • If the user selects this alternative flow:
      • The system detects the retrieved list of lesson review reports is empty.
      • The system displays an empty state message indicating that no review reports were found.
      • Use case ends.
  • Postconditions
    • The system displays the paginated list of pending lesson review reports.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the actor does not have the Staff or Admin role. The system redirects the actor to the home page or access denied page.
    • Database Query Failure: The system encounters an error while retrieving the reports from the database. The system displays an error message regarding the data retrieval failure. Back to step 5.
  • Security Measures
    • Access Control — Report Center: Only authenticated Staff and Admin actors are permitted to access the Report Center.
    • Status Badges: Report statuses (Pending, Resolved, Rejected, Escalated) are visually distinguished using specific status badges.
    • Action Availability: The "Moderate" action button is available only for reports in Pending, Under Review, or Escalated status. All other statuses display an "Info" button instead.
    • Staff Escalation Limitation: Staff actors cannot moderate Escalated reports. Only Admins may moderate reports in the Escalated status.
