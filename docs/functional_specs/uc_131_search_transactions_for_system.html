2.2.3.131. Search transactions (For System)

Function Trigger: Admin enters search text in the transaction search input field and submits the filter form.

Function Description: Allows authenticated Admin users to search the system-wide transaction history by course title, student (buyer) name, or instructor name.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • - User must be successfully authenticated as an Admin.
    • - Admin is on the Financial Management dashboard under the Transaction History tab.
  • Process
    • Search Transactions Successfully:
      • 1. Admin accesses the Financial Management page.
      • 2. Admin selects the "Transaction History" tab.
      • 3. Admin enters a search term (e.g. course title, student name, or instructor name) in the search input box.
      • 4. Admin clicks the "Filter" button or presses Enter.
      • 5. The system requests the transaction list matching the keyword from the server, while preserving active date period filters (year/month) and paging parameters.
      • 6. The server validates that the active user possesses an Admin role.
      • 7. The server queries the database for transactions matching the search term within the selected year and month.
      • 8. The server returns the matching results.
      • 9. The system displays the matching transactions in the table.
  • Alternative Flow (Clear search keyword)
    • 1. Admin clicks the "Clear Filters" button.
    • 2. The system resets the search keyword, status filter, and sorting order to their defaults.
    • 3. The system requests the transaction list for the active period from the server.
    • 4. Back to step 6.
  • Alternative Flow (No matching transactions found)
    • 1. The server returns an empty list of transactions matching the keyword.
    • 2. The system displays a message indicating "No transactions matched your criteria".
  • Postconditions
    • - The system displays only those transactions that match the search keyword.
  • Error Handling
    • - Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • - Unauthorized Access: The server detects that the user is not an Admin. The system redirects the user to the access denied error page.
  • Security Measures
    • - N/A
