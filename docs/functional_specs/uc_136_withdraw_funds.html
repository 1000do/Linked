2.2.3.136. Withdraw Funds

Function Trigger: Admin clicks the "Withdraw Now" button on the withdrawal settings card.

Function Description: Allows administrators to manually transfer accumulated platform net profit from the Stripe available balance directly into the platform's corporate bank account.

Function Details

  • Preconditions
    • - The administrator must be successfully logged into the system.
    • - The platform's Stripe Connect available balance must be at least $0.50.
    • - Account successfully linked to Stripe's bank account.
  • Process
    • Withdraw Funds Successfully:
      • 1. Admin accesses the admin finance dashboard page AdminFinance.
      • 2. Admin locates the "Withdraw Funds to Bank" settings card showing the available balance.
      • 3. Admin enters the target payout amount in the "Amount" input field.
      • 4. Admin enters an optional transfer explanation in the "Description" input field.
      • 5. Admin clicks the "Withdraw Now" button.
      • 6. The system displays the "Confirm Withdrawal" warning dialog.
      • 7. Admin clicks the "Confirm & Initiate" button.
      • 8. The system requests Stripe to create a payout to the platform's registered bank account.
      • 9. Stripe processes the request and returns the payout transaction identifier.
      • 10. The system creates a withdrawal log record with a pending status in the database.
      • 11. The system broadcasts real-time SignalR notifications to update active dashboard balances.
      • 12. The system closes the confirmation dialog and displays a success notification.
  • Alternative Flow (Click Cancel)
    • 1. Admin clicks the "Cancel" button on the confirmation dialog.
    • 2. The system closes the confirmation dialog.
    • 3. Back to step 3.
  • Alternative Flow (Invalid Amount Entered)
    • 1. Admin enters a withdrawal value less than $0.50, a non-numeric value, or an amount exceeding the available balance.
    • 2. Admin clicks the "Withdraw Now" button.
    • 3. The system displays the "Invalid Withdrawal Amount" error dialog.
    • 4. Admin clicks the "Understood" button.
    • 5. Back to step 3.
  • Postconditions
    • - A Stripe Payout is successfully initiated to the corporate bank account.
    • - A withdrawal record is added to the database with a pending status.
    • - The dashboard balances are updated in real time via SignalR.
  • Error Handling
    • - Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
  • Security Measures
    • - Minimum Withdrawal Limit: Platform withdrawals must be at least $0.50 (USD).
    • - Maximum Available Balance: The maximum amount allowed to be withdrawn cannot exceed the platform's current Stripe Available balance. Cents are rounded to two decimal places.
