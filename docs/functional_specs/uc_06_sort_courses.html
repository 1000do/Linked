4.10.5. Sort Courses

Function Trigger: Actor selects a sorting option from the sort by dropdown list on the explore courses page.

Function Description: Allows Guests and Users to order the displayed courses by Newest, Highest Rated, Price: Low to High, or Price: High to Low.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • None.
  • Process
    • Sort Courses Successfully:
      • Actor accesses the explore courses page.
      • Actor clicks the "Sort By" dropdown.
      • Actor selects a sorting parameter (e.g., "Newest", "Highest Rated", "Price: Low to High", or "Price: High to Low").
      • The system updates the sort parameter.
      • The system retrieves the paginated list of courses ordered by the selected sort parameter.
      • The system displays the sorted courses in the catalog grid.
  • Postconditions
    • List of courses is displayed ordered by the selected sorting parameter.
  • Error Handling
    • None.
  • Security Measures
    • Newest Ordering: Courses are sorted by their creation date in descending order.
    • Highest Rated Ordering: Courses are sorted by their average rating in descending order.
    • Price Low-to-High Ordering: Courses are sorted by their price in ascending order.
    • Price High-to-Low Ordering: Courses are sorted by their price in descending order.
    • Sort Order Persistence: The selected sorting parameter is saved in the browser's local storage and persists across page reloads.
