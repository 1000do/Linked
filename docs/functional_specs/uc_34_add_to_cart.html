UC-34. Add to Cart

Function Trigger: Actor clicks the "Add to Cart" button on a course details card or page.

Function Description: Allows an authenticated user to add one or more courses to their shopping cart to purchase them together in a single transaction.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated.
    • Actor must have a verified email status.
  • Process
    • Access Course Details:
      • Actor accesses a course detail page.
      • Actor clicks the "Add to Cart" button.
    • Validate and Request:
      • The system checks for a valid authenticated session.
      • The system requests the backend to add the course to the user's shopping cart.
      • The server validates that the active user's email address is verified.
      • The server checks if the course is already in the user's cart, already purchased, or if the user is the instructor of the course.
    • Add Course:
      • The server adds the course record to the user's shopping cart in the database.
      • The system displays a success message indicating the course was added to the cart.
      • The system redirects the Actor to the Cart page.
  • Alternative Flow (Add to Cart without Redirect)
    • If the Actor adds a course to the cart without redirection:
      • Actor clicks the configured "Add to Cart" button.
      • The system requests the backend to add the course to the cart.
      • The server validates the request and adds the course record to the user's shopping cart.
      • The system displays a success toast notification on the current page and updates the cart item count badge in the header.
  • Postconditions
    • The course is successfully added to the user's shopping cart in the database.
    • The system redirects the Actor to the Cart Summary page, or displays a success toast notification on the current page.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
  • Security Measures
    • Session and Ownership Verification: The server verifies the active user session and validates that the user is not attempting to buy their own course or a course already in possession.
    • Email Verification Restriction: Only users with a verified email status are allowed to perform write operations on the cart.
