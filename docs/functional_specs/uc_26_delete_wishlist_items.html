1. Delete Wishlist Items

Function Trigger: User clicks the "Remove" button next to a course on the Wishlist page.

Function Description: Allows an authenticated User to delete a course from their personal wishlist. The system checks if the course is in their wishlist, deletes the record, and reloads the wishlist page.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • User must be logged in.
    • The target course must exist in the User's wishlist.
  • Process
    • Delete Wishlist Item Successfully:
      • User accesses the Wishlist page.
      • User clicks the Remove button next to a specific course.
      • The system checks if the User is authenticated.
      • The system extracts the user ID from the session claims.
      • The system queries the database to find the wishlist record matching the user ID and course ID.
      • The system deletes the wishlist record.
      • The system saves the changes to the database.
      • The system refreshes and displays the updated Wishlist page.
  • Postconditions
    • The wishlist item record is successfully deleted from the database.
    • The system refreshes and renders the updated Wishlist page.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the User is not authenticated and redirects the User to the Login page.
    • Course Not Found in Wishlist: The system detects that the course does not exist in the User's wishlist, displays a "Course not found in your wishlist" error message, and returns the user to the initial step.
    • Database Save Failure: The system detects that saving the changes failed, displays an error message, and returns the user to the initial step.
  • Security Measures
    • User must be logged in.
