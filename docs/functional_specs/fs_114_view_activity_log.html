UC-114 - View Activity Log

Function Trigger: Admin clicks the "AI Activity Logs" tab on the AI Service management page.

Function Description: Displays chronological paginated logs of AI moderation events across three categories: course moderation, course review moderation, and lesson review moderation. All category logs are loaded at page load; Admin can switch between category subtabs without a page reload.

Function Details

  • Preconditions
    • Admin must be successfully authenticated.
  • Process
    • View Activity Log Successfully:
      • Admin accesses the AI Service management page.
      • The system validates the Admin's access authorization.
      • The system retrieves the paginated activity logs for all three moderation categories.
      • Admin clicks the "AI Activity Logs" tab.
      • The system displays the activity log list under the course moderation subtab by default.
      • Admin views the activity log entries.
  • Alternative Flow (No Logs Found)
    • If the user selects this alternative flow:
      • The system detects no activity logs exist for one or more moderation categories.
      • The system displays an empty state message in the affected category's log list.
      • Back to step 6.
  • Alternative Flow (Admin Switches Category Subtab)
    • If the user selects this alternative flow:
      • Admin clicks a different category subtab.
      • The system displays the activity log list for the selected category.
      • Back to step 6.
  • Alternative Flow (Admin Navigates to Another Page)
    • If the user selects this alternative flow:
      • Admin clicks the "Next" or "Previous" pagination button.
      • The system reloads the page with the requested page of logs for the active category.
      • Back to step 1.
  • Postconditions
    • The activity log list for the selected category is displayed to the Admin.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the current user does not have Admin authorization. The system displays an unauthorized access error message. The system redirects the user to the login page.
    • Database Query Failed: The system fails to retrieve activity logs from the database. The system displays an error message regarding the retrieval failure. Back to step 1.
  • Security Measures
    • Universal Empty State Display: Whenever a list retrieval operation yields no records (either due to an empty database or active filter criteria), the system displays a user-friendly empty state message instead of a blank table or list container.
    • Admin Authorization for AI Services: Only authenticated Admin actors are authorized to access and perform actions within the AI Service and Model Management module (including adding/updating models, toggling status, viewing details/list, configuring thresholds, and viewing activity logs).
    • List Sorting and Ordering: Administrative lists (such as the AI model list and activity logs) are sorted in descending order of their latest timestamp (last modified date for models, creation date for logs), ensuring that the most recent entries are displayed first.
    • Universal List Pagination: All list retrieval operations displaying tabular or grid data must display a limited number of entries per page with pagination controls.
    • Default Subtab — Activity Log: The course moderation category is displayed by default when the logs tab is opened.
