UC-35. Delete Cart Items

Function Trigger: Actor clicks the delete button next to a course item on the Cart page.

Function Description: Allows authenticated Users or Instructors to remove a course from their shopping cart.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated.
    • Actor has at least one course item in their shopping cart.
    • Actor is on the Cart page.
  • Process
    • Trigger Deletion:
      • Actor accesses the Cart page.
      • Actor clicks the delete icon button next to the target course item.
    • Remove Course:
      • The system requests the backend to remove the course from the user's shopping cart.
      • The server validates that the active Actor session is valid.
      • The server deletes the course record from the user's shopping cart in the database.
      • The server returns a success response.
      • The system displays a success notification indicating the course was removed.
      • The system recalculates the cart summary prices and re-renders the Cart page.
  • Postconditions
    • The course item is permanently removed from the user's shopping cart in the database.
    • The system recalculates the cart summary prices and refreshes the Cart page.
  • Error Handling
    • Session Invalid (Not Logged In): The system redirects the Actor to the login page.
    • Course Not in Cart: The server detects that the course is not present in the user's shopping cart, and the system displays an error message indicating that the course was not found in the cart or cannot be removed.
  • Security Measures
    • Session Verification: The server validates that only the owner of the cart can remove items from it.
