UC-29. View Notification list

Function Trigger: User or Instructor clicks on the notification icon or accesses the notifications page.

Function Description: Allows users and instructors to view their list of notifications, search, sort, filter, view details, mark notifications as read, mark all notifications as read, or delete notifications.

Function Details

  • Preconditions
    • User or Instructor must be successfully authenticated.
    • Instructor must have an approved instructor status to access the instructor-specific notifications.
    • User or Instructor must have a stable internet connection.
    • SignalR Service must be working properly.
  • Process
    • Access Page:
      • User or Instructor accesses the notifications page.
      • The system sends a request to retrieve the notification list.
    • Retrieve Notifications:
      • The system retrieves the notifications from the database.
    • Display List:
      • The system displays the list of notifications.
  • Alternative Flow (Search notifications)
    • If the user searches notifications:
      • User or Instructor enters a keyword in the "Search by title or content..." input field.
      • The system filters and displays notifications containing the keyword in the title or content.
      • The process returns to displaying the list of notifications.
  • Alternative Flow (Filter notifications by status)
    • If the user filters by status:
      • User or Instructor selects a status option from the status filter dropdown.
      • The system filters and displays notifications matching the selected status.
      • The process returns to displaying the list of notifications.
  • Alternative Flow (Sort notifications)
    • If the user sorts notifications:
      • User or Instructor selects a sorting option from the sorting dropdown.
      • The system sorts and displays notifications matching the selected sorting order.
      • The process returns to displaying the list of notifications.
  • Postconditions
    • The read state of the notifications is updated in the database.
    • Deleted notifications are permanently removed from the database.
    • The frontend receives real-time updates when new notifications are sent.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Notification not found or access denied: The system displays an error message regarding missing notification or permission, and the process returns to displaying the list of notifications.
  • Security Measures
    • Instructor Access Restriction: Only instructors with an "Approved" approval status are permitted to access the instructor-specific notifications page (/Instructor/Notifications). If an instructor is not approved, they are redirected to the instructor application status page.
    • Read Status Sync: Viewing a notification's details modal automatically marks it as read in the database and updates the UI state.
