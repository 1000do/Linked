UC-95. View Instructor Applications

Function Trigger: Actor accesses the instructor registration requests page.

Function Description: Allows the Actor to view the list of pending instructor registration requests with applicant details, search, filter, and sort options.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
  • Process
    • Access Page:
      • Actor accesses the instructor registration requests page.
      • The system sends a request to retrieve the list of pending instructor applications.
    • Retrieve Applications:
      • The system validates that the active user possesses a Staff or Admin role.
      • The system retrieves the pending instructor applications from the database.
    • Display Applications:
      • The system displays the list of pending instructor applications to the Actor, including applicant details, specialization, LinkedIn link, and resume/portfolio documents.
  • Alternative Flow (No Pending Applications)
    • If no pending applications are found:
      • The system retrieves an empty list of pending applications from the database.
      • The system displays an empty state message indicating no registration requests were found.
  • Alternative Flow (Search Applications)
    • If the Actor searches applications:
      • Actor enters a search query in the search input field.
      • The system filters and displays applications containing the query in the applicant name or email.
      • The process returns to displaying the list of pending applications.
  • Alternative Flow (Filter Applications)
    • If the Actor filters applications:
      • Actor selects a specialization filter from the dropdown.
      • The system filters and displays applications matching the selected specialization.
      • The process returns to displaying the list of pending applications.
  • Alternative Flow (Sort Applications)
    • If the Actor sorts applications:
      • Actor selects a sorting order option from the dropdown.
      • The system sorts and displays applications based on the selected sorting order.
      • The process returns to displaying the list of pending applications.
  • Postconditions
    • The system displays the list of pending instructor applications with applicant details, portfolio/resume links, and processing controls.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff member or Admin, displays an access denied page or error message, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to the instructor registration requests page is restricted strictly to authenticated users with Staff or Admin roles.
    • Data Validation: The server validates the Actor's session and role permissions before retrieving and returning any application data from the database.
