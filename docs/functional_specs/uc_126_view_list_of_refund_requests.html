2.2.3.126. View List of Refund Requests

Function Trigger: The administrator clicks the "Refund Requests" tab in the Admin Finance dashboard.

Function Description: Allows administrators to view the list of pending refund requests submitted by learners within the 14-day window. Shows purchase and request details, and provides actionable links and decision triggers.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • - The administrator must be successfully logged into the system.
    • - The administrator account must have platform-wide financial management permissions.
  • Process
    • View and Approve Refund Requests Successfully:
      • 1. Administrator accesses the Admin Finance dashboard page.
      • 2. The system retrieves the list of pending refund requests from the database.
      • 3. Administrator clicks the "Refund Requests" tab.
      • 4. The system displays the refund requests table containing the following columns:
        • - ID: The sequential identifier for the refund request.
        • - Course: The title of the requested course.
        • - Student: The name of the student requesting the refund.
        • - Amount: The refund amount.
        • - Purchase Date: The date and time of the original purchase.
        • - Request Date: The date and time the refund was requested.
        • - Refund Reason: The reason provided by the student for the refund.
        • - Actions: Buttons to view "Details", "Approve", or "Reject" the request.
      • 5. Administrator reviews the list of pending refund requests.
      • 6. Administrator clicks the "Approve" button for a specific refund request.
      • 7. The system displays a confirmation popup asking the administrator to confirm the approval.
      • 8. Administrator clicks "Confirm" on the popup.
      • 9. The system processes the approval, initiates the refund via Stripe, and revokes the student's course access.
  • Postconditions
    • - The administrator views the list of pending refund requests including the transaction ID, course title, student name, amount, purchase date, request date, and student refund reason.
    • - If approved, the refund is processed, and the student's course access is revoked.
  • Error Handling
    • - Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
  • Security Measures
    • - Action Confirmation: The system requires a confirmation popup when approving or rejecting a refund request to prevent accidental actions.
