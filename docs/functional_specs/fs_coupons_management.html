1. Add Coupons

Function Trigger: Admin clicks the "Create Coupon" button on the coupon list page.

Function Description: Allows an Admin to create a new coupon in the coupon warehouse by specifying a unique code, coupon type, discount value, minimum order value, validity dates, and usage limit.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
  • Process
    • Add Coupon Successfully:
      • Admin accesses the Coupon List view.
      • Admin clicks the Add Coupon button.
      • The system displays the coupon creation form.
      • Admin enters the coupon details: Code, Type, Value, Min Order Value, Start Date, End Date, and Limit.
      • Admin clicks the Save button.
      • The system normalizes the coupon type and validates form inputs.
      • The system checks if the coupon code is unique in the database.
      • The system instantiates a new Coupon entity with Used Count = 0 and Is Active = true.
      • The system saves the coupon details to the database.
      • The system displays a success message and renders the updated Coupon List.
  • Postconditions
    • The new coupon is successfully saved to the database.
    • The system renders the updated Coupon List / Success message.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Validation Failure: The system detects invalid coupon parameters, displays a validation error message explaining the failure, and returns the user to the input step.
    • Coupon Code Already Exists: The system detects that the coupon code already exists in the database, displays an error message, and returns the user to the input step.
    • Database Save Failure: The system detects that saving the coupon details failed, displays an error message regarding database save failure, and returns the user to the input step.
  • Security Measures
    • Admin must be logged in with Admin role privileges.

2. View Coupons List

Function Trigger: Admin accesses the "Coupons & Offers" page.

Function Description: Allows an Admin to browse a list of all coupons in the system. The Admin can search by code or ID, filter by active status or type, and sort the list client-side.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
  • Process
    • View Coupons List Successfully:
      • Admin accesses the Coupon List page.
      • The system retrieves the list of coupons sorted by CouponId descending.
      • The system maps the coupon list to the coupon response details (ID, Code, Type, Value, Min Order Value, Start Date, End Date, Usage Limit, Used Count, Active Status).
      • The system renders and displays the Coupon List table.
  • Alternative Flow (Filter Coupons)
    • Admin filters by active status or coupon type (Fixed/Percentage):
      • Admin selects active status filter or coupon type filter.
      • The system retrieves the matched coupon list from the database.

  • Alternative Flow (Search Coupons)
    • Admin searches by keyword:
      • Admin enters search keyword (Coupon Code).
      • The system retrieves the matching coupon list from the database.
      • The system renders and displays the searched table.
  • Alternative Flow (Sort Coupons)
    • Admin sorts by table header:
      • Admin clicks on a table header column (Date Create, Expiring Soon, Most Popular) to sort.
      • The system sorts the cached coupon list in memory (ascending or descending order).
      • The system rerenders and displays the sorted table.
  • Postconditions
    • The coupon list is successfully retrieved and rendered on the frontend.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.

Function Description: Allows an Admin to update properties of an existing coupon (End Date, Usage Limit, or Active Status). Core parameters such as Coupon Code, Coupon Type, and Discount Value are locked to preserve transactional history.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • The target coupon must exist in the system.
  • Process
    • Edit Coupon Successfully:
      • Admin accesses the Coupon List page.
      • Admin clicks the Edit button on a specific coupon.
      • The system displays the coupon edit form with current coupon details (Coupon Code, Coupon Type, and Discount Value are read-only).
      • Admin modifies the allowed fields: End Date, Usage Limit, or Active Status.
      • Admin clicks the Save button.
      • The system retrieves the target coupon by ID.
      • The system validates the modified fields against business rules.
      • The system updates the allowed properties of the Coupon entity.
      • The system saves the changes to the database.
      • The system displays a success message and renders the updated Coupon List.
  • Postconditions
    • The updated coupon properties are successfully saved to the database.

4. Delete Coupons

Function Trigger: Admin clicks the "Delete" button on a coupon in the Actions row of the coupon list.

Function Description: Allows an Admin to soft-delete (disable) a coupon. Instead of physically removing the record from the database, the coupon is marked inactive and its end date is moved to the past to preserve historical data for accounting/auditing.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • The target coupon must exist in the system.
  • Process
    • Delete Coupon Successfully:
      • Admin accesses the Coupon List page.
      • Admin clicks the Delete button on a specific coupon.
      • The system retrieves the target coupon by ID.
      • The system updates the coupon status to inactive (IsActive = false) and shifts the End Date to yesterday.
      • The system saves the changes to the database.
      • The system displays a success message and renders the updated Coupon List.
  • Postconditions
    • The coupon is successfully soft-deleted (marked inactive) in the database.
    • The system renders the updated Coupon List / Success message.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Coupon Not Found: The system detects that the target coupon ID does not exist in the database, displays a "Coupon not found" error message, and returns the user to the Coupon List page.
    • Database Save Failure: The system detects that saving the changes failed, displays an error message regarding database save failure, and returns the user to the Coupon List page.
  • Security Measures
    • Admin must be logged in with Admin role privileges.
    • Hard deletion (physical delete) is strictly forbidden to maintain accounting integrity and historical traceability.

8. Apply Coupons

Function Trigger: User enters a coupon code in the coupon input field and clicks the "Apply" button on the Cart page.

Function Description: Allows an authenticated User to apply a coupon code to their shopping cart. The system validates the coupon's status, expiry, limits, and minimum order requirements, calculates the eligible discount, and updates the cart totals.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • User must be logged in.
    • User must have at least one item in their shopping cart.
  • Process
    • Apply Coupon Successfully:
      • User enters a coupon code in the coupon input field on the Cart page.
      • User clicks the Apply button.
      • The system saves the coupon code to the session cookie map.
      • The system requests the cart summary details including the coupon code.
      • The system queries the database to retrieve the coupon details.
      • The system validates that the coupon exists, is active, has not expired, is under the usage limit, and meets the minimum order requirements.
      • The system calculates the discount amount according to the coupon type (Fixed or Percentage).
      • The system deducts the discount from the subtotal to calculate the final total.
      • The system displays the updated cart with the discount value, updated total, and a success message.
  • Postconditions
    • The coupon code is stored in the User's session state.
    • The cart totals (discount, subtotal, and total price) are recalculated and displayed on the screen.
  • Error Handling
    • Invalid or Ineligible Coupon: The system detects that the coupon code is invalid (e.g., does not exist, is disabled, has expired, has met its limit, or fails minimum order criteria), resets the discount to zero, records a validation error message, and displays the cart view with the warning message.
  • Security Measures
    • User must be logged in.
    • Applied coupons are written to local cookies to maintain the discount status across browsing activities until checkout.

9. Cancel Coupons

Function Trigger: User clicks the "Cancel" / "Remove Coupon" button next to the applied coupon on the Cart page.

Function Description: Allows an authenticated User to remove an applied coupon from their shopping cart. The system removes the coupon code from the user's session cookies, recalculates the cart summary totals without the coupon discount, and refreshes the cart view showing original prices.

Screen Layout:
[Placeholder for UI mockups]

Function Details

  • Preconditions
    • User must be logged in.
    • A coupon must be currently applied to the shopping cart.
  • Process
    • Cancel Coupon Successfully:
      • User clicks the Cancel / Remove Coupon button next to the applied coupon on the Cart page.
      • The system captures the cancellation request.
      • The system deletes the coupon code from the session cookie map.
      • The system requests the cart summary details without any coupon code.
      • The system queries the database to retrieve the User's cart items.
      • The system recalculates the cart totals using the original course prices without applying any coupon discount.
      • The system displays the updated cart with the original prices, zero discount, and an empty coupon input field.
  • Postconditions
    • The coupon code is deleted from the User's session cookies.
    • The cart totals are recalculated to reflect original prices and rendered on the screen.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the User is not authenticated and redirects the User to the Login page.
  • Security Measures
    • User must be logged in.
