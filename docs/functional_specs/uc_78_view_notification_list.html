UC-78. View Notification List

Function Trigger: Actor accesses the System Notifications settings page.

Function Description: Allows the Actor to view the list of all system-sent notifications, with search, role tab filtering, status filtering, sorting, page navigation, and detail view modal options.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
  • Process
    • Access Page:
      • Actor accesses the System Notifications page.
      • The system sends a request to retrieve the list of all notifications.
    • Retrieve Notifications:
      • The server validates that the active user possesses a Staff or Admin role.
      • The server retrieves all notifications log from the database and returns the list.
    • Display List:
      • The system displays the notifications log table list, along with search and filter inputs.
  • Alternative Flow (Filter by Role Tab)
    • If the Actor filters by role tab:
      • Actor clicks a role tab (such as "Students", "Instructors", or "Staff & Admin").
      • The system filters the list to display only notifications sent to recipients matching the selected role.
  • Alternative Flow (Search Notifications)
    • If the Actor searches notifications:
      • Actor enters a search term in the keyword search box.
      • The system filters the list to display only notifications matching the keyword in recipient name, email, title, or content.
  • Alternative Flow (Filter by Status)
    • If the Actor filters by status:
      • Actor selects a status option (such as "Unread") from the status filter dropdown.
      • The system filters the list to display only notifications matching the selected status.
  • Alternative Flow (Sort Notifications)
    • If the Actor sorts notifications:
      • Actor selects a sorting option (such as "Recipient (A - Z)") from the sort dropdown.
      • The system sorts the displayed notification list accordingly.
  • Alternative Flow (Clear Filters)
    • If the Actor clears filters:
      • Actor clicks the "Reset" button.
      • The system resets the search keyword, status filter, sorting order, and active tab to their default states.
      • The system displays the full list of notifications.
  • Alternative Flow (View Details Modal)
    • If the Actor views the details modal:
      • Actor clicks on a notification row.
      • The system displays the notification detail modal showing the recipient, title, full content, timestamp, and status.
  • Postconditions
    • The system displays the comprehensive notifications log table, role tabs, search bar, and filter controls.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to system-sent notification lists is restricted strictly to authenticated users with Staff or Admin roles.
    • Data Sanitization: The system validates and sanitizes search queries to protect system data integrity.
