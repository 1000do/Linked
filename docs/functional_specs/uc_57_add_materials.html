4.10.16. Add Learning Materials

Function Trigger: Instructor clicks the "Upload Video" area for video content, or clicks the "Choose File" button for supplementary files in the lesson editor.

Function Description: Allows an instructor to add new learning materials (such as videos or document files) to an empty lesson. The system enforces business rules including Stripe account limits, handles file uploads to the cloud, and updates the course status to draft if it was previously published.

Function Details

  • Preconditions
    • User must be successfully authenticated as an approved instructor.
    • The lesson must exist and belong to a course owned by the active instructor.
  • Process
    • Add Material Successfully:
      • Instructor accesses the course edit page.
      • Instructor selects a specific lesson from the curriculum list.
      • Instructor clicks the "Upload Video" area for video content, or clicks the "Choose File" button for supplementary files.
      • The frontend system directly uploads the file to the Cloud Storage Service and submits the resulting URL to the backend.
      • The system validates the instructor's rights.
      • The system validates the course status.
      • The system validates the document limits.
      • The system creates the new material record in the database.
      • The system updates the course status to draft and clears previous moderation feedback (if previously published).
      • The system invalidates the cached course data.
      • The system displays a success message.
  • Postconditions
    • The new material is uploaded and linked to the lesson.
    • The physical file is stored in the Cloud Storage Service.
    • If the course was previously published, its status is reverted to draft and moderation feedback is cleared.
    • Stale cache data is cleared.
  • Error Handling
    • .1 Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Lesson Not Found: The system halts execution and displays a lesson not found error message.
    • Invalid Instructor Rights: The system detects the instructor is unauthorized to add material to this lesson or has an active lockout status. The system displays an error message regarding invalid instructor rights. Back to step 3.
    • Invalid Course Status: The system detects the course is pending review or is permanently archived due to policy violations. The system displays an error message regarding invalid course status. Back to step 3.
    • Document Limit Exceeded: The system detects the instructor has not linked an active Stripe Connect profile and already reached the maximum of 2 documents for the lesson. The system displays an error message regarding the document limit. Back to step 3.
    • Database Save Failed: The system fails to save the material changes to the database. The system displays a database transaction failure message. Back to step 3.
  • Security Measures
    • Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from submitting courses for review or changing course status.
    • Pending Review Modification Lock: Learning materials and lessons cannot be added, edited, or removed if the course is currently in 'Pending' review status.
    • Permanently Discontinued Block: A course that is 'archived' and has 3 or more flags is considered permanently discontinued due to policy violations and cannot have its status changed or be edited.
    • Supplementary Document Limits: Instructors who have not linked an active Stripe Connect profile are only allowed to attach up to 2 supplementary documents per lesson.
    • Syllabus Change Reversion: Adding or removing lessons and materials, or editing their content, in a 'Published' course automatically changes the course status back to 'Draft' and clears any existing moderation feedback.
