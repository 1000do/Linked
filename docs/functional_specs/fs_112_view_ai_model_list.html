UC-112 - View AI Model List

Function Trigger: Admin accesses the AI Service management page.

Function Description: Displays a paginated list of all registered AI moderation models, showing each model's name, version, type, process type, status, and last modified date. Admin can navigate between pages using pagination controls.

Function Details

  • Preconditions
    • Admin must be successfully authenticated.
  • Process
    • View AI Model List Successfully:
      • Admin accesses the AI Service management page.
      • The system validates the Admin's access authorization.
      • The system retrieves the paginated list of AI models from the database.
      • The system displays the AI model list under the model management tab.
      • Admin views the AI model list.
  • Alternative Flow (No AI Models Registered)
    • If the user selects this alternative flow:
      • The system detects no AI models are registered in the database.
      • The system displays an empty state message in the model list area.
      • Back to step 5.
  • Alternative Flow (Admin Navigates to Another Page)
    • If the user selects this alternative flow:
      • Admin clicks the "Next" or "Previous" pagination button.
      • The system reloads the page with the requested page of AI models.
      • Back to step 1.
  • Postconditions
    • The AI model list is displayed to the Admin in paginated form.
    • The model list is ordered by last modified date in descending order.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the current user does not have Admin authorization. The system displays an unauthorized access error message. The system redirects the user to the login page.
    • Database Query Failed: The system fails to retrieve the AI model list from the database. The system displays an error message regarding the retrieval failure. Back to step 1.
  • Security Measures
    • Universal Empty State Display: Whenever a list retrieval operation yields no records (either due to an empty database or active filter criteria), the system displays a user-friendly empty state message instead of a blank table or list container.
    • Admin Authorization for AI Services: Only authenticated Admin actors are authorized to access and perform actions within the AI Service and Model Management module (including adding/updating models, toggling status, viewing details/list, configuring thresholds, and viewing activity logs).
    • List Sorting and Ordering: Administrative lists (such as the AI model list and activity logs) are sorted in descending order of their latest timestamp (last modified date for models, creation date for logs), ensuring that the most recent entries are displayed first.
    • Universal List Pagination: All list retrieval operations displaying tabular or grid data must display a limited number of entries per page with pagination controls.
