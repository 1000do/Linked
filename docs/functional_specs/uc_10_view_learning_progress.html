4.10.7. View Learning Progress

Function Trigger: Actor accesses the enrolled courses page or accesses the course player/learning page.

Function Description: Allows Users and Instructors to track learning progress for enrolled or owned courses. Actors can view progress percentages on their enrolled courses dashboard, continue learning in the course player, view videos, read text documents, download learning materials, and automatically update their progress checks. When all materials are completed, the course is marked as completed.

Function Details

  • Preconditions
    • Actor must be authenticated.
    • Actor must be enrolled in the course, or be the owner (Instructor) of the course.
  • Process
    • View Course Progress Successfully:
      • Actor accesses the "My Enrolled Courses" page.
      • The system checks the actor's session authentication.
      • The system retrieves the list of courses the actor is enrolled in.
      • For each course, the system retrieves the total material count and the completed material count.
      • For each course, the system calculates the progress percentage.
      • The system displays the enrolled courses page showing the course title, instructor, thumbnail, progress percentage, completion status, and action buttons.
      • Actor clicks the "Continue learning" button (or "Review" button if the course is completed) on a course card.
      • The system retrieves the course details (lessons and learning materials).
      • The system validates the actor's enrollment status or course ownership.
      • The system retrieves the actor's learning progress for this course, including completed material IDs.
      • The system calculates the overall completion percentage.
      • The system displays the course player showing the media viewer, the overall progress percentage and progress bar in the header, and checkmarks next to completed items in the syllabus sidebar.
  • Postconditions
    • Detailed course list with progress percentage, completion status, and navigation controls is displayed.
    • Overall progress percentage and completion bar are rendered in the course player header.
    • Checkmark icons are displayed next to completed learning materials in the player's curriculum sidebar.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Access Denied (Not Enrolled): The system detects the actor is not enrolled in the course and is not the course owner. The system redirects the actor to the error page.
  • Security Measures
    • Progress Calculation: Course progress percentage is calculated as (Completed Materials / Total Materials) * 100, rounded to 1 decimal place.
    • Video Completion Trigger: A video material is marked as completed only when the video playback reaches the end ("onended" event).
    • Document/Text Completion Trigger: A text document (".txt") is marked as completed immediately upon viewing. Other downloadable resources are marked as completed when the User clicks the download button.
    • Course Completion State: When all learning materials in a course are marked as completed, the system automatically sets the enrollment status IsCompleted = true and updates the completion date in the database.
    • Instructor Bypass: Course owners/instructors can access the Course Player ("Learn" page) for preview, and their progress is treated as 100% completed virtual progress without writing completions to the database.
