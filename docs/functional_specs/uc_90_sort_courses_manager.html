4.4.3. Sort Courses

Function Trigger: Actor selects a sorting option from the "sortBy" dropdown on the Course Moderation Page.

Function Description: Sorts courses in the moderation list by submission date or threat level.

Function Details

  • Preconditions
    • Actor must be authenticated.
    • Actor must have staff or admin role.
    • Actor must be on the Course Moderation Page.
  • Process
    • Sort Courses Successfully:
      • Actor selects a sorting option from the "sortBy" dropdown.
      • The system retrieves the courses list sorted according to the selected option and other active filters.
      • The system updates the course moderation table and statistics cards.
      • The system displays the sorted course list.
  • Postconditions
    • The system displays the course moderation list sorted by the selected criteria.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Unauthorized Access: The system detects that the actor does not have staff or admin roles. The system displays an access denied page or error message. The flow terminates.
    • API Call Failed: The system fails to retrieve the sorted course list or stats from the API. The system displays an error message regarding sorting failure. The system renders the Course Moderation page with an empty list. The flow terminates.
  • Security Measures
    • Exclusion of Drafts: Courses with status "Draft" are never retrieved or displayed in the moderation queues.
    • Sorting Options:
      • "Threat Level (High to Low)" ("threat_desc"): Sorts descending by threat level (Critical Threat > High Threat > Clean > None), then ascending by creation date. This is the default sort.
      • "Threat Level (Low to High)" ("threat_asc"): Sorts ascending by threat level, then ascending by creation date.
      • "Oldest (Priority)" ("oldest"): Sorts ascending by creation/submission date (oldest first).
      • "Newest" ("newest"): Sorts descending by creation/submission date (newest first).
    • Preservation of Filters: Searching preserves all other active filters (Category, Status, Sort Option).
