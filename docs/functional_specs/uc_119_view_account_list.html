UC-119. View Account List

Function Trigger: Admin accesses the Manage Accounts page.

Function Description: Allows an Admin to view a paginated list of all registered accounts. The Admin can search for accounts, filter them by role, and navigate through the pages.

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • Admin has stable internet connection.
  • Process
    • View Account List Successfully:
      • Admin accesses the Account List page.
      • The system checks if the Admin is authenticated.
      • The system requests the account list using default query parameters (page = 1, pageSize = 10, all roles).
      • The system queries the database to fetch account records, including associated manager, user, and instructor details, ordered by creation date descending.
      • The system maps the account records to list details.
      • The system calculates pagination values and total record counts.
      • The system renders and displays the Account List table and navigation controls.
  • Alternative Flow (Filter Accounts)
    • If the Admin filters accounts:
      • Admin selects a role filter value from the dropdown menu.
      • The system queries the database using the selected role filter criteria.
      • The process returns to displaying the Account List table and navigation controls.
  • Alternative Flow (Search Accounts)
    • If the Admin searches accounts:
      • Admin enters a keyword in the search input box.
      • The system queries the database for accounts whose email, phone, display name, or full name match the keyword.
      • The process returns to displaying the Account List table and navigation controls.
  • Alternative Flow (Paginate Accounts)
    • If the Admin paginates accounts:
      • Admin clicks a page number or page navigation button.
      • The system requests the page index using page parameter.
      • The process returns to displaying the Account List table and navigation controls.
  • Postconditions
    • The paginated account list is successfully displayed on the screen.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
  • Security Measures
    • Default Ordering: The list is ordered by creation date descending (AccountCreatedAt desc) by default.
    • Search Criteria: Search keywords are matched (case-insensitive substring) against the account's Email, Phone Number, Manager Display Name, or User Full Name.
    • Role Filters: Supported role filters include: "user" (students), "instructor" (instructors), "staff" (all staff members), "registered_staff" (verified staff members), or "all" roles.
    • Default Pagination: The page size defaults to 10 accounts per page if not specified otherwise.
