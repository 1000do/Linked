4.10.12. Edit Courses

Function Trigger: Instructor clicks the "edit" icon of a course from his/her course list.

Function Description: Allows instructors to modify details of their courses (title, description, category, price, requirements, and promotional thumbnails) with integrated automatic debounced saving and content security hashing.

Function Details

  • Preconditions
    • User must be successfully authenticated as an approved Instructor.
    • The course must exist in the database and belong to the active Instructor's Account ID.
  • Process
    • Edit Course Details Successfully:
      • Instructor accesses the course editor.
      • Instructor modifies the course details.
      • The system triggers a brief debounce delay.
      • The system validates the form inputs.
      • The system validates the instructor's rights.
      • The system validates the course status.
      • The system retrieves the thumbnail from the Cloud Storage Service.
      • The system computes the course fingerprint hashes.
      • The system saves the updated course details and fingerprint hashes in the database.
      • The system invalidates the cached course data.
      • The system displays a success message.
  • Postconditions
    • Course details are updated in the primary course tables.
    • Content hashes are computed and updated in the secondary table.
    • The stale Redis cache for this course is invalidated.
  • Error Handling
    • .1 Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • .2 Course Not Found: The system displays a course not found error message. The system redirects the user back to the course list page.
    • Invalid Instructor Rights: The system detects the instructor is unauthorized to edit this course or has an active lockout status. The system displays an error message regarding invalid instructor rights. Back to step 1.
    • Invalid Course Status: The system detects the course is pending review or is permanently discontinued due to policy violations. The system displays an error message regarding invalid course status. Back to step 1.
    • File Upload Failed: The system fails to upload the thumbnail file to the Cloud Storage Service. The system displays a file upload error message. Back to step 2.
    • Duplicate Course Content Detected: The system detects a duplication in course content using the computed fingerprint. The system displays an error message regarding content duplication. Back to step 2.
    • Database Save Failed: The system fails to save the course updates or fingerprint hashes to the database. The system displays a database transaction failure message. Back to step 2.
  • Security Measures
    • Debounce Optimization: Auto-saving uses a 1.5-second debounce delay to prevent database spamming during rapid typing.
    • Draft Reversion: To ensure content integrity, any edits made to an already "published" course will automatically degrade its status back to "draft", requiring a new moderation review submission and clearing previous moderation feedback.
    • Stripe Payout Connectivity: Instructors must connect and activate their Stripe accounts to set premium prices. If Stripe is disconnected or inactive, the course price is locked to 0.00 (Free).
    • Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from submitting courses for review or changing course status.
    • Content Integrity Hashing: Textual details and the thumbnail file are hashed using MD5 and stored as a unique fingerprint to prevent duplication and support anti-plagiarism tracking.
