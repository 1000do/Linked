UC-22. Update Personal Info

Function Trigger: User clicks "Edit Profile" button, enters personal profile details, and clicks the "Save Changes" button on the Edit Profile form.

Function Description: Allows an authenticated User to update their personal details. The system processes validations, handles image upload, updates information within a transaction, and synchronizes frontend display cookies.

Function Details

  • Preconditions
    • User must be logged in with a valid session.
    • User has stable internet connection.
  • Process
    • Update Personal Info Successfully:
      • User enters personal info.
      • User clicks the Save Changes button.
      • The system validates the input formats.
      • The system verifies that the User has a valid AccessToken cookie.
      • The system extracts the user ID from the session.
      • The system retrieves the User and Account records from the database.
      • The system validates the email format (ends with "@gmail.com") and ensures it is not already taken (if modified).
      • The system uploads the avatar image file (if provided) and retrieves the uploaded image URL.
      • The system updates the User and Account entities and saves the changes in a database transaction.
      • The system queries the updated user profile from the database.
      • The system updates local display cookies (UserName, AvatarUrl, IsVerified) with the new details.
      • The system redirects the User to the Profile view with a success message.
  • Postconditions
    • The user profile information is updated in the database.
    • Local display cookies are synchronized, and the User is redirected to their Profile page with a success message.
  • Error Handling
    • No Access Token / Invalid Session: The system detects that the AccessToken cookie is missing or invalid. The system redirects the User to the Login page.
    • User Not Found: The system detects that the user profile does not exist. The system displays an error message.
    • Google Account Email Modification Blocked: The system detects that the user registered via Google OAuth and attempted to modify their email address. The system displays a block message indicating email changes are disabled for Google accounts.
    • Invalid Email Format: The system detects that the modified email does not end with "@gmail.com". The system displays an email format error message.
    • Avatar Upload Failure: The system detects that the avatar file upload failed. The system displays an image upload error message.
    • Email Already Taken: The system detects that the modified email is already in use by another account. The system rolls back the database transaction and displays an error message.
  • Security Measures
    • Google Email Lock: Email modification is strictly locked for Google OAuth accounts to prevent authorization mismatches.
    • Transactional Integrity: Profile modifications spanning User and Account database tables must be executed within a single transaction. If any part of the query fails (e.g., duplicate email checks), the transaction rolls back completely.
    • Session Synchronization: Client-side display cookies (such as UserName and AvatarUrl) must be immediately updated on success to ensure the page headers dynamically reflect the changes.
