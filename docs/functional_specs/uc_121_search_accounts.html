UC-121. Search Accounts

Function Trigger: Admin enters a search keyword in the search input box.

Function Description: Allows an Admin to search for accounts by specifying a keyword that matches the account's Email, Phone Number, Display Name, or Full Name.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • Admin is currently on the Account List page.
    • Admin has stable internet connection.
  • Process
    • Search Accounts Successfully:
      • Admin enters a keyword in the search input box.
      • The system captures the keyword and triggers a query request.
      • The system queries the database for accounts whose email, phone number, display name, or full name match the search keyword.
      • The system paginates the matched results and calculates the total count.
      • The system maps the matching account records to list details.
      • The system renders and displays the searched Account List table.
  • Postconditions
    • The matching accounts are successfully retrieved and rendered on the screen.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
  • Security Measures
    • Flexible Keyword Match: The system queries database records using case-insensitive substring comparisons on:
      • Email (from the Account database records).
      • PhoneNumber (from either Account or Manager profiles).
      • DisplayName (from Manager profiles).
      • FullName (from either User or Manager profiles).
