2.2.3.18. Pay for a gift

Function Trigger: System requests recipient details within the Gift course configuration workspace.

Function Description: Allows the giver to enter and validate the recipient's personal identity details (full name and email address), ensuring accurate delivery coordinates for the automated redeem voucher.

Function Details

  • Preconditions
    • - The user must be authenticated.
    • - The gift details must have been successfully configured and passed to the checkout page.
  • Process
    • Pay for a Gift Successfully:
      • 1. The user accesses the Gift Checkout page.
      • 2. The system displays the billing contact email, the gift card details summary, the course information, the pricing details, and the payment card entry form.
      • 3. The user inputs the Cardholder Name, Card Number, Expiry Date, CVC, and Country or Region.
      • 4. The user clicks the "Pay Securely" button.
      • 5. The system validates that the Cardholder Name is entered.
      • 6. The system communicates with the Stripe Payment Gateway to process the card transaction.
      • 7. The Stripe Payment Gateway confirms the payment is successful.
      • 8. The system inserts the order, order item, transaction, and gift records in a single database transaction.
      • 9. The system calculates the Stripe transaction fee and the instructor's payout amount, then saves the payout record in the database.
      • 10. The system triggers the background tasks to send a gift card email with the redemption link to the recipient.
      • 11. The system detects the recipient email is not registered in the system and skips sending an in-app notification.
      • 12. The system triggers the background tasks to send a sales notification to the course instructor.
      • 13. The system redirects the user to the course list page.
      • 14. The system displays a checkout success message.
  • Alternative Flow (Missing cardholder name)
    • 1. The system highlights the Cardholder Name field and displays a validation error message.
    • 2. Back to step 3.
  • Alternative Flow (Payment declined)
    • 1. The Stripe Payment Gateway declines the payment card.
    • 2. The system displays a payment error message.
    • 3. Back to step 3.
  • Alternative Flow (Recipient email is registered)
    • 1. The system triggers the background task to send an in-app notification to the recipient.
    • 2. Back to step 12.
  • Postconditions
    • - Payment has been processed and successfully recorded.
    • - A unique redemption code has been generated and sent to the recipient's Gmail address.
    • - Payment schedule for the instructor has been planned.
  • Error Handling
    • - Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • - Gift Details Missing: The system detects that the required gift parameters (recipient email, etc.) are missing. The system displays an error message. The system redirects the user to the home page.
    • - Payment Intent Creation Failed: The system fails to create the payment intent on the backend checkout service. The system displays an error message. The system redirects the user back to the gift setup page.
    • - Email Delivery Failed: The system logs the email delivery failure. Back to step 11.
  • Security Measures
    • - Gift Price Restrictions: Gift purchases are always made at the course's original price. Promotional coupons and discount codes are not applicable to gift purchases.
    • - Claim and Refund Mutual Exclusion: Once a gift is successfully claimed by the recipient, any pending refund request for that gift transaction is automatically rejected.
    • - No Cart Modification: Gifting is treated as a direct purchase and does not clear or modify the items in the user's active shopping cart.
