4.10.2. View Course Detail

Function Trigger: Actor clicks on a course from the public course list, category page, or search results.

Function Description: Allows Guests and Users to view the detailed information of a course, including its description, curriculum, instructor profile, and student reviews. The system determines the visibility of course content and available actions (e.g., "Add to cart", "Enroll Now", "Go to learning") based on the actor's authentication and enrollment status.

Function Details

  • Preconditions
    • None.
  • Process
    • View Course Details Successfully:
      • Actor accesses the course details page.
      • The system retrieves the course details.
      • The system retrieves the active instructor's statistics and biography.
      • The system retrieves the default course review statistics.
      • The system retrieves the first page of course reviews.
      • The system checks the actor's authentication status.
      • The system retrieves the actor's wishlist status for this course (if authenticated).
      • The system retrieves the actor's enrollment status for this course (if authenticated).
      • The system retrieves the actor's review eligibility (if authenticated).
      • The system displays the course details page, showing the categories, metadata, curriculum, and reviews.
  • Postconditions
    • Detailed course details, including landing section, pricing actions, curriculum accordion, instructor bio, and rating statistics are displayed matching the selected course.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Course Not Found: The system detects the course does not exist. The system redirects the actor to the Not Found page.
    • Unauthorized Access (Unpublished Course): The system detects the course is not published. The system detects the actor is not the course owner, a system manager, or an enrolled student. The system redirects the actor to the Error page.
    • .14.1 Delete Review Blocked (Under Moderation): The system detects the review is under moderation. The system displays a warning message stating the review cannot be deleted. Back to step 10.
  • Security Measures
    • Curriculum Locking: Only the first material of the first lesson of the first section is available for preview to non-enrolled actors. All subsequent materials are visually locked (showing a lock icon) and cannot be accessed from the details page.
    • Unpublished Course Access Restrictions: Courses with statuses other than "published" (e.g., draft, pending, archived) are strictly hidden from the public and can only be viewed by the original instructor, system administrators, or students who were previously enrolled.
    • Review Visibility: Course reviews and rating statistics are publicly visible to all actors.
    • Wishlist State: The wishlist toggle option is only displayed to Users. Instructors viewing their own course and system administrators cannot add courses to their wishlist.
    • Course Reporting: The option to report a course is only visible to enrolled students. It is hidden from the course owner, administrators, and for courses that are pending or already permanently archived with 3 or more flags.
    • Review Eligibility: The ability to write and submit a course review is strictly limited to authenticated Users who are enrolled in the course and have completed 100% of the lessons.
