UC-108 - Add AI Model

Function Trigger: Admin clicks the "Add AI Model" button on the AI Service management page.

Function Description: Allows an Admin to register a new AI moderation model configuration in the system by filling out the model registration form. The system validates all required fields before persisting the new model record to the database.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be successfully authenticated.
  • Process
    • Add AI Model Successfully:
      • Admin accesses the AI Service management page.
      • The system validates the Admin's access authorization.
      • Admin clicks the "Add AI Model" button.
      • The system displays the model registration modal.
      • Admin fills in the model details.
      • Admin clicks the "Add Model" button.
      • The system displays a confirmation dialog.
      • Admin confirms the action.
      • The system validates the form inputs.
      • The system persists the new AI model record to the database.
      • The system displays a success notification.
      • The system reloads the page.
  • Alternative Flow (Admin Cancels Confirmation)
    • If the user selects this alternative flow:
      • Admin clicks the cancel button on the confirmation dialog.
      • Back to step 4.
  • Alternative Flow (Form Validation Failed)
    • If the user selects this alternative flow:
      • The system detects one or more required fields are invalid.
      • The system displays a validation error message.
      • Back to step 4.
  • Postconditions
    • A new AI model configuration record is persisted to the database.
    • The model list on the UI is refreshed with the added model as its first entry.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • Unauthorized Access: The system detects the current user does not have admin-level authorization. The system displays an unauthorized access error message. The system redirects the user to the login page.
    • Database Save Failed: The system fails to persist the model record to the database. The system displays an error message regarding the database save failure. Back to step 4.
  • Security Measures
    • Universal Form Data Validation: All submitted form fields must pass strict data type, format, length, and value constraints (e.g., report descriptions must be at least 20 characters, and confidence thresholds must be decimals between 0.01 and 1.0) before the system processes the submission.
    • Admin Authorization for AI Services: Only authenticated Admin actors are authorized to access and perform actions within the AI Service and Model Management module (including adding/updating models, toggling status, viewing details/list, configuring thresholds, and viewing activity logs).
    • List Sorting and Ordering: Administrative lists (such as the AI model list and activity logs) are sorted in descending order of their latest timestamp (last modified date for models, creation date for logs), ensuring that the most recent entries are displayed first.
    • Status Default: The model status defaults to active upon creation.
