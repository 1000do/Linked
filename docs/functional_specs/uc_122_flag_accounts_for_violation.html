UC-122. Flag Accounts for Violation

Function Trigger: Admin clicks the "Flag Account" button and enters a violation reason.

Function Description: Allows an Admin to flag a user account for a policy violation. The flag count increments, and the account status transitions through warning levels. On the 3rd flag, the account is automatically banned for 30 days. The system also sends a notification alert to the user.

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • The target account must exist and not be currently banned or already at the max flag limit.
    • Admin has stable internet connection.
  • Process
    • Flag Account Successfully:
      • Admin accesses the Account Management page.
      • Admin clicks the Flag Account button next to the target account.
      • The system displays a dialog box asking for a violation reason.
      • Admin enters the reason and clicks Submit.
      • The system verifies that the Admin is authenticated.
      • The system retrieves the target account and checks that it is eligible for flagging (not banned and flags < 3).
      • The system increments the account flag count by 1 and updates the account status (to flagged_1, flagged_2, or banned).
      • The system saves the changes to the database.
      • The system sends a notification (warning or ban notice depending on flag level) to the target user.
      • The system displays the updated flag count and account status to the Admin.
  • Postconditions
    • The account's flag count and status are updated in the database.
    • If the flag count reaches 3, a 30-day lockout record is created in the database.
    • A notification (warning or ban notice) is sent to the target user account.
    • The updated flag count and status are rendered to the Admin.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
    • Account Not Found: The system detects that the target account ID does not exist. The system displays the error message: "Account not found."
    • Account Ineligible: The system detects that the account is already banned or has reached the maximum flag limit of 3. The system displays an error message indicating that the account cannot be flagged.
    • Database Save Failure: The system detects that saving the flag update to the database failed (rows affected <= 0). The system displays the error message: "Failed to flag account."
  • Security Measures
    • Flag Thresholds and Status Progression:
      • 1st Flag: Status updates to "flagged_1" (warning level 1).
      • 2nd Flag: Status updates to "flagged_2" (warning level 2).
      • 3rd Flag: Status updates to "banned" and triggers an automatic 30-day lockout suspension.
    • Non-Blocking Notifications: The notification delivery process is executed inside a try-catch block. A notification dispatch error must not block or roll back the flagging transaction under the database.
