UC-44. Delete Chats

Function Trigger: Actor clicks the delete icon in the active chat header.

Function Description: Allows the Actor to clear their personal chat history of a conversation. Clearing history only removes the messages from the active participant's perspective by updating their participant cleared timestamp.

Function Details

  • Preconditions
    • Actor must be successfully authenticated.
    • Actor must have an active conversation open.
  • Process
    • Trigger Deletion:
      • Actor accesses the messages page.
      • Actor clicks the "delete" icon in the active conversation header.
      • The system displays a confirmation dialog.
    • Confirm Deletion:
      • Actor clicks the "OK" button.
      • The system requests the backend to clear the chat history.
      • The server validates that the active user is a participant of the chat.
      • The server updates the cleared timestamp of the participant to the current time.
      • The server saves the participant record changes to the database.
      • The system clears the message list from the viewport.
      • The system displays the empty chat state.
      • The system updates the active chat list in the sidebar.
  • Alternative Flow (Cancel Chat Deletion)
    • If the Actor cancels the chat deletion:
      • Actor clicks the "Cancel" button.
      • The system closes the confirmation dialog.
  • Postconditions
    • The participant's cleared timestamp is updated in the database.
    • The message viewport is cleared.
    • The active chat list is refreshed in the sidebar.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Participant Not Found: The system displays an error message regarding chat access permission and halts execution.
  • Security Measures
    • Participant Ownership Verification: The server validates that the requesting Actor is an authorized participant of the chat before modifying any participant records.
