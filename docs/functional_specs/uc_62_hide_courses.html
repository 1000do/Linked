4.10.21. Hide courses

Function Trigger: Instructor clicks the "archive" icon button on a published course.

Function Description: Temporarily hides a course from public display on the marketplace.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Instructor must be authenticated.
    • Instructor must be the owner of the course.
    • Course status must be "Published".
  • Process
    • Hide Course Successfully:
      • Instructor accesses the Instructor Studio dashboard page.
      • The system checks the instructor's session authentication.
      • The system checks the instructor's approval status.
      • The system retrieves the instructor's courses list.
      • The system displays the courses list on the dashboard.
      • Instructor locates the published course they wish to hide.
      • Instructor clicks the "archive" icon button on the corresponding course card.
      • The system displays a confirmation popup dialog prompting the instructor to confirm hiding the course.
      • Instructor clicks the "Yes" button on the popup dialog.
      • The system checks the instructor's session authentication.
      • The system checks if the instructor's rights are locked.
      • The system checks if the course is permanently discontinued.
      • The system checks if the instructor is the owner of the course.
      • The system updates the course status to "archived" in the database.
      • The system clears the cache for the course details.
      • The system displays a success notification indicating the course has been hidden.
      • The system reloads the page dynamically, showing the updated course status badge as "Archived".
  • Postconditions
    • The course status is updated to "Archived" in the database.
    • The course is no longer visible to guests/learners to buy on the public marketplace.
    • Active student enrollments are preserved, allowing enrolled students to continue accessing the course.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Account Not Approved: The system detects the actor's instructor approval status is not "Approved". The system redirects the actor to the application status or apply page. The flow terminates.
    • Instructor Rights Locked: The system detects that the instructor's account rights are locked due to policy violations. The system displays an error message indicating their rights are locked and status cannot be changed. The flow terminates.
    • Course Permanently Discontinued: The system detects the course status is "archived" and it has 3 or more flags. The system displays an error message indicating the course has been permanently discontinued due to policy violations and status cannot be changed. The flow terminates.
    • Unauthorized Access: The system detects that the instructor is not the owner of the course. The system displays an error message indicating they do not have permission to modify this course. The flow terminates.
    • Database Update Failure: The database fails to save the status change (0 rows affected). The system displays a system error message detailing the status update failure. The flow terminates.
  • Security Measures
    • Status Transition Restrictions: An instructor can only set a course to status 'pending' (moderation), 'archived' (hidden), or 'published' (only when unarchiving from status 'archived').
    • Permanently Discontinued Block: A course that is 'archived' and has 3 or more flags is considered permanently discontinued due to policy violations and cannot have its status changed or be edited.
    • Active Enrollment Preservation: Archiving a course does not delete it or impact active student enrollments; active students enrolled in the course continue to have full access to its contents on their dashboards.
    • Public Visibility: The course list displays only courses that are currently published. Draft, pending, rejected, archived, or locked courses are excluded from the public list.
