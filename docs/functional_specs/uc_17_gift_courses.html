2.2.3.17. Gift Courses

Function Trigger: The user clicks the "Gift this course" button on the course details page.

Function Description: Allows the user to configure gift details—including the recipient's name, email, card theme, and a personal message with a live preview—before proceeding to the checkout page.

Function Details

  • Preconditions
    • - The user must be authenticated.
    • - The target course must be retrievable from the system.
  • Process
    • Configure Gift Details Successfully:
      • 1. The user accesses the Gift Setup page.
      • 2. The system displays the gift setup form, the course summary card, and the live gift card preview.
      • 3. User fills in the Recipient's Email, Recipient's Name (Optional), Choose Card Theme, and Gift Message (Optional) fields.
      • 4. User clicks the "Proceed to Payment" button.
      • 5. The system validates all form inputs.
      • 6. The server queries the database to verify the recipient's enrollment status for the selected course.
      • 7. The system redirects the user to the Gift Checkout page.
  • Alternative Flow (Invalid form inputs)
    • 1. The system highlights the invalid fields and displays inline validation error messages.
    • 2. Back to step 3.
  • Alternative Flow (Recipient already enrolled)
    • 1. The system displays a validation error on the recipient email field indicating the recipient already owns the course.
    • 2. Back to step 3.
  • Postconditions
    • - The user is redirected to the Gift Checkout page with all gift details passed as parameters.
  • Error Handling
    • - Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • - Course Not Found: The system fails to retrieve course information from the backend. The system displays an error message. The system redirects the user to the home page.
    • - Recipient Already Enrolled: The system detects via real-time API check that the recipient already has an active enrollment in the course. The system highlights the recipient email field with an error message indicating the recipient already owns the course. The system disables the "Proceed to Payment" button. Back to step 4.
  • Security Measures
    • - Enrollment Pre-check: The system checks recipient enrollment as the user types in the email field (debounced). The "Proceed to Payment" button is disabled until the check confirms the recipient is not enrolled.
