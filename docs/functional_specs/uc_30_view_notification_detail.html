UC-30. View Notification Detail

Function Trigger: User or Instructor clicks on a notification row in the notifications list.

Function Description: Allows users and instructors to view the detailed content of a notification in a modal dialog. If the notification is unread, the system automatically marks it as read.

Function Details

  • Preconditions
    • User or Instructor must be successfully authenticated.
    • The notifications page must be successfully loaded and displaying notifications.
    • User or Instructor has a stable internet connection.
  • Process
    • View Details:
      • User or Instructor clicks on a notification row.
      • The system retrieves the notification details from the client-side list.
      • The system displays the notification details modal.
  • Alternative Flow (Auto-mark notification as read)
    • If the notification is unread:
      • The system detects the notification is unread.
      • The system requests the backend to mark the notification as read.
      • The system updates the notification status to read in the database.
      • The system updates the list UI to display the notification as read.
      • The process returns to displaying the notification details modal.
  • Alternative Flow (Redirect to action link)
    • If the notification has an action link:
      • The system detects the notification has an action link.
      • The system displays the "View Details" button.
      • User or Instructor clicks the "View Details" button.
      • The system redirects the actor to the corresponding action URL.
  • Alternative Flow (Close details modal)
    • If the user closes the modal:
      • User or Instructor clicks the close icon, clicks the "Close" button, or clicks the backdrop area.
      • The system closes the details modal.
  • Postconditions
    • The detailed content of the notification is displayed in the modal.
    • If the notification was unread, its status is updated to read in the database.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Database update failed (marking as read): The system fails to update the notification status in the database, displays a database transaction failure message, and returns to displaying the notification details modal.
  • Security Measures
    • Read Status Sync: Viewing a notification's details modal automatically marks it as read in the database and updates the UI state.
