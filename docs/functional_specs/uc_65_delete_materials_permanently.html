4.10.24. Permanently delete learning materials

Function Trigger: Instructor clicks the "Delete Forever" button next to a removed material on the Trash Bin page.

Function Description: Permanently deletes files from the storage server.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Instructor must be authenticated.
    • Instructor's status must be "Approved".
    • The material must be currently in the trash bin (soft-deleted status is "removed").
  • Process
    • Permanently Delete Learning Material Successfully:
      • Instructor accesses the Trash Bin page (displays the list of removed learning materials).
      • Instructor locates the removed material they wish to delete forever.
      • Instructor clicks the "Delete Forever" button on the corresponding material row.
      • The system displays a confirmation popup dialog warning of irreversible deletion.
      • Instructor clicks the "Yes, delete forever" button.
      • The system checks the instructor's session authentication.
      • The system checks if the instructor's rights are locked.
      • The system checks if the parent course containing the lesson is pending review.
      • The system checks if the parent course is permanently discontinued.
      • The system checks the instructor's ownership of the material.
      • The system deletes the material file from cloud storage.
      • The system deletes the material record from the database.
      • The system clears the course details cache.
      • The system displays a success confirmation message.
      • The system dynamically removes the deleted material row from the list.
  • Postconditions
    • The learning material record is permanently deleted from the database.
    • The material file is permanently deleted from cloud storage (Cloudinary).
    • The material is no longer retrievable by the instructor or any other user.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the actor to the login page.
    • Instructor Rights Locked: The system detects that the instructor's account rights are locked due to policy violations. The system displays an error message indicating their rights are locked and changes cannot be saved. The flow terminates.
    • Course Pending Review: The system detects the course containing the lesson is currently pending review. The system displays an error message indicating curriculum changes are blocked during review. The flow terminates.
    • Course Permanently Discontinued: The system detects the course is permanently discontinued due to policy violations (3 or more flags). The system displays an error message indicating modifications are blocked. The flow terminates.
    • Unauthorized Access: The system detects that the instructor does not own the course/material. The system displays an error message indicating permission denied. The flow terminates.
    • Database Save Failure: The database fails to commit changes. The system displays a system error message. The flow terminates.
  • Security Measures
    • Irreversible Deletion: Permanent deletion removes both the database record and the asset hosted on cloud storage (Cloudinary) and cannot be recovered or restored.
    • Cloud Storage Prefix: The file is deleted from the virtual trash path (prefix `trash/`) in cloud storage using its `CloudPublicId`.
    • Rights & Mod Lockouts: Restoration is locked if the instructor is under active suspension/lockout, or if the parent course is pending review, or if the parent course is permanently discontinued due to 3 or more flags.
