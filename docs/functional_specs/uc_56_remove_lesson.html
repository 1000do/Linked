4.10.15. Remove Lessons

Function Trigger: Instructor clicks the "Remove Lesson" icon for a lesson in the course syllabus.

Function Description: Allows an instructor to remove an existing lesson from a course. The system requires confirmation, validates business rules (such as pending or archived statuses), soft deletes the lesson and its associated materials, and moves any cloud materials to the trash folder.

Function Details

  • Preconditions
    • User must be successfully authenticated as an approved instructor.
    • The course must exist and belong to the active instructor.
  • Process
    • Remove Lesson Successfully:
      • Instructor accesses the course editor.
      • Instructor clicks the "Remove Lesson" icon for a lesson.
      • The system displays a confirmation modal.
      • Instructor clicks the "Delete" button.
      • The system validates the instructor's rights.
      • The system validates the course status.
      • The system moves associated cloud materials to the trash folder.
      • The system updates the materials' learning status to removed.
      • The system updates the lesson status to removed, updates the course status to draft, and clears previous moderation feedback in the database.
      • The system invalidates the cached course data.
      • The system displays a success message.
      • The system removes the lesson from the UI curriculum list.
  • Postconditions
    • The lesson and its associated materials are marked as removed.
    • Associated cloud materials are moved to the trash folder.
    • If the course was previously published, its status is reverted to draft and moderation feedback is cleared.
    • Stale cache data is cleared.
  • Error Handling
    • .1 Session Invalid (Not Logged In): The system displays an authentication error message. The system redirects the user to the login page.
    • .2 Lesson Not Found: The system halts execution and displays a lesson not found error message.
    • Invalid Instructor Rights: The system detects the instructor is unauthorized to modify this lesson or has an active lockout status. The system displays an error message regarding invalid instructor rights. Back to step 2.
    • Invalid Course Status: The system detects the course is pending review or is permanently archived due to policy violations. The system displays an error message regarding invalid course status. Back to step 2.
    • Database Save Failed: The system fails to save the lesson changes to the database. The system displays a database transaction failure message. Back to step 2.
  • Security Measures
    • Locked Out Instructor Rights: Instructors with an active lockout status are prohibited from submitting courses for review or changing course status.
    • Pending Review Modification Lock: Learning materials and lessons cannot be added, edited, or removed if the course is currently in 'Pending' review status.
    • Permanently Discontinued Block: A course that is 'archived' and has 3 or more flags is considered permanently discontinued due to policy violations and cannot have its status changed or be edited.
    • Lesson Soft Deletion: Deleting a lesson soft-deletes the lesson record and all its associated materials (IsRemoved = true) in the database, and moves physical cloud files to the trash.
    • Syllabus Change Reversion: Adding or removing lessons and materials, or editing their content, in a 'Published' course automatically changes the course status back to 'Draft' and clears any existing moderation feedback.
