UC-115. Add Account

Function Trigger: Admin clicks the "Add Staff Member" button, fills in details, and saves.

Function Description: Allows an Admin to register a new Staff account within the account management panel by specifying details.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • Admin has stable internet connection.
  • Process
    • Add Account Successfully:
      • Admin accesses the Manage Accounts page.
      • Admin clicks the Add Staff Member button.
      • The system displays the account creation form.
      • Admin enters the staff details: Email, Display Name, Temporary Password, Phone Number.
      • Admin clicks the Save button.
      • The system checks that the mandatory fields are provided.
      • The system checks if the email is already registered in the database.
      • The system hashes the password using BCrypt.
      • The system saves the new Account details to the database.
      • The system displays a success message and redirects to the account list view.
  • Postconditions
    • A new Account and Manager record (role = "staff") are successfully created in the database.
    • The system displays a success message and redirects to the account list view.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
    • Mandatory Fields Missing: The system detects that Email, Password, or Display Name is empty. The system displays a validation error message.
    • Email Already Registered: The system detects that the entered email address already exists in the database. The system displays the error message: "Email is already registered."
    • Database Save Failure: The system detects that saving the account or manager details to the database failed. The system displays an error message.
  • Security Measures
    • Required Credentials: Email, Password, and Display Name are mandatory fields that cannot be empty.
    • Email Uniqueness: The email must be unique across all accounts in the system.
    • Password Security: The user password must be hashed using BCrypt before database persistence.
    • Auto-Verification: Since the account is directly created by the Admin, the new Staff account is initialized as verified (IsVerified = true) and active (AccountStatus = "Active") by default.
