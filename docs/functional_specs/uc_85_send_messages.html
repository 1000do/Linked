UC-85. Send Messages

Function Trigger: Actor clicks the send button or presses the Enter key in the message input area of the active chat.

Function Description: Allows the Actor to send text messages and optionally attach files (images or documents) within an active support/moderation conversation room. The message is transmitted in real time to the recipient.

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • Actor must have an active conversation open.
  • Process
    • Enter Message:
      • Actor accesses the active conversation panel.
      • Actor enters the message text in the "Type a message..." textarea.
      • Actor clicks the "send" button.
    • Validate and Transmit:
      • The system validates that the message content is not empty.
      • The system sends the message payload to the server via the active connection.
      • The server checks if the sender is an authorized participant of the chat.
      • The server creates a new message record with "ok" status.
      • The server saves the message record to the database.
      • The server increments the unread count for recipients in the cache.
      • The server broadcasts the message to all participants in the chat room.
    • Update UI:
      • The system displays the sent message in the message list viewport with a sent indicator.
      • The system clears the input textarea.
  • Alternative Flow (Attach and Send Files)
    • If the Actor attaches and sends files:
      • Actor clicks the "attach_file" button.
      • The system opens the device's file picker.
      • Actor selects a file.
      • The system displays the selected file in the attachment preview container.
      • Actor clicks the "send" button.
      • The system requests the backend to upload the file.
      • The server saves the file to the storage server and returns the file metadata.
      • The system includes the attachment metadata in the message payload.
  • Alternative Flow (Remove Selected Attachment)
    • If the Actor removes a selected attachment before sending:
      • Actor clicks the "close" icon next to the attached file name in the preview container.
      • The system removes the file from the preview list.
  • Postconditions
    • The message record is saved in the database.
    • The message is broadcasted to all participants in the conversation via SignalR in real time.
    • The unread count of the recipients is updated in the system cache.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, redirects the Actor to the access denied error page, and terminates the flow.
    • Empty Message: The system detects both the message text and selected attachments are empty, cancels the submission, and returns to composing the message.
  • Security Measures
    • Participant Validation: The server verifies that the message sender possesses the necessary permissions (Staff/Admin) and belongs to the conversation room before storing or broadcasting the message.
    • Attachment Safety Validation: The system validates uploaded files for type, extension, and size limit restrictions.
