2.2.3.128. Reject Refund Request

Function Trigger: Admin clicks the "Reject" button on the pending refund request row.

Function Description: Allows administrators to reject a student's pending refund request. The system reverts the transaction status to success, records the administrative rejection note, retains the student's course access, and notifies the student.

Function Details

  • Preconditions
    • The administrator must be successfully logged into the system.
    • The administrator account must have platform-wide financial management permissions.
    • The user must have at least one refund request record.
  • Process
    • Reject Refund Request Successfully:
      • Admin accesses the admin finance dashboard page AdminFinance.
      • Admin selects the "Refund Requests" tab.
      • Admin clicks the "Reject" button on the pending refund request row.
      • The system displays the "Reject Refund" confirmation dialog containing a required feedback input field.
      • Admin inputs the rejection reason and clicks the "Reject Request" button.
      • The system updates the transaction status to "succeeded" and saves the rejection note in the database.
      • The system sends a notification to the student about the rejected refund.
      • The system closes the confirmation dialog and displays a rejection success notification.
  • Alternative Flow (Click Cancel)
    • Admin clicks the "Cancel" button on the confirmation dialog.
    • The system closes the confirmation dialog.
    • Back to step 3.
  • Alternative Flow (Reject Reason Missing)
    • Admin clicks the "Reject Request" button without entering a rejection reason.
    • The system displays a validation error message: "You must enter a rejection reason!".
    • Back to step 5.
  • Postconditions
    • The transaction status is reverted to "succeeded" in the database.
    • The administrator's rejection note is saved in the database.
    • The student's course enrollment remains active in the database.
    • The student receives a notification indicating the rejection of the refund.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the user to the login page.
    • Transaction Not Found: The system displays a transaction not found error message. Back to step 1.
    • Status is Not Pending: The system displays an error message indicating the transaction is not in a pending status. Back to step 2.
    • Transfer Reversal Failed: The system displays an error message regarding the failure to reverse the instructor's transfer. Back to step 3.
    • Stripe Refund Failed: The system displays an error message regarding the failure to process the refund on Stripe. Back to step 3.
  • Security Measures
    • Rejection Reason Required: Unlike approval, rejecting a refund request strictly requires a non-empty administrative explanation to provide clarity to the student.
    • Status Restored to Succeeded: The transaction state returns to succeed as if the refund request was never initiated.
    • No Financial Changes: No funds are transferred, reversed, or refunded via Stripe Connect. The platform commissions and instructor earnings remain intact.
    • Uninterrupted Learning Access: The student's course enrollment status (EnrollmentStatus) remains active, ensuring they continue to have full access to learning materials.
