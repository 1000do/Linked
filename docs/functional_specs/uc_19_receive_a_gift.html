2.2.3.19. Receive a gift

Function Trigger: The user clicks the claim link from their email or in-app notification.

Function Description: Allows the recipient to view the gift card and redeem the gifted course. If the user is not logged in, the system prompts them to log in or register. Once authenticated, the user clicks the claim button to enroll in the course and mark the gift as claimed, which also automatically cancels any pending refund requests for that gift.

Function Details

  • Preconditions
    • - The redemption token must be valid and exist in the system.
    • - The gift must not be already claimed or refunded.
  • Process
    • Receive a Gift Successfully:
      • 1. The user clicks the redemption link in their email or in-app notification.
      • 2. The system validates the redemption token.
      • 3. The system verifies that the user is authenticated.
      • 4. The system displays the customized gift card preview, course information, and the "Claim Your Gift" button.
      • 5. The user clicks the "Claim Your Gift" button.
      • 6. The system verifies that the user does not already own the course.
      • 7. The system creates an active enrollment record for the user.
      • 8. The system updates the gift record as claimed and links it to the user's account in a single database transaction.
      • 9. The system rejects any pending refund request for this gift transaction.
      • 10. The system displays a claim success message and redirects the user to the course study page.
  • Alternative Flow (User is not logged in)
    • 1. The system displays the gift card preview and hides the claim button, showing "Login" and "Register" buttons instead.
    • 2. User clicks the "Login" or "Register" button.
    • 3. The system redirects the user to the login/registration page, passing the gift token as a return URL parameter.
    • 4. User logs in or registers successfully.
    • 5. The system redirects the user back to the Gift Claim page.
    • 6. Back to step 4.
  • Alternative Flow (Recipient already enrolled)
    • 1. The system detects that the claiming user is already enrolled in the course.
    • 2. The system displays an error message: "You already own this course in your library."
    • 3. The system disables the "Claim Your Gift" button.
  • Postconditions
    • - The user is enrolled in the gifted course with active status.
    • - The gift record is marked as claimed and associated with the claiming user's ID.
    • - Any pending refund request for the gift transaction is rejected.
  • Error Handling
    • - Invalid or Expired Token: The system detects that the redemption token does not exist. The system displays a token invalid error message. The system redirects the user to the Claim Error page.
    • - Refunded Gift: The system detects that the gift has been refunded. The system displays an error message indicating the gift is refunded. The system redirects the user to the Claim Error page.
    • - Gift Already Claimed: The system detects that the gift has already been claimed. The system displays a message indicating the gift is already claimed. The system disables the claiming options.
  • Security Measures
    • - Unclaimed Refund Restriction: Gifts can only be claimed if they have not been refunded. A refund reverses the redemption token validity.
    • - Claim and Refund Mutual Exclusion: Once a gift is successfully claimed, any pending refund request for that transaction is automatically rejected and the transaction status remains locked.
    • - No Double Enrollment: A user cannot claim a gift course if they already possess an active enrollment in that course.
