UC-120. Filter Accounts

Function Trigger: Admin selects a role filter value from the table menu.

Function Description: Allows an Admin to filter the displayed account list by selecting specific role criteria, such as User (student), Instructor, Staff, or Registered Staff.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Admin must be logged in with Admin role privileges.
    • Admin is currently on the Account List page.
    • Admin has stable internet connection.
  • Process
    • Filter Accounts Successfully:
      • Admin selects a role filter value from the table menu.
      • The system captures the selected role filter and triggers a query request.
      • The system queries the database to retrieve accounts matching the selected role.
      • The system paginates the results and calculates the total count.
      • The system maps the matching account records to list details.
      • The system renders and displays the filtered Account List table.
  • Postconditions
    • The filtered account list matching the selected role is successfully retrieved and displayed on the screen.
  • Error Handling
    • Session Invalid (Not Logged In): The system detects that the Admin is not authenticated or lacks admin privileges. The system redirects the Admin to the Login page.
  • Security Measures
    • Role Mapping Filters: The dropdown supports filtering specifically by:
      • "user": Accounts representing standard student users.
      • "instructor": Users who have registered and been approved as instructors.
      • "staff": All manager accounts holding the staff role.
      • "registered_staff": Active manager accounts holding the staff role that are verified.
      • "all": Returns all accounts regardless of roles.
