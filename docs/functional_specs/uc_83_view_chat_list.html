UC-83. View Chat List (For System)

Function Trigger: Actor accesses the Chat messages page under the Admin Panel dashboard.

Function Description: Allows the Actor to view their list of active support or moderation conversations, including partner avatar, partner name, last message snippet, timestamp, online status, and unread message count badge.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
  • Process
    • Access Page:
      • Actor accesses the Chat page under the Admin Panel dashboard.
      • The system sends a request to retrieve the list of active support conversations.
    • Retrieve Conversations:
      • The server validates that the active user possesses a Staff or Admin role.
      • The server queries the database for active chat rooms where the current Staff or Admin is a participant and returns the list.
    • Display List:
      • The system displays the list of conversations in the chat sidebar, showing partner details, last message content, last message timestamp, unread message count, and online status.
  • Alternative Flow (Filter by Unread)
    • If the Actor filters by unread messages:
      • Actor selects "Unread" from the message filter dropdown.
      • The system filters the chat sidebar list to show only conversations with an unread message count greater than zero.
  • Alternative Flow (Sort Conversations)
    • If the Actor sorts conversations:
      • Actor selects a sorting option from the sort dropdown.
      • The system re-orders the conversation cards in the sidebar accordingly.
  • Postconditions
    • The system displays the list of active support conversations in the sidebar.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to system support chat conversations is restricted strictly to authenticated users with Staff or Admin roles.
    • Data Access Restriction: The server validates the active user session and role before returning any conversation records from the database.
