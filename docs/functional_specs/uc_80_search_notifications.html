UC-80. Search Notifications

Function Trigger: Actor enters text in the search input field on the System Notifications page.

Function Description: Allows the Actor to search the system notifications log by recipient name, email address, message title, or content.

[Placeholder for UI mockups]

Function Details

  • Preconditions
    • Actor must be successfully authenticated as a Staff member or Admin.
    • Actor is on the System Notifications page.
  • Process
    • Enter Search Query:
      • Actor accesses the System Notifications page.
      • Actor types a search query (such as recipient name, email, title keyword, or content keyword) in the search input box.
    • Filter List:
      • The system filters the notifications list in real time as the text is entered.
      • The system displays the matching notifications in the table, preserving any active role tab filters, status filters, or sorting orders.
  • Alternative Flow (Clear Search Query)
    • If the Actor clears the search query:
      • Actor deletes all text from the search input box.
      • The system filters the notifications list using only the remaining active role tab filters, status filters, and sorting orders.
      • The system displays the updated list in the table.
  • Alternative Flow (No Matching Notifications)
    • If no notifications match the search query:
      • The search query does not match any notifications in the list.
      • The system displays a message indicating "No notifications found matching your filters".
  • Postconditions
    • The system displays only those notifications in the table list that match the search query.
  • Error Handling
    • Session Invalid (Not Logged In): The system displays an authentication error message and redirects the Actor to the login page.
    • Unauthorized Access: The system detects that the Actor is not a Staff or Admin, displays an access denied error page, and terminates the flow.
  • Security Measures
    • Role-based Access Control: Access to search system notifications is restricted strictly to authenticated users with Staff or Admin roles.
    • Input Sanitization: The system sanitizes search inputs to prevent cross-site scripting or injection vulnerabilities.
