No Function Name Sheet Name Description Pre-Condition
1 Report A Course Manage User Report Allows a user to report a course for violations (e.g., misleading content, copyright violation, spam, adult content). The system allows adding multiple reasons, validates the input, and submits the report to the server for moderation. Authenticated enrolled user (not owner/manager). Course is in a reportable state.
2 Report A Review Manage User Report Allows a user to report a course or lesson review for violations such as spam, offensive content, or fake reviews. The system prompts the user to select a reason and optionally provide a description before submitting the report for moderation. Authenticated user.
3 View List of User Reports Manage User Report Displays reported courses, reported course reviews, and reported lesson reviews submitted by users. All category reports are accessible via tabs and subtabs; Actor can switch between category subtabs without a page reload. Authenticated Staff or Admin.
4 View Report Detail Manage User Report Displays details and evidence of a reported violation. Authenticated Staff or Admin.
5 Filter Reports Manage User Report Filters reports by type (Course, Review, Lesson) or status. Authenticated Staff or Admin.
6 Resolve Report Manage User Report Resolves a report by taking moderation action (e.g., removing content) and updating status. Authenticated Staff or Admin. Target report is in a resolvable state.
7 Add AI Model Manage AI Service Allows an Admin to register a new AI moderation model configuration in the system by filling out the model registration form. The system validates all required fields before persisting the new model record to the database. Admin must be successfully authenticated.
8 Update AI Model Manage AI Service Allows an Admin to update an existing AI model's configuration details. The edit form allows modification of provider, version, path, and description. The system validates all required fields before persisting the changes to the database. Admin must be successfully authenticated.
9 Delete AI Model Manage AI Service Allows an Admin to soft-delete an AI model by toggling its status between active and inactive. No data is permanently removed; an active model is set to inactive and an inactive model is restored to active. Admin must be successfully authenticated.
10 View AI Model Details Manage AI Service Displays the full configuration details of a selected AI model in a details dialog. The detail data is pre-loaded with the page and presented client-side without an additional server request. Admin must be successfully authenticated.
11 View AI Model List Manage AI Service Displays a paginated list of all registered AI moderation models, showing each model's name, version, type, process type, status, and last modified date. Admin can navigate between pages using pagination controls. Admin must be successfully authenticated.
12 Set Confidence Threshold Manage AI Service Allows an Admin to configure the spam confidence, toxicity confidence, and similarity score thresholds used by the automated AI moderation system. The current threshold values are pre-filled from the database on page load. If no configuration exists, system defaults are applied. Admin must be successfully authenticated.
13 View Activity Log Manage AI Service Displays chronological paginated logs of AI moderation events across three categories: course moderation, course review moderation, and lesson review moderation. All category logs are loaded at page load; Admin can switch between category subtabs without a page reload. Admin must be successfully authenticated.
14 Add to cart Manage Cart This use case allows an authenticated user to add one or more courses to their shopping cart to purchase them together in a single transaction. User or Instructor must be successfully authenticated and have a verified email status.
15 View Cart items Manage Cart Displays the list of courses added to the active user's shopping cart, calculates the pricing summary (subtotal, coupon discounts, final payable amount), and lists available checkout coupon options. User must be successfully authenticated.
16 Delete cart items Manage Cart Allows authenticated Users or Instructors to remove a course from their shopping cart. User or Instructor must be successfully authenticated and has at least one course item in their shopping cart.
17 View Chat List Chat Allows authenticated Users or Instructors to view their list of active conversations, including partner avatar, partner name, last message snippet, timestamp, online status, and unread message count badge. Actor must be successfully authenticated and approved.
18 Delete Chats Chat Allows the User or Instructor to clear their personal chat history of a conversation. Clearing history only removes the messages from the active participant's perspective by updating their participant cleared timestamp. Actor must be successfully authenticated and have an active conversation open.
19 Send Messages Chat Allows the User or Instructor to send text messages and optionally attach files within an active conversation room. The message is transmitted in real time to other participants. Actor must be successfully authenticated and have an active conversation open.
20 View Chat History Chat Allows the User or Instructor to view the entire message exchange history of a selected conversation, which automatically marks unread messages in the room as read. Actor must be successfully authenticated and have access to the selected chat.
21 Search Chat History Chat Allows the Actor to search their conversation list by sending a query to the backend, which searches participant profiles and returns matching conversation rooms. Actor must be successfully authenticated and is on the Chat messages page.
22 View Chat List Manage Chat Allows the Actor to view their list of active support or moderation conversations, view pending support request tickets, search for specific conversations, filter by unread status, and sort the conversation list. Actor must be successfully authenticated as Staff or Admin.
23 View Chat History Manage Chat Allows authenticated Actor users to view the entire message exchange history of a selected conversation, which automatically marks unread messages in the room as read. Actor must be successfully authenticated as Staff or Admin and have access to the selected chat room.
24 Send Messages Manage Chat Allows authenticated Actor users to send text messages and optionally attach files (images or documents) within an active support/moderation conversation room. The message is transmitted in real time to the recipient. Actor must be successfully authenticated as Staff or Admin and have an active conversation open.
25 Delete Chats Manage Chat Allows authenticated Staff or Admin users to clear their personal chat history of a conversation. Clearing history only removes the messages from the active participant's perspective by updating their participant cleared timestamp. Actor must be successfully authenticated as Staff or Admin and have an active conversation open.
26 Search Chat History Manage Chat Allows authenticated Staff or Admin users to search their conversation list by sending a query to the backend, which searches participant profiles and returns matching conversation rooms. Actor must be successfully authenticated as Staff or Admin and is on the Chat page under the Admin Panel dashboard.
27 View Instructor Applications Manage Instructor Application Allows authenticated Staff and Admin users to view the list of pending instructor registration requests with applicant details, search, filter, and sort options. User must be successfully authenticated as a Staff member or Admin.
28 Approve Instructor Applications Manage Instructor Application Allows authenticated Staff or Admin users to approve an instructor's pending registration application, which promotes their role to Instructor in the database and updates their status. User must be successfully authenticated as a Staff member or Admin and the application must exist in a pending status.
29 Reject Instructor Application Manage Instructor Application Allows authenticated Staff or Admin users to reject an instructor's pending registration application. A detailed rejection reason is required, which is recorded in the database. Users must be successfully authenticated as a Staff member or Admin and the application must exist in a pending status.
30 View Instructor Applications Detail Manage Instructor Application Allows authenticated Staff or Admin users to inspect the detailed credentials of an instructor applicant, including their external LinkedIn, YouTube, and Facebook profiles, and submitted resume or portfolio documents. User must be successfully authenticated as a Staff member or Admin and the application must exist in a pending status.
31 Search Instructor Applications Manage Instructor Application Allows authenticated Staff or Admin users to search the pending instructor applications list by name, email, professional title, or expertise specialization in real-time. User must be successfully authenticated as a Staff member or Admin.
32 Filter Instructor Applications Manage Instructor Application Allows authenticated Staff or Admin users to filter the pending instructor applications list by applicant expertise or specialization. User must be successfully authenticated as a Staff member or Admin.
33 Sort Instructor Applications Manage Instructor Application Allows authenticated Staff or Admin users to sort the pending instructor applications list by application date (newest/oldest) or alphabetical order of the applicant's name. User must be successfully authenticated as a Staff member or Admin.
34 View Notification list Notification Allows users and instructors to view their list of notifications, search, sort, filter, view details, mark notifications as read, mark all notifications as read, or delete notifications. Actor must be successfully authenticated.
35 View Notification Detail Notification Allows users and instructors to view the detailed content of a notification in a modal dialog. If the notification is unread, the system automatically marks it as read. User or Instructor must be successfully authenticated and the notifications page must be successfully loaded.
36 Mark As Read Notification Allows users and instructors to mark their received notifications as read. The system updates the notification status in the database, notifies active client sessions in real-time, and updates the unread count in the UI. User or Instructor must be successfully authenticated.
37 Delete Notifications Notification Allows users and instructors to permanently delete a notification. The system displays a confirmation dialog before sending the request to the database. User or Instructor must be successfully authenticated and the notification must exist.
38 Send Notifications Manage Notification Allows authenticated Staff or Admin users to compose and send notifications either to a specific registered user (using email tags) or broadcasted to all users. Staff or Admin must be successfully authenticated.
39 View Notification List Manage Notification Allows authenticated Staff or Admin to view the list of all system-sent notifications, with search, role tab filtering, status filtering, sorting, page navigation, and detail view modal options. Staff or Admin must be successfully authenticated.
40 View Notification Detail Manage Notification Allows authenticated Staff or Admin to view the detailed recipient, title, content, creation timestamp, and read status of a notification. If a Staff member views an unread notification addressed to themselves, the system automatically marks it as read. Staff or Admin must be successfully authenticated and the target notification must exist.
41 Search Notifications Manage Notification Allows authenticated Staff or Admin to search the system notifications log by recipient name, email address, message title, or content. Staff or Admin must be successfully authenticated and is on the System Notifications page.
42 Sort Notifications Manage Notification Allows authenticated Staff or Admin to sort the system notifications log by timestamp, title, or recipient name. Staff or Admin must be successfully authenticated and is on the System Notifications page.
43 Filter Notifications Manage Notification Allows authenticated Staff or Admin users to filter the system notifications log by recipient role and/or read status. Staff or Admin must be successfully authenticated and is on the System Notifications page.
44 Add wishlist items Manage Wishlist Allows an authenticated User to add a course to their personal wishlist. The system checks if the course is already in the wishlist, creates a wishlist record, saves it to the database, and updates the heart icon state dynamically via AJAX. User must be logged in. The target course must exist and not already be in the User's wishlist.
45 View Wishlist Manage Wishlist Allows an authenticated User to view all courses they have saved in their personal wishlist. For each course, the system retrieves title, thumbnail, price, instructor name, average rating, enrollment count, and check if the User is currently enrolled in it. User must be logged in.
46 Delete Wishlist Items Manage Wishlist Allows an authenticated User to delete a course from their personal wishlist. The system checks if the course is in their wishlist, deletes the record, and reloads the wishlist page. User must be logged in. The target course must exist in the User's wishlist.
